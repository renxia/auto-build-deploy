DM-связывание и контроль доступа: защита вашего AI-помощника ​

Что вы сможете сделать ​

После завершения этого учебника вы сможете:

  • ✅ Понять механизм защиты DM-связывания по умолчанию
  • ✅ Одобрять запросы на связывание от неизвестных отправителей
  • ✅ Просматривать и управлять ожидающими одобрения запросами на связывание
  • ✅ Настраивать различные политики доступа DM (pairing/allowlist/open)
  • ✅ Запускать doctor для проверки конфигурации безопасности

Ваша текущая ситуация ​

Возможно, вы только что настроили канал WhatsApp или Telegram и хотите общаться с AI-помощником, но столкнулись со следующими проблемами:

  • "Почему Clawdbot не отвечает на сообщения от незнакомцев?"
  • "Получил код связывания, не понимаю, что это значит"
  • "Хочу одобрить чей-то запрос, но не знаю, какую команду использовать"
  • "Как узнать, кто сейчас ожидает одобрения?"

Хорошая новость: Clawdbot по умолчанию включает защиту DM-связывания, это гарантирует, что только авторизованные вами отправители могут общаться с AI-помощником.

Когда использовать этот метод ​

Когда вам нужно:

  • 🛡 Защитить приватность: гарантировать, что только доверенные лица могут общаться с AI-помощником
  • ✅ Одобрить незнакомцев: разрешить новым отправителям доступ к вашему AI-помощнику
  • 🔒 Строгий контроль доступа: ограничить права доступа конкретных пользователей
  • 📋 Пакетное управление: просматривать и управлять всеми ожидающими одобрения запросами на связывание

Основная концепция ​

Что такое DM-связывание? ​

Clawdbot подключается к реальным платформам сообщений (WhatsApp, Telegram, Slack и т.д.), и личные сообщения (DM) на этих платформах по умолчанию считаются ненадежным вводом.

Для защиты вашего AI-помощника Clawdbot предоставляет механизм связывания:

Процесс связывания

  1. Незнакомый отправитель отправляет вам сообщение
  2. Clawdbot обнаруживает, что отправитель не авторизован
  3. Clawdbot возвращает код связывания (8 символов)
  4. Отправитель должен предоставить вам код связывания
  5. Вы одобряете код через CLI
  6. ID отправителя добавляется в список разрешений
  7. Отправитель может нормально общаться с AI-помощником

Политика DM по умолчанию ​

Все каналы по умолчанию используют dmPolicy="pairing", это означает:

ПолитикаПоведение
pairingНеизвестные отправители получают код связывания, сообщения не обрабатываются (по умолчанию)
allowlistРазрешены только отправители из списка allowFrom
openРазрешены все отправители (требует явной конфигурации "*")
disabledФункция DM полностью отключена

Предупреждение о безопасности

Режим pairing по умолчанию — самый безопасный выбор. Если у вас нет особых требований, не меняйте его на режим open.

  • [x] Завершили учебник Запуск Gateway

Следуйте инструкциям ​

Шаг 1: Понять происхождение кода связывания ​

Когда неизвестный отправитель отправляет сообщение вашему Clawdbot, он получит ответ, подобный следующему:

Clawdbot: access not configured.

Telegram ID: 123456789

Pairing code: AB3D7X9K

Ask the bot owner to approve with:
clawdbot pairing approve telegram <code>

Ключевые особенности кода связывания (источник: src/pairing/pairing-store.ts):

  • 8 символов: удобно для ввода и запоминания
  • Прописные буквы и цифры: избегает путаницы
  • Исключение легко путаемых символов: не содержит 0, O, 1, I
  • Срок действия 1 час: автоматически становится недействительным по истечении времени
  • Максимум 3 ожидающих запроса: самый старый запрос автоматически очищается при превышении

Шаг 2: Просмотреть список ожидающих запросов на связывание ​

Запустите следующую команду в терминале:

bash
clawdbot pairing list telegram

Вы должны увидеть:

Pairing requests (1)

┌──────────────────┬────────────────┬────────┬──────────────────────┐
│ Code            │ ID            │ Meta   │ Requested            │
├──────────────────┼────────────────┼────────┼──────────────────────┤
│ AB3D7X9K        │ 123456789      │        │ 2026-01-27T10:30:00Z │
└──────────────────┴────────────────┴────────┴──────────────────────┘

Если нет ожидающих одобрения запросов, вы увидите:

No pending telegram pairing requests.

Поддерживаемые каналы

Функция связывания поддерживает следующие каналы:

  • telegram
  • whatsapp
  • slack
  • discord
  • signal
  • imessage
  • msteams
  • googlechat
  • bluebubbles

Шаг 3: Одобрить запрос на связывание ​

Используйте код связывания, предоставленный отправителем, для одобрения доступа:

bash
clawdbot pairing approve telegram AB3D7X9K

Вы должны увидеть:

✅ Approved telegram sender 123456789

Результат после одобрения

После одобрения ID отправителя (123456789) будет автоматически добавлен в список разрешений для этого канала и сохранен в: ~/.clawdbot/credentials/telegram-allowFrom.json

Шаг 4: Уведомить отправителя (опционально) ​

Если вы хотите автоматически уведомить отправителя, используйте флаг --notify:

bash
clawdbot pairing approve telegram AB3D7X9K --notify

Отправитель получит следующее сообщение (источник: src/channels/plugins/pairing-message.ts):

✅ Clawdbot access approved. Send a message to start chatting.

Примечание: Флаг --notify требует, чтобы Clawdbot Gateway был запущен, а канал находился в активном состоянии.

Шаг 5: Проверить, что отправитель может нормально общаться ​

Попросите отправителя снова отправить сообщение, AI-помощник должен нормально ответить.

Контрольная точка ✅ ​

Выполните следующие проверки для подтверждения правильности конфигурации:

  • [ ] Команда clawdbot pairing list <channel> показывает ожидающие запросы
  • [ ] Команда clawdbot pairing approve <channel> <code> успешно одобряет
  • [ ] Одобренный отправитель может нормально общаться с AI-помощником
  • [ ] Код связывания автоматически истекает через 1 час (можно проверить, отправив еще одно сообщение)

Ошибки и их решения ​

Ошибка 1: Код связывания не найден ​

Сообщение об ошибке:

No pending pairing request found for code: AB3D7X9K

Возможные причины:

  • Код связывания уже истек (более 1 часа)
  • Код связывания введен неправильно (проверьте регистр)
  • Отправитель не отправлял сообщение (код связывания генерируется только при получении сообщения)

Решение:

  • Попросите отправителя снова отправить сообщение, чтобы сгенерировать новый код связывания
  • Убедитесь, что код связывания скопирован правильно (обратите внимание на регистр)

Ошибка 2: Канал не поддерживает связывание ​

Сообщение об ошибке:

Channel xxx does not support pairing

Возможные причины:

  • Ошибка в написании названия канала
  • Канал не поддерживает функцию связывания

Решение:

  • Запустите clawdbot pairing list для просмотра списка поддерживаемых каналов
  • Используйте правильное название канала

Ошибка 3: Сбой уведомления ​

Сообщение об ошибке:

Failed to notify requester: <error details>

Возможные причины:

  • Gateway не запущен
  • Соединение с каналом прервано
  • Проблемы с сетью

Решение:

  • Убедитесь, что Gateway запущен
  • Проверьте статус соединения канала: clawdbot channels status
  • Не используйте флаг --notify, уведомите отправителя вручную

Итоги урока ​

Этот учебник представил механизм защиты DM-связывания в Clawdbot:

  • Безопасность по умолчанию: все каналы по умолчанию используют режим pairing, защищая ваш AI-помощник
  • Процесс связывания: неизвестные отправители получают 8-значный код связывания, вам нужно одобрить его через CLI
  • Команды управления:
    • clawdbot pairing list <channel>: просмотр ожидающих запросов
    • clawdbot pairing approve <channel> <code>: одобрение связывания
  • Место хранения: список разрешений хранится в ~/.clawdbot/credentials/<channel>-allowFrom.json
  • Автоматическое истечение: запросы на связывание автоматически становятся недействительными через 1 час

Помните: механизм связывания — это фундамент безопасности Clawdbot, гарантирующий, что только авторизованные вами лица могут общаться с AI-помощником.
