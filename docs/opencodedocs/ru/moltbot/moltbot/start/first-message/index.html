Отправка первого сообщения: общение с ИИ через WebChat или каналы ​

Чему вы научитесь ​

После завершения этого урока вы сможете:

  • ✅ Общаться с помощником ИИ через CLI
  • ✅ Отправлять сообщения через интерфейс WebChat
  • ✅ Общаться с ИИ в настроенных каналах (WhatsApp, Telegram, Slack и других)
  • ✅ Понимать ожидаемые результаты отправки сообщений и коды состояний

Ваша текущая проблема ​

Возможно, вы только что завершили установку Clawdbot и запуск Gateway, но не знаете, как проверить, что всё работает правильно.

Вы можете задаться вопросом:

  • "Gateway запущен, как подтвердить, что он отвечает на сообщения?"
  • "Есть ли графический интерфейс помимо командной строки?"
  • "Я настроил WhatsApp/Telegram, как общаться с ИИ на этих платформах?"

Хорошая новость: Clawdbot предоставляет несколько способов отправки первого сообщения, и один из них вам обязательно подойдёт.

Когда использовать этот метод ​

Когда вам нужно:

  • 🧪 Проверить установку: подтвердить, что Gateway и помощник ИИ работают корректно
  • 🌐 Протестировать каналы: проверить подключение каналов WhatsApp/Telegram/Slack и других
  • 💬 Быстрый диалог: общаться с ИИ напрямую через CLI или WebChat без открытия приложений каналов
  • 🔄 Отправка ответов: отправлять ответы ИИ в определённый канал или контакт

🎒 Подготовка ​

Перед отправкой первого сообщения убедитесь:

Обязательные условия ​

Предварительные курсы

Этот урок предполагает, что вы уже завершили:

  • Быстрый старт - установка, настройка и запуск Clawdbot
  • Запуск Gateway - изучение различных режимов запуска Gateway

Если ещё не завершили, вернитесь к этим урокам.

Опционально: настройка каналов ​

Если вы хотите отправлять сообщения через WhatsApp/Telegram/Slack и другие каналы, сначала настройте их.

Быстрая проверка:

bash
## Просмотр настроенных каналов
clawdbot channels list

Если возвращается пустой список или отсутствует нужный вам канал, обратитесь к соответствующему руководству по настройке (в разделе platforms/).

Основная идея ​

Clawdbot поддерживает три основных способа отправки сообщений:

┌─────────────────────────────────────────────────────────────┐
│              Способы отправки сообщений Clawdbot              │
├─────────────────────────────────────────────────────────────┤
│                                                         │
│  Способ 1: Диалог через CLI Agent                         │
│  ┌─────────────┐                                       │
│  │ clawdbot   │ → Gateway → ИИ → возврат результата     │
│  │ agent        │                                       │
│  │ --message    │                                       │
│  └─────────────┘                                       │
│                                                         │
│  Способ 2: Отправка сообщения в канал через CLI            │
│  ┌─────────────┐                                       │
│  │ clawdbot   │ → Gateway → канал → отправка сообщения   │
│  │ message send │                                       │
│  │ --target     │                                       │
│  └─────────────┘                                       │
│                                                         │
│  Способ 3: WebChat / настроенные каналы                     │
│  ┌─────────────┐               ┌──────────────┐   │
│  │ WebChat     │   или         │ WhatsApp    │   │
│  │ интерфейс браузера  │              │ Telegram    │ → Gateway → ИИ → ответ канала │
│  └─────────────┘               │ Slack       │   │
│                                 │ Discord     │   │
│                                 └──────────────┘   │
└─────────────────────────────────────────────────────────────┘

Ключевые различия:

СпособПройдёт через ИИНазначение
clawdbot agent✅ ДаДиалог с ИИ, получение ответов и процесса мышления
clawdbot message send❌ НетПрямая отправка сообщения в канал без ИИ
WebChat / каналы✅ ДаДиалог с ИИ через графический интерфейс

Выбор подходящего способа

  • Проверка установки: используйте clawdbot agent или WebChat
  • Тестирование каналов: используйте приложения каналов WhatsApp/Telegram
  • Массовая отправка: используйте clawdbot message send (без ИИ)

Следуйте за мной ​

Шаг 1: Общение с ИИ через CLI ​

Зачем CLI — это самый быстрый способ проверки, не требующий открытия браузера или приложений каналов.

Базовый диалог ​

bash
## Отправка простого сообщения помощнику ИИ
clawdbot agent --message "Hello, I'm testing Clawdbot!"

Что вы должны увидеть:

[clawdbot] Thinking...
[clawdbot] Hello! I'm your AI assistant powered by Clawdbot. How can I help you today?

Использование уровня мышления ​

Clawdbot поддерживает различные уровни мышления, контролирующие "прозрачность" ИИ:

bash
## Высокий уровень мышления (показывает полный процесс рассуждений)
clawdbot agent --message "Ship checklist" --thinking high

## Отключить мышление (показать только окончательный ответ)
clawdbot agent --message "What's 2+2?" --thinking off

Что вы должны увидеть (высокий уровень мышления):

[clawdbot] I'll create a comprehensive ship checklist for you.

[THINKING]
Let me think about what needs to be checked for shipping:

1. Code readiness
   - All tests passing?
   - Code review completed?
   - Documentation updated?

2. Build configuration
   - Environment variables set correctly?
   - Build artifacts generated?

[THINKING END]

[clawdbot] 🚢 Ship checklist:
1. Check Node.js version (≥ 22)
2. Install Clawdbot globally
3. Run onboarding wizard
4. Start Gateway
5. Send test message

Опции уровня мышления:

УровеньОписаниеПрименимые сценарии
offНе показывать процесс мышленияПростые вопросы-ответы, быстрый ответ
minimalМинимизировать вывод мыслейОтладка, проверка процессов
lowНизкая подробностьЕжедневные диалоги
mediumСредняя подробностьСложные задачи
highВысокая подробность (включает полный процесс рассуждений)Обучение, генерация кода

Указание канала получения ​

Вы можете отправить ответ ИИ в определённый канал (вместо канала по умолчанию):

bash
## Отправить ответ ИИ в Telegram
clawdbot agent --message "Send me a weather update" --deliver --reply-channel telegram

Полезные параметры

  • --to <номер>: указать номер E.164 получателя (для создания определённого диалога)
  • --agent <id>: использовать определённый ID агента (вместо main по умолчанию)
  • --session-id <id>: продолжить существующий диалог вместо создания нового
  • --verbose on: включить подробный вывод журнала
  • --json: вывод в формате JSON (подходит для разбора скриптами)

Шаг 2: Отправка сообщений через интерфейс WebChat ​

Зачем WebChat предоставляет графический интерфейс в браузере, более интуитивный, поддерживает форматированный текст и вложения.

Доступ к WebChat ​

WebChat использует службу WebSocket Gateway, не требует отдельной настройки или дополнительного порта.

Способы доступа:

  1. Откройте браузер, перейдите к: http://localhost:18789
  2. Или выполните в терминале: clawdbot dashboard (автоматически откроет браузер)

Порт WebChat

WebChat использует тот же порт, что и Gateway (по умолчанию 18789). Если вы изменили порт Gateway, WebChat также будет использовать этот порт.

Что вы должны увидеть:

┌─────────────────────────────────────────────┐
│          Clawdbot WebChat              │
│  ┌───────────────────────────────────┐   │
│  │  Привет! Я ваш ИИ-помощник.       │   │
│  │  Чем я могу вам помочь?        │   │
│  └───────────────────────────────────┘   │
│  [Поле ввода...                       │   │
│  [Отправить]                            │   │
└─────────────────────────────────────────────┘

Отправка сообщения ​

  1. Введите ваше сообщение в поле ввода
  2. Нажмите "Отправить" или нажмите Enter
  3. Подождите ответа ИИ

Что вы должны увидеть:

  • Ответ ИИ отображается в интерфейсе чата
  • Если включён уровень мышления, будет показана метка [THINKING]

Функции WebChat:

ФункцияОписание
Форматированный текстПоддержка формата Markdown
ВложенияПоддержка загрузки изображений, аудио, видео
ИсторияАвтоматическое сохранение истории диалогов
Переключение диалоговПереключение между разными диалогами на левой панели

Приложение в меню macOS

Если вы установили приложение Clawdbot для macOS, вы также можете открыть WebChat напрямую из кнопки "Open WebChat" в меню.

Шаг 3: Отправка сообщений через настроенные каналы ​

Зачем Проверить правильность подключения каналов (WhatsApp, Telegram, Slack и других) и почувствовать реальный межплатформенный диалог.

Пример WhatsApp ​

Если вы настроили WhatsApp в onboarding или настройках:

  1. Откройте приложение WhatsApp (мобильное или desktop)
  2. Найдите ваш номер Clawdbot (или сохранённый контакт)
  3. Отправьте сообщение: Hello from WhatsApp!

Что вы должны увидеть:

[WhatsApp]
Вы → Clawdbot: Hello from WhatsApp!

Clawdbot → Вы: Hello! I received your message via WhatsApp.
How can I help you today?

Пример Telegram ​

Если вы настроили Telegram Bot:

  1. Откройте приложение Telegram
  2. Найдите ваш Bot (используя имя пользователя)
  3. Отправьте сообщение: /start или Hello from Telegram!

Что вы должны увидеть:

[Telegram]
Вы → @your_bot: /start

@your_bot → Вы: Welcome! I'm Clawdbot's AI assistant.
You can talk to me here, and I'll respond via AI.

Примеры Slack/Discord ​

Для Slack или Discord:

  1. Откройте соответствующее приложение
  2. Найдите канал или сервер, где находится Bot
  3. Отправьте сообщение: Hello from Slack!

Что вы должны увидеть:

  • Bot отвечает на ваше сообщение
  • Перед сообщением может отображаться метка "AI Assistant"

Защита сопряжения DM

По умолчанию Clawdbot включает защиту сопряжения DM:

  • Неизвестные отправители получают код сопряжения
  • Сообщения не обрабатываются, пока вы не утвердите сопряжение

Если вы отправляете сообщение из канала впервые, возможно, потребуется:

bash
## Просмотр запросов на сопряжение, ожидающих утверждения
clawdbot pairing list

## Утверждение запроса на сопряжение (замените <channel> и <code> на реальные значения)
clawdbot pairing approve <channel> <code>

Подробности: Сопряжение DM и контроль доступа

Шаг 4 (опционально): Прямая отправка сообщений в канал ​

Зачем Отправить сообщения в канал без ИИ. Подходит для массовых уведомлений, push-сообщений и других сценариев.

Отправка текстового сообщения ​

bash
## Отправка текстового сообщения в WhatsApp
clawdbot message send --target +15555550123 --message "Hello from CLI!"

Отправка сообщения с вложением ​

bash
## Отправка изображения
clawdbot message send --target +15555550123 \
  --message "Check out this photo" \
  --media ~/Desktop/photo.jpg

## Отправка изображения по URL
clawdbot message send --target +15555550123 \
  --message "Here's a link" \
  --media https://example.com/image.png

Что вы должны увидеть:

[clawdbot] Message sent successfully
[clawdbot] Message ID: 3EB0A1234567890

Полезные параметры message send

  • --channel: указать канал (по умолчанию: whatsapp)
  • --reply-to <id>: ответить на указанное сообщение
  • --thread-id <id>: ID темы Telegram
  • --buttons <json>: инлайн-кнопки Telegram (формат JSON)
  • --card <json>: Adaptive Card (поддерживаемые каналы)

Распространённые вопросы

Q: ИИ не отвечает на мои сообщения?

A: Проверьте следующие пункты:

  1. Запущен ли Gateway: clawdbot gateway status
  2. Настроена ли модель ИИ: clawdbot models list
  3. Просмотрите подробный журнал: clawdbot agent --message "test" --verbose on

Q: WebChat не открывается?

A: Проверьте:

  1. Запущен ли Gateway
  2. Правильный ли порт: по умолчанию 18789
  3. Открыт ли браузер http://127.0.0.1:18789 (а не localhost)

Q: Не удаётся отправить сообщение в канал?

A: Проверьте:

  1. Выполнен ли вход в канал: clawdbot channels status
  2. Работает ли сетевое подключение
  3. Просмотрите журнал ошибок конкретного канала: clawdbot gateway --verbose

Предупреждения о подводных камнях ​

❌ Gateway не запущен ​

Неправильный подход:

bash
clawdbot agent --message "Hello"
## Ошибка: Gateway connection failed

Правильный подход:

bash
## Сначала запустите Gateway
clawdbot gateway --port 18789

## Затем отправьте сообщение
clawdbot agent --message "Hello"

Gateway должен быть запущен первым

Все способы отправки сообщений (CLI, WebChat, каналы) зависят от службы WebSocket Gateway. Убедитесь, что Gateway работает — это первый шаг.

❌ Канал не авторизован ​

Неправильный подход:

bash
## Отправка сообщения без входа в WhatsApp
clawdbot message send --target +15555550123 --message "Hi"
## Ошибка: WhatsApp not authenticated

Правильный подход:

bash
## Сначала войдите в канал
clawdbot channels login whatsapp

## Проверьте статус
clawdbot channels status

## Затем отправьте сообщение
clawdbot message send --target +15555550123 --message "Hi"

❌ Забыли сопряжение DM ​

Неправильный подход:

bash
## Первая отправка сообщения из Telegram, но сопряжение не утверждено
## Результат: Bot получает сообщение, но не обрабатывает его

Правильный подход:

bash
## 1. Просмотр запросов на сопряжение, ожидающих утверждения
clawdbot pairing list

## 2. Утверждение сопряжения
clawdbot pairing approve telegram ABC123
## 3. Отправьте сообщение снова

### Теперь сообщение будет обработано и вы получите ответ ИИ

❌ Путаница между agent и message send ​

Неправильный подход:

bash
## Хотите поговорить с ИИ, но используете message send
clawdbot message send --target +15555550123 --message "Help me write code"
## Результат: сообщение отправлено напрямую в канал, ИИ его не обрабатывает

Правильный подход:

bash
## Диалог с ИИ: используйте agent
clawdbot agent --message "Help me write code" --to +15555550123

## Прямая отправка сообщения: используйте message send (без ИИ)
clawdbot message send --target +15555550123 --message "Meeting at 3pm"
