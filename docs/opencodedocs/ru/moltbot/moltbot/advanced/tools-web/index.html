Инструменты веб-поиска и скрейпинга ​

Что вы сможете сделать после ​

  • Настроить инструмент web_search, чтобы ИИ-ассистент мог использовать Brave Search или Perplexity Sonar для веб-поиска
  • Настроить инструмент web_fetch, чтобы ИИ-ассистент мог выполнять скрейпинг и извлечение веб-контента
  • Понять разницу между двумя инструментами и их сценарии использования
  • Настроить API Key и расширенные параметры (регион, язык, время кеширования и т.д.)
  • Решать общие проблемы (ошибки API Key, сбои скрейпинга, проблемы с кешем и т.д.)

Ваша текущая проблема ​

База знаний ИИ-ассистента статична и не может получать доступ к веб-информации в реальном времени:

  • ИИ не знает новости сегодняшнего дня
  • ИИ не может искать последние документы API или технические блоги
  • ИИ не может получить последнее содержимое определённых веб-сайтов

Вы хотите, чтобы ИИ-ассистент "пошёл в интернет", но не знаете:

  • Следует ли использовать Brave или Perplexity?
  • Где получить API Key? Как его настроить?
  • В чём разница между web_search и web_fetch?
  • Как работать с динамическими веб-страницами или сайтами, требующими входа?

Когда использовать этот метод ​

  • web_search: Когда нужно быстро найти информацию, искать на нескольких веб-сайтах, получать данные в реальном времени (например, новости, цены, погоду)
  • web_fetch: Когда нужно извлечь полное содержимое определённой веб-страницы, читать страницы документации, анализировать статьи блога

Руководство по выбору инструментов

СценарийРекомендуемый инструментПричина
Поиск нескольких источниковweb_searchВозвращает несколько результатов в одном запросе
Извлечение содержимого одной страницыweb_fetchПолучает полный текст, поддерживает markdown
Динамические страницы/требуют входаbrowserТребуется выполнение JavaScript
Простые статические страницыweb_fetchЛёгкий и быстрый

🎒 Подготовка перед началом ​

Предварительные требования

Этот учебник предполагает, что вы завершили Быстрый старт, установили и запустили Gateway.

  • Демон Gateway запущен
  • Базовая настройка каналов завершена (доступен хотя бы один канал связи)
  • API Key хотя бы одного поставщика поиска подготовлена (Brave или Perplexity/OpenRouter)

Примечание

web_search и web_fetch — это лёгкие инструменты, которые не выполняют JavaScript. Для веб-сайтов, требующих входа, или сложных динамических страниц используйте инструмент browser.

Основные концепции ​

Разница между двумя инструментами ​

web_search: Инструмент веб-поиска

  • Вызывает поисковые системы (Brave или Perplexity) для возврата результатов поиска
  • Brave: Возвращает структурированные результаты (заголовок, URL, описание, дата публикации)
  • Perplexity: Возвращает ответы, синтезированные ИИ, со ссылками на цитаты

web_fetch: Инструмент скрейпинга веб-контента

  • Отправляет HTTP GET-запросы на определённый URL
  • Использует алгоритм Readability для извлечения основного содержимого (удаляя навигацию, рекламу и т.д.)
  • Преобразует HTML в Markdown или простой текст
  • Не выполняет JavaScript

Почему нам нужны два инструмента? ​

┌─────────────────┐     web_search      ┌──────────────────┐
│  Пользователь спрашивает у ИИ│ ──────────────────→  │   API поисковой системы   │
│ "Последние новости"│                      │   (Brave/Perplexity) │
└─────────────────┘                      └──────────────────┘
        ↓                                        ↓
   ИИ получает 5 результатов                    Возвращает результаты поиска
        ↓
┌─────────────────┐     web_fetch       ┌──────────────────┐
│  ИИ выбирает результат│ ──────────────────→  │   Целевая веб-страница   │
│ "Открыть ссылку 1"│                      │   (HTTP/HTTPS)   │
└─────────────────┘                      └──────────────────┘
        ↓                                        ↓
   ИИ получает полное содержимое                    Извлекает Markdown

Типичный рабочий процесс:

  1. ИИ использует web_search для поиска соответствующей информации
  2. ИИ выбирает соответствующие ссылки из результатов поиска
  3. ИИ использует web_fetch для скрейпинга содержимого конкретной страницы
  4. ИИ отвечает на вопрос пользователя на основе содержимого

Механизм кеширования ​

Оба инструмента включают встроенный кеш для уменьшения дублирующих запросов:

ИнструментКлюч кешаTTL по умолчаниюЭлемент конфигурации
web_searchprovider:query:count:country:search_lang:ui_lang:freshness15 минутtools.web.search.cacheTtlMinutes
web_fetchfetch:url:extractMode:maxChars15 минутtools.web.fetch.cacheTtlMinutes

Преимущества кеширования

  • Уменьшает количество внешних API-вызовов (экономит затраты)
  • Ускоряет время отклика (один и тот же запрос возвращает кеш напрямую)
  • Избегает ограничения скорости частыми запросами

Следуйте за мной ​

Шаг 1: Выбор поставщика поиска ​

Clawdbot поддерживает два поставщика поиска:

ПоставщикПреимуществаНедостаткиAPI Key
Brave (по умолчанию)Быстрый, структурированные результаты, бесплатный уровеньТрадиционные результаты поискаBRAVE_API_KEY
PerplexityСинтезированные ИИ ответы, цитаты, в реальном времениТребует доступ Perplexity или OpenRouterOPENROUTER_API_KEY или PERPLEXITY_API_KEY

Рекомендуемый выбор

  • Начинающим: Рекомендуется использовать Brave (бесплатный уровень достаточен для ежедневного использования)
  • Требуется сводка ИИ: Выберите Perplexity (возвращает синтезированные ответы вместо оригинальных результатов)

Шаг 2: Получение API Key от Brave Search ​

Почему использовать Brave: Щедрый бесплатный уровень, быстрый, структурированные результаты легко анализировать

2.1 Зарегистрироваться в Brave Search API ​

  1. Посетите https://brave.com/search/api/
  2. Создайте аккаунт и войдите
  3. В Dashboard выберите план "Data for Search" (не "Data for AI")
  4. Сгенерируйте API Key

2.2 Настройка API Key ​

Метод A: Использование CLI (рекомендуется)

bash
# Запустить интерактивный мастер настройки
clawdbot configure --section web

CLI попросит вас ввести API Key и сохранит его в ~/.clawdbot/clawdbot.json.

Метод B: Использование переменных окружения

Добавьте API Key к переменным окружения процесса Gateway:

bash
# Добавить в ~/.clawdbot/.env
echo "BRAVE_API_KEY=вашAPIKey" >> ~/.clawdbot/.env

# Перезапустить Gateway для вступления переменных окружения в силу
clawdbot gateway restart

Метод C: Непосредственное редактирование файла конфигурации

Отредактируйте ~/.clawdbot/clawdbot.json:

json5
{
  "tools": {
    "web": {
      "search": {
        "apiKey": "BRAVE_API_KEY_HERE",
        "provider": "brave"
      }
    }
  }
}

Что вы должны увидеть:

  • После сохранения конфигурации перезапустите Gateway
  • В настроенном канале (например, WhatsApp) отправьте сообщение: "Помогите мне найти последние новости ИИ"
  • ИИ должен вернуть результаты поиска (заголовок, URL, описание)

Шаг 3: Настройка расширенных параметров web_search ​

Вы можете настроить больше параметров в ~/.clawdbot/clawdbot.json:

json5
{
  "tools": {
    "web": {
      "search": {
        "enabled": true,           // Включено ли (по умолчанию true)
        "provider": "brave",       // Поставщик поиска
        "apiKey": "BRAVE_API_KEY_HERE",
        "maxResults": 5,          // Количество возвращаемых результатов (1-10, по умолчанию 5)
        "timeoutSeconds": 30,       // Тайм-аут (по умолчанию 30)
        "cacheTtlMinutes": 15      // Время кеша (по умолчанию 15 минут)
      }
    }
  }
}

3.1 Настройка региона и языка ​

Сделайте результаты поиска более точными:

json5
{
  "tools": {
    "web": {
      "search": {
        "provider": "brave",
        "apiKey": "BRAVE_API_KEY_HERE",
        "maxResults": 10,
        // Опционально: ИИ может переопределить эти значения при вызове
        "defaultCountry": "US",   // Страна по умолчанию (двухбуквенный код)
        "defaultSearchLang": "en",  // Язык результатов поиска
        "defaultUiLang": "en"      // Язык элементов интерфейса
      }
    }
  }
}

Общие коды стран: US (США), DE (Германия), FR (Франция), CN (Китай), JP (Япония), ALL (Глобальный)

Общие коды языков: en (английский), zh (китайский), fr (французский), de (немецкий), es (испанский)

3.2 Настройка фильтра времени (эксклюзивно Brave) ​

json5
{
  "tools": {
    "web": {
      "search": {
        "provider": "brave",
        "apiKey": "BRAVE_API_KEY_HERE",
        // Опционально: ИИ может переопределить при вызове
        "defaultFreshness": "pw"  // Фильтровать результаты за последнюю неделю
      }
    }
  }
}

Значения Freshness:

  • pd: Последние 24 часа
  • pw: Последняя неделя
  • pm: Последний месяц
  • py: Последний год
  • YYYY-MM-DDtoYYYY-MM-DD: Пользовательский диапазон дат (например, 2024-01-01to2024-12-31)

Шаг 4: Настройка Perplexity Sonar (опционально) ​

Если вы предпочитаете ответы, синтезированные ИИ, вы можете использовать Perplexity.

4.1 Получение API Key ​

Метод A: Прямое подключение к Perplexity

  1. Посетите https://www.perplexity.ai/
  2. Создайте аккаунт и подпишитесь
  3. Сгенерируйте API Key в Settings (начинается с pplx-)

Метод B: Через OpenRouter (не требуется кредитная карта)

  1. Посетите https://openrouter.ai/
  2. Создайте аккаунт и пополните баланс (поддерживает криптовалюту или предоплату)
  3. Сгенерируйте API Key (начинается с sk-or-v1-)

4.2 Настройка Perplexity ​

Отредактируйте ~/.clawdbot/clawdbot.json:

json5
{
  "tools": {
    "web": {
      "search": {
        "enabled": true,
        "provider": "perplexity",
        "perplexity": {
          // API Key (опционально, также можно настроить через переменные окружения)
          "apiKey": "sk-or-v1-...",  // или "pplx-..."
          // Base URL (опционально, Clawdbot автоматически определит по API Key)
          "baseUrl": "https://openrouter.ai/api/v1",  // или "https://api.perplexity.ai"
          // Модель (по умолчанию perplexity/sonar-pro)
          "model": "perplexity/sonar-pro"
        }
      }
    }
  }
}

Автоматическое определение Base URL

Если вы опустите baseUrl, Clawdbot автоматически выберет в зависимости от префикса API Key:

  • pplx-... → https://api.perplexity.ai
  • sk-or-... → https://openrouter.ai/api/v1

4.3 Выбор модели Perplexity ​

МодельОписаниеСценарий использования
perplexity/sonarБыстрые Q&A + веб-поискПростые запросы, быстрый поиск
perplexity/sonar-pro (по умолчанию)Многоэтапное рассуждение + веб-поискСложные проблемы, требуется рассуждение
perplexity/sonar-reasoning-proАнализ цепочки мыслейГлубокое исследование, требуется процесс рассуждения

Шаг 5: Настройка инструмента web_fetch ​

web_fetch включён по умолчанию и может использоваться без дополнительной настройки. Но вы можете настроить параметры:

json5
{
  "tools": {
    "web": {
      "fetch": {
        "enabled": true,           // Включено ли (по умолчанию true)
        "maxChars": 50000,        // Максимальное количество символов (по умолчанию 50000)
        "timeoutSeconds": 30,       // Тайм-аут (по умолчанию 30)
        "cacheTtlMinutes": 15,     // Время кеша (по умолчанию 15 минут)
        "maxRedirects": 3,         // Максимальное количество перенаправлений (по умолчанию 3)
        "userAgent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 14_7_2) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/122.0.0.0 Safari/537.36",
        "readability": true         // Включить ли Readability (по умолчанию true)
      }
    }
  }
}

5.1 Настройка резервного копирования Firecrawl (опционально) ​

Если извлечение Readability не удаётся, вы можете использовать Firecrawl как резервное копирование (требуется API Key):

json5
{
  "tools": {
    "web": {
      "fetch": {
        "readability": true,
        "firecrawl": {
          "enabled": true,
          "apiKey": "FIRECRAWL_API_KEY_HERE",  // или установить переменную окружения FIRECRAWL_API_KEY
          "baseUrl": "https://api.firecrawl.dev",
          "onlyMainContent": true,  // Извлекать только основное содержимое
          "maxAgeMs": 86400000,    // Время кеша (миллисекунды, по умолчанию 1 день)
          "timeoutSeconds": 60
        }
      }
    }
  }
}

Преимущества Firecrawl

  • Поддерживает рендеринг JavaScript (требует включения)
  • Более сильная защита от анти-скрейпинга
  • Поддерживает сложные веб-сайты (SPA, одностраничные приложения)

Получение API Key Firecrawl:

  1. Посетите https://www.firecrawl.dev/
  2. Создайте аккаунт и сгенерируйте API Key
  3. Настройте в конфигурации или используйте переменную окружения FIRECRAWL_API_KEY

Избегание ловушек ​

Общая ошибка 1: API Key не настроена ​

Сообщение об ошибке:

json
{
  "error": "missing_brave_api_key",
  "message": "web_search needs a Brave Search API key. Run `clawdbot configure --section web` to store it, or set BRAVE_API_KEY in Gateway environment."
}

Решение:

  1. Выполните clawdbot configure --section web
  2. Введите API Key
  3. Перезапустите Gateway: clawdbot gateway restart

Общая ошибка 2: Сбой скрейпинга (динамические веб-страницы) ​

Проблема: web_fetch не может извлечь содержимое, требующее JavaScript.

Решение:

  1. Подтвердите, является ли веб-сайт SPA (одностраничным приложением)
  2. Если да, используйте инструмент browser
  3. Или настройте резервное копирование Firecrawl (требуется API Key)

Общая ошибка 3: Устаревшее содержимое из-за кеша ​

Проблема: Результаты поиска или извлечённое содержимое старые.

Решение:

  1. Настройте cacheTtlMinutes
  2. Или явно запросите "не использовать кеш" в диалоге с ИИ
  3. Перезапустите Gateway для очистки кеша в памяти

Общая ошибка 4: Истечение времени ожидания запроса ​

Проблема: Тайм-аут при скрейпинге больших страниц или медленных веб-сайтов.

Решение:

json5

Общая ошибка 5: Внутренний сетевой IP заблокирован SSRF ​

Проблема: Скрейпинг к внутренним сетевым адресам (например, http://localhost:8080) заблокирован.

Решение:

web_fetch по умолчанию блокирует внутренние сетевые IP-адреса для предотвращения атак SSRF. Если вам действительно нужен доступ к внутренней сети:

  1. Используйте инструмент browser (более гибкий)
  2. Или отредактируйте конфигурацию для разрешения определённых хостов (требует изменения исходного кода)

Итог этого урока ​

  • web_search: Инструмент веб-поиска, поддерживает Brave (структурированные результаты) и Perplexity (ответы, синтезированные ИИ)
  • web_fetch: Инструмент скрейпинга веб-контента, использует Readability для извлечения основного содержимого (HTML → Markdown/text)
  • Оба включают встроенный кеш (по умолчанию 15 минут), уменьшают дублирующие запросы
  • API Key Brave можно настроить через CLI, переменные окружения или файл конфигурации
  • Perplexity поддерживает два метода: прямое подключение и OpenRouter
  • Для веб-сайтов, требующих JavaScript, используйте инструмент browser
  • Параметры конфигурации включают: количество результатов, время ожидания, регион, язык, фильтр времени и т.д.
