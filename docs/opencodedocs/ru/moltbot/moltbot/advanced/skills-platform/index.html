Платформа навыков и ClawdHub для расширения ИИ-ассистента | Учебник Clawdbot ​

Что вы сможете сделать после этого урока ​

После завершения этого урока вы сможете:

  • Понять архитектуру системы навыков Clawdbot (три типа навыков: Bundled, Managed, Workspace)
  • Обнаруживать, устанавливать и обновлять навыки через ClawdHub для расширения возможностей вашего ИИ-ассистента
  • Настраивать состояние активации, переменные окружения и ключи API навыков
  • Использовать правила gating для обеспечения того, чтобы навыки загружались только при выполнении условий
  • Управлять совместным использованием и приоритетом перезаписи навыков в сценариях с несколькими агентами

Ваша текущая проблема ​

Clawdbot уже предоставляет богатый набор встроенных инструментов (браузер, выполнение команд, веб-поиск и т.д.), но когда вам нужно:

  • Вызывать сторонние инструменты CLI (например, gemini, peekaboo)
  • Добавлять сценарии автоматизации для конкретной области
  • Заставить ИИ научиться использовать ваш набор пользовательских инструментов

Вы можете подумать: "Как сообщить ИИ, какие инструменты доступны? Где следует разместить эти инструменты? Могут ли несколько агентов совместно использовать навыки?"

Система навыков Clawdbot разработана именно для этого: объявите навыки через файл SKILL.md, и агент автоматически загрузит и использует их.

Когда использовать этот подход ​

  • Когда вам нужно расширить возможности ИИ: вы хотите добавить новые инструменты или процессы автоматизации
  • При совместной работе нескольких агентов: разные агенты должны совместно использовать или иметь эксклюзивный доступ к навыкам
  • При управлении версиями навыков: устанавливать, обновлять и синхронизировать навыки через ClawdHub
  • При gating навыков: гарантировать, что навыки загружаются только в определённых средах (ОС, бинарные файлы, конфигурация)

🎒 Подготовка перед началом ​

Перед началом подтвердите:

  • [ ] Вы завершили Быстрый старт и Gateway работает нормально
  • [ ] Вы настроили хотя бы одну модель ИИ (Anthropic, OpenAI, Ollama и т.д.)
  • [ ] Вы понимаете основные операции командной строки (mkdir, cp, rm)

Основные концепции ​

Что такое навык? ​

Навык - это каталог, содержащий файл SKILL.md (инструкции для LLM и определения инструментов), а также необязательные скрипты или ресурсы. Файл SKILL.md использует YAML frontmatter для определения метаданных и Markdown для описания использования навыка.

Clawdbot совместим со спецификацией AgentSkills, поэтому навыки могут быть загружены другими инструментами, которые следуют этой спецификации.

Три места загрузки навыков ​

Навыки загружаются из трёх мест в порядке убывания приоритета:

  1. Навыки Workspace: <workspace>/skills (наивысший приоритет)
  2. Навыки Managed/локальные: ~/.clawdbot/skills
  3. Навыки Bundled: поставляются с установочным пакетом (наименьший приоритет)

Правило приоритета

Если навык с одинаковым именем существует в нескольких местах, навыки Workspace заменяют навыки Managed и Bundled.

Кроме того, вы можете настроить дополнительные каталоги навыков через skills.load.extraDirs (наименьший приоритет).

Совместное использование навыков в сценариях с несколькими агентами ​

В конфигурации с несколькими агентами каждый агент имеет свой собственный workspace:

  • Навыки Per-agent: расположены в <workspace>/skills, видны только для этого агента
  • Совместные навыки: расположены в ~/.clawdbot/skills, видны для всех агентов на одной машине
  • Общая папка: может быть добавлена через skills.load.extraDirs (наименьший приоритет), используется для совместного использования одного и того же пакета навыков несколькими агентами

В случае конфликта имён правило приоритета также применяется: Workspace > Managed > Bundled.

Gating навыков (Gating) ​

Clawdbot фильтрует навыки на основе поля metadata при загрузке, гарантируя, что навыки загружаются только при выполнении условий:

markdown
---
name: nano-banana-pro
description: Generate or edit images via Gemini 3 Pro Image
metadata: {"clawdbot":{"requires":{"bins":["uv"],"env":["GEMINI_API_KEY"],"config":["browser.enabled"]},"primaryEnv":"GEMINI_API_KEY"}}
---

Поля под metadata.clawdbot:

  • always: true: всегда загружать навык (пропустить другие gating)
  • emoji: эмодзи, отображаемый в интерфейсе навыков macOS
  • homepage: ссылка на веб-сайт, отображаемую в интерфейсе навыков macOS
  • os: список платформ (darwin, linux, win32), навык доступен только на этих операционных системах
  • requires.bins: список, каждый должен существовать в PATH
  • requires.anyBins: список, по крайней мере один должен существовать в PATH
  • requires.env: список, переменные окружения должны существовать или быть предоставлены в конфигурации
  • requires.config: список путей clawdbot.json, должны быть истинными
  • primaryEnv: имя переменной окружения, связанной с skills.entries.<name>.apiKey
  • install: массив спецификаций установщика (для интерфейса навыков macOS)

Проверка бинарных файлов в среде sandbox

requires.bins проверяется на хосте при загрузке навыка. Если агент выполняется в sandbox, бинарный файл также должен существовать внутри контейнера. Зависимости могут быть установлены через agents.defaults.sandbox.docker.setupCommand.

Внедрение переменных окружения ​

Когда начинается выполнение агента, Clawdbot:

  1. Читает метаданные навыка
  2. Применяет все skills.entries.<key>.env или skills.entries.<key>.apiKey к process.env
  3. Строит системный промпт, используя навыки, которые соответствуют условиям
  4. Восстанавливает исходную среду после завершения выполнения агента

Это ограничено областью выполнения агента, а не глобальной средой Shell.

Выполните следующие действия ​

Шаг 1: Просмотр установленных навыков ​

Используйте CLI для просмотра списка доступных навыков:

bash
clawdbot skills list

Или просмотреть только навыки, которые соответствуют условиям:

bash
clawdbot skills list --eligible

Вы должны увидеть: список навыков, включая название, описание, соответствуют ли условиям (например, бинарные файлы, переменные окружения)

Шаг 2: Установка навыков через ClawdHub ​

ClawdHub - это публичный реестр навыков Clawdbot, где вы можете просматривать, устанавливать, обновлять и публиковать навыки.

Установка CLI ​

Выберите один из методов для установки ClawdHub CLI:

bash
npm i -g clawdhub

или

bash
pnpm add -g clawdhub

Поиск навыков ​

bash
clawdhub search "postgres backups"

Установка навыка ​

bash
clawdhub install <skill-slug>

По умолчанию CLI устанавливает в подкаталог ./skills текущего рабочего каталога (или возвращается к настроенному workspace Clawdbot). Clawdbot загрузит его как <workspace>/skills в следующей сессии.

Вы должны увидеть: вывод установки, показывающий папку навыка и информацию о версии.

Шаг 3: Обновление установленных навыков ​

Обновить все установленные навыки:

bash
clawdhub update --all

Или обновить определённый навык:

bash
clawdhub update <slug>

Вы должны увидеть: статус обновления каждого навыка, включая изменения версий.

Шаг 4: Настройка перезаписи навыков ​

В ~/.clawdbot/clawdbot.json настройте состояние активации, переменные окружения и т.д. навыков:

json5
{
  "skills": {
    "entries": {
      "nano-banana-pro": {
        "enabled": true,
        "apiKey": "GEMINI_KEY_HERE",
        "env": {
          "GEMINI_API_KEY": "GEMINI_KEY_HERE"
        },
        "config": {
          "endpoint": "https://example.invalid",
          "model": "nano-pro"
        }
      },
      "peekaboo": { "enabled": true },
      "sag": { "enabled": false }
    }
  }
}

Правила:

  • enabled: false: отключает навык, даже если он Bundled или установлен
  • env: внедряет переменные окружения (только когда переменная не установлена в процессе)
  • apiKey: удобное поле для навыков, которые объявляют metadata.clawdbot.primaryEnv
  • config: пакет необязательных пользовательских полей, пользовательские ключи должны быть размещены здесь

Вы должны увидеть: после сохранения конфигурации Clawdbot применит эти настройки при следующем выполнении агента.

Шаг 5: Включение монитора навыков (необязательно) ​

По умолчанию Clawdbot отслеживает папку навыков и обновляет снимок навыков при изменении файла SKILL.md. Вы можете настроить это в skills.load:

json5
{
  "skills": {
    "load": {
      "watch": true,
      "watchDebounceMs": 250
    }
  }
}

Вы должны увидеть: после изменения файла навыка без необходимости перезапуска Gateway Clawdbot автоматически обновит список навыков в следующем ходе агента.

Шаг 6: Отладка проблем с навыками ​

Просмотр подробной информации о навыке и отсутствующих зависимостях:

bash
clawdbot skills info <name>

Проверка статуса зависимостей всех навыков:

bash
clawdbot skills check

Вы должны увидеть: подробную информацию о навыке, включая бинарные файлы, переменные окружения, статус конфигурации и отсутствующие условия.

Контрольная точка ✅ ​

После выполнения вышеуказанных шагов вы должны быть способны:

  • [ ] Использовать clawdbot skills list для просмотра всех доступных навыков
  • [ ] Устанавливать новые навыки через ClawdHub
  • [ ] Обновлять установленные навыки
  • [ ] Настраивать перезапись навыков в clawdbot.json
  • [ ] Использовать skills check для отладки проблем с зависимостями навыков

Распространённые предупреждения ​

Распространённая ошибка 1: название навыка содержит дефисы ​

Проблема: использование названия навыка с дефисами в качестве ключа в skills.entries

json
// ❌ Ошибка: без кавычек
{
  "skills": {
    "entries": {
      nano-banana-pro: { "enabled": true }  // Ошибка синтаксиса JSON
    }
  }
}

Исправление: заключить ключ в кавычки (JSON5 поддерживает ключи с кавычками)

json
// ✅ Правильно: с кавычками
{
  "skills": {
    "entries": {
      "nano-banana-pro": { "enabled": true }
    }
  }
}

Распространённая ошибка 2: переменные окружения в среде sandbox ​

Проблема: навык выполняется в sandbox, но skills.entries.<skill>.env или apiKey не имеют эффекта

Причина: глобальный env и skills.entries.<skill>.env/apiKey применяются только к выполнению на хосте, sandbox не наследует process.env хоста.

Исправление: использовать один из следующих методов:

json5
{
  "agents": {
    "defaults": {
      "sandbox": {
        "docker": {
          "env": {
            "GEMINI_API_KEY": "your_key_here"
          }
        }
      }
    }
  }
}

Или запечь переменные окружения в пользовательский образ sandbox.

Распространённая ошибка 3: навык не отображается в списке ​

Проблема: навык установлен, но clawdbot skills list не отображает его

Возможные причины:

  1. Навык не соответствует условиям gating (отсутствуют бинарные файлы, переменные окружения, конфигурация)
  2. Навык отключён (enabled: false)
  3. Навык не находится в каталогах, сканируемых Clawdbot

Этапы устранения неполадок:

bash
# Проверить зависимости навыка
clawdbot skills check

# Проверить, сканируется ли каталог навыков
ls -la ~/.clawdbot/skills/
ls -la <workspace>/skills/

Распространённая ошибка 4: конфликты навыков и путаница с приоритетами ​

Проблема: существует навык с одинаковым именем в нескольких местах, какой загружается?

Помните приоритет:

<workspace>/skills (наивысший) → ~/.clawdbot/skills → bundled skills (наименьший)

Если вы хотите использовать навыки Bundled вместо перезаписи Workspace:

  1. Удалить или переименовать <workspace>/skills/<skill-name>
  2. Или отключить этот навык в skills.entries

Итоги урока ​

В этом уроке вы изучили основные концепции платформы навыков Clawdbot:

  • Три типа навыков: Bundled, Managed, Workspace, загружаемые по приоритету
  • Интеграция ClawdHub: публичный реестр для просмотра, установки, обновления и публикации навыков
  • Gating навыков: фильтрация навыков по полю requires метаданных
  • Конфигурация перезаписи: управление активацией, переменными окружения и пользовательской конфигурацией в clawdbot.json
  • Монитор навыков: автоматически обновляет список навыков без необходимости перезапуска Gateway

Система навыков - это центральный механизм расширения возможностей Clawdbot. Владение им позволяет вашему ИИ-ассистенту адаптироваться к большему количеству сценариев и инструментов.
