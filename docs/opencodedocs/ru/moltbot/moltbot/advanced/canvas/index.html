Визуальный интерфейс Canvas и A2UI ​

Что вы сможете делать после этого урока ​

После завершения этого урока вы сможете:

  • Настроить Canvas Host и развернуть пользовательские интерфейсы HTML/CSS/JS
  • Использовать инструмент canvas для управления Canvas на узлах (показать, скрыть, навигация, выполнить JS)
  • Освоить протокол A2UI для того, чтобы ИИ отправлял динамические обновления UI
  • Делать скриншоты Canvas для контекста ИИ
  • Понять механизмы безопасности и контроля доступа Canvas

Ваша текущая ситуация ​

У вас есть ассистент ИИ, но он может взаимодействовать с вами только через текст. Вы хотите:

  • Чтобы ИИ отображал визуальные интерфейсы, такие как таблицы, графики, формы
  • Видеть динамические интерфейсы, генерируемые Агентами на мобильных устройствах
  • Создавать интерактивный опыт типа "приложение" без отдельной разработки

Когда использовать этот метод ​

Canvas + A2UI подходит для этих сценариев:

СценарийПример
Визуализация данныхОтображение статистических диаграмм, индикаторов прогресса, временных шкал
Интерактивные формыПопросить пользователя подтвердить действия, выбрать параметры
Панели состоянияОтображение прогресса задач в реальном времени, состояния системы
Игры и развлеченияПростые мини-игры, интерактивные демонстрации

A2UI vs. статический HTML

  • A2UI(Agent-to-UI): ИИ динамически генерирует и обновляет UI, подходит для данных в реальном времени
  • Статический HTML: Предопределённые интерфейсы, подходит для фиксированных макетов и сложных взаимодействий

🎒 Подготовка ​

Прежде чем начать, убедитесь, что вы выполнили:

  • [ ] Gateway запущен: Canvas Host автоматически запускается с Gateway по умолчанию (порт 18793)
  • [ ] Узлы сопряжены: Узлы macOS/iOS/Android подключены к Gateway
  • [ ] Узлы поддерживают Canvas: Подтвердите, что узел имеет возможность canvas (clawdbot nodes list)

Предварительные знания

Этот учебник предполагает, что вы уже знаете:

  • Базовое сопряжение узлов
  • Механизм вызова инструментов ИИ

Основные концепции ​

Система Canvas включает три основных компонента:

┌─────────────────┐
│   Canvas Host  │ ────▶ HTTP-сервер (порт 18793)
│   (Gateway)   │        └── Служит файлы ~/clawd/canvas/
└─────────────────┘
        │
        │ Коммуникация WebSocket
        │
┌─────────────────┐
│    Node App   │ ────▶ WKWebView рендерит Canvas
│ (iOS/Android) │        └── Получает отправки через A2UI
└─────────────────┘
        │
        │ События userAction
        │
┌─────────────────┐
│   AI Agent    │ ────▶ Вызовы инструмента canvas
│  (pi-mono)   │        └── Отправляет обновления A2UI
└─────────────────┘

Ключевые концепции:

  1. Canvas Host(сторона Gateway)

    • Предоставляет службу статических файлов: http://<gateway-host>:18793/__clawdbot__/canvas/
    • Размещает хост A2UI: http://<gateway-host>:18793/__clawdbot__/a2ui/
    • Поддерживает горячую перезагрузку: автоматическое обновление после изменения файлов

  2. Canvas Panel(сторона узла)

    • Узлы macOS/iOS/Android встраивают WKWebView
    • Подключаются к Gateway через WebSocket (горячая перезагрузка в реальном времени, коммуникация A2UI)
    • Поддерживают eval для выполнения JS, snapshot для захвата экрана

  3. Протокол A2UI(v0.8)

    • Агент отправляет обновления UI через WebSocket
    • Поддерживает: beginRendering, surfaceUpdate, dataModelUpdate, deleteSurface

Следуйте шагам ​

Шаг 1: Проверить статус Canvas Host ​

Зачем Обеспечить работу Canvas Host, чтобы узлы могли загружать содержимое Canvas.

bash
# Проверить, слушается ли порт 18793
lsof -i :18793

Вы должны увидеть:

text
COMMAND   PID   USER   FD   TYPE   DEVICE SIZE/OFF NODE NAME
node     12345  user   16u  IPv6  0x1234      0t0  TCP *:18793 (LISTEN)

Пути конфигурации

  • Корневой каталог Canvas: ~/clawd/canvas/(изменяемый через canvasHost.root)
  • Порт: 18793 = gateway.port + 4(изменяемый через canvasHost.port)
  • Горячая перезагрузка: Включена по умолчанию(отключаемая через canvasHost.liveReload: false)

Шаг 2: Создать первую страницу Canvas ​

Зачем Создать пользовательский интерфейс HTML для отображения вашего содержимого на узле.

bash
# Создать корневой каталог Canvas (если не существует)
mkdir -p ~/clawd/canvas

# Создать простой файл HTML
cat > ~/clawd/canvas/hello.html <<'EOF'
<!doctype html>
<meta charset="utf-8" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
<title>Hello Canvas</title>
<style>
  body {
    font-family: -apple-system, BlinkMacSystemFont, system-ui, sans-serif;
    padding: 20px;
    background: #000;
    color: #fff;
    text-align: center;
  }
  h1 { color: #24e08a; }
</style>
<h1>🎉 Hello from Canvas!</h1>
<p>Это ваша первая страница Canvas.</p>
<button onclick="alert('Кнопка нажата!')">Нажмите здесь</button>
EOF

Вы должны увидеть:

text
Файл создан: ~/clawd/canvas/hello.html

Шаг 3: Отобразить Canvas на узле ​

Зачем Заставить узел загрузить и отобразить страницу, которую вы только что создали.

Сначала найдите ID вашего узла:

bash
clawdbot nodes list

Вы должны увидеть:

text
ID                                  Name          Type       Capabilities
──────────────────────────────────────────────────────────────────────────
abc123-def456-ghi789               iOS Phone     canvas, camera, screen
jkl012-mno345-pqr678               Android Tab   canvas, camera

Затем отобразите Canvas (используя узел iOS в качестве примера):

bash
# Метод 1: Через команду CLI
clawdbot nodes canvas present --node abc123-def456-ghi789 --target http://127.0.0.1:18793/__clawdbot__/canvas/hello.html

Вы должны увидеть:

  • Панель без границ появляется на устройстве iOS, отображая ваше содержимое HTML
  • Панель появляется рядом с меню или позицией мыши
  • Содержимое центрировано с зелёным заголовком и кнопкой

Пример вызова ИИ:

ИИ: Я открыл панель Canvas на вашем устройстве iOS, отображая страницу приветствия.

Формат URL Canvas

  • Локальный файл: http://<gateway-host>:18793/__clawdbot__/canvas/hello.html
  • Внешний URL: https://example.com(требует сетевых разрешений узла)
  • Вернуться к умолчанию: / или пустая строка, отображает встроенную страницу скаффолдинга

Шаг 4: Использовать A2UI для отправки динамического UI ​

Зачем ИИ может отправлять обновления UI непосредственно на Canvas без изменения файлов, подходит для данных в реальном времени и взаимодействия.

Метод A: Быстрая отправка текста

bash
clawdbot nodes canvas a2ui push --node abc123-def456-ghi789 --text "Hello from A2UI"

Вы должны увидеть:

  • Canvas отображает синий интерфейс A2UI
  • Центрированный текст показывает: Hello from A2UI

Метод B: Полная отправка JSONL

Создайте файл определения A2UI:

bash
cat > /tmp/a2ui-demo.jsonl <<'EOF'
{"surfaceUpdate":{"surfaceId":"main","components":[{"id":"root","component":{"Column":{"children":{"explicitList":["title","status","button"]}}}},{"id":"title","component":{"Text":{"text":{"literalString":"Демо A2UI"},"usageHint":"h1"}}},{"id":"status","component":{"Text":{"text":{"literalString":"Состояние системы: Выполняется"},"usageHint":"body"}}},{"id":"button","component":{"Button":{"label":{"literalString":"Тестовая кнопка"},"onClick":{"action":{"name":"testAction","sourceComponentId":"demo.test"}}}}}]}
{"beginRendering":{"surfaceId":"main","root":"root"}}
EOF

Отправьте A2UI:

bash
clawdbot nodes canvas a2ui push --node abc123-def456-ghi789 --jsonl /tmp/a2ui-demo.jsonl

Вы должны увидеть:

┌────────────────────────────┐
│     Демо A2UI         │
│                        │
│  Состояние системы: Выполняется  │
│                        │
│   [ Тестовая кнопка ]          │
└────────────────────────────┘
Объяснение формата JSONL A2UI

JSONL (JSON Lines) содержит один объект JSON на строку, подходит для обновлений в потоковом режиме:

jsonl
{"surfaceUpdate":{...}}   // Обновить компоненты поверхности
{"beginRendering":{...}}   // Начать рендеринг
{"dataModelUpdate":{...}} // Обновить модель данных
{"deleteSurface":{...}}   // Удалить поверхность

Шаг 5: Выполнить JavaScript Canvas ​

Зачем Выполнять пользовательский JS в Canvas, например изменить DOM, прочитать состояние.

bash
clawdbot nodes canvas eval --node abc123-def456-ghi789 --js "document.title"

Вы должны увидеть:

text
"Hello from Canvas"

Примеры выполнения JS

  • Прочитать элемент: document.querySelector('h1').textContent
  • Изменить стиль: document.body.style.background = '#333'
  • Вычислить значение: innerWidth + 'x' + innerHeight
  • Выполнить замыкание: (() => { ... })()

Шаг 6: Захватить скриншот Canvas ​

Зачем Позволить ИИ увидеть текущее состояние Canvas для понимания контекста.

bash
# Формат по умолчанию (JPEG)
clawdbot nodes canvas snapshot --node abc123-def456-ghi789

# Формат PNG + ограничение максимальной ширины
clawdbot nodes canvas snapshot --node abc123-def456-ghi789 --format png --max-width 1200

# JPEG высокого качества
clawdbot nodes canvas snapshot --node abc123-def456-ghi789 --format jpg --quality 0.9

Вы должны увидеть:

text
Canvas snapshot saved to: /var/folders/.../canvas-snapshot.jpg

Путь к файлу генерируется автоматически системой, обычно во временном каталоге.

Шаг 7: Скрыть Canvas ​

Зачем Закрыть панель Canvas для освобождения места на экране.

bash
clawdbot nodes canvas hide --node abc123-def456-ghi789

Вы должны увидеть:

  • Панель Canvas на устройстве iOS исчезает
  • Состояние узла восстанавливается (если ранее было занято)

Предупреждения ​

Ограничения передний план/фон

  • Узлы iOS/Android: Команды canvas.* и camera.* должны выполняться на переднем плане
  • Вызовы в фоновом режиме вернут: NODE_BACKGROUND_UNAVAILABLE
  • Решение: Разбудить устройство на передний план

Меры безопасности

  • Защита от обхода каталогов: URL Canvas запрещают .. для доступа к верхним каталогам
  • Пользовательская схема: clawdbot-canvas:// ограничена только внутренним использованием узла
  • Ограничения HTTPS: Внешние URL HTTPS требуют сетевых разрешений узла
  • Доступ к файлам: Canvas Host позволяет только доступ к файлам под canvasHost.root

Советы по отладке

  • Просмотреть журналы Gateway: clawdbot gateway logs
  • Просмотреть журналы узла: iOS Настройки → Debug Logs, Журналы внутри приложения Android
  • Проверить URL: Доступитесь напрямую в браузере к http://<gateway-host>:18793/__clawdbot__/canvas/

Ключевые константы:

  • A2UI_PATH = "/__clawdbot__/a2ui": Путь хоста A2UI
  • CANVAS_HOST_PATH = "/__clawdbot__/canvas": Путь файлов Canvas
  • CANVAS_WS_PATH = "/__clawdbot__/ws": Путь горячей перезагрузки WebSocket

Ключевые функции:

  • createCanvasHost(): Запустить HTTP-сервер Canvas (порт 18793)
  • injectCanvasLiveReload(): Вставить скрипт горячей перезагрузки WebSocket в HTML
  • handleA2uiHttpRequest(): Обрабатывать запросы ресурсов A2UI
  • createCanvasTool(): Зарегистрировать инструмент canvas (present/hide/navigate/eval/snapshot/a2ui_push/a2ui_reset)

Поддерживаемые действия Canvas:

  • present: Отобразить Canvas (URL, позиция, размер опционально)
  • hide: Скрыть Canvas
  • navigate: Перейти к URL (локальный путь/HTTP/file://)
  • eval: Выполнить JavaScript
  • snapshot: Захватить скриншот (PNG/JPEG, maxWidth/quality опционально)
  • a2ui_push: Отправить обновления A2UI (JSONL или текст)
  • a2ui_reset: Сбросить состояние A2UI

Схема конфигурации:

  • canvasHost.root: Корневой каталог Canvas (по умолчанию ~/clawd/canvas)
  • canvasHost.port: HTTP-порт (по умолчанию 18793)
  • canvasHost.liveReload: Включить горячую перезагрузку (по умолчанию true)
  • canvasHost.enabled: Включить Canvas Host (по умолчанию true)

Сообщения, поддерживаемые A2UI v0.8:

  • beginRendering: Начать рендеринг определённой поверхности
  • surfaceUpdate: Обновить компоненты поверхности (Column, Text, Button и т.д.)
  • dataModelUpdate: Обновить модель данных
  • deleteSurface: Удалить определённую поверхность
