Полное руководство по настройке канала WhatsApp ​

Чему вы научитесь ​

  • Подключить аккаунт WhatsApp к Clawdbot через QR-код
  • Настроить поддержку нескольких аккаунтов WhatsApp
  • Установить контроль доступа для DM (пара/белый список/открытый)
  • Включить и управлять поддержкой групп WhatsApp
  • Настроить автоматическое подтверждение сообщений и отчёты о прочтении

Ваша текущая проблема ​

WhatsApp - это ваша самая используемая мессенджер-платформа, но ваш AI-ассистент ещё не может получать сообщения WhatsApp. Вы хотите:

  • Общаться с AI напрямую в WhatsApp, не переключая приложения
  • Контролировать, кто может отправлять сообщения вашему AI
  • Поддерживать несколько аккаунтов WhatsApp (разделение работы и личного)

Когда использовать этот метод ​

  • Вам нужно подключить AI-ассистента к WhatsApp
  • Вам нужно разделить рабочие и личные аккаунты WhatsApp
  • Вы хотите точно контролировать, кто может отправлять сообщения AI

Что такое Baileys?

Baileys - это библиотека WhatsApp Web, позволяющая программам отправлять и получать сообщения через протокол WhatsApp Web. Clawdbot использует Baileys для подключения к WhatsApp без использования WhatsApp Business API, что более приватно и гибко.

🎒 Подготовка перед началом ​

Перед настройкой канала WhatsApp убедитесь:

  • [ ] Установлен и запущен Clawdbot Gateway
  • [ ] Завершён Быстрый старт
  • [ ] Есть доступный номер телефона (рекомендуется использовать запасной номер)
  • [ ] Телефон WhatsApp имеет доступ к интернету (для сканирования QR-кода)

Важные замечания

  • Рекомендуется использовать отдельный номер: отдельная SIM-карта или старый телефон, чтобы избежать помех для личного использования
  • Избегайте виртуальных номеров: TextNow, Google Voice и другие виртуальные номера будут заблокированы WhatsApp
  • Среда выполнения Node: WhatsApp и Telegram нестабильны на Bun, используйте Node ≥22

Основная идея ​

Основная архитектура канала WhatsApp:

Ваш телефон WhatsApp ←--(QR-код)--> Baileys ←--→ Clawdbot Gateway
                                                       ↓
                                                  AI Agent
                                                       ↓
                                                  Ответ на сообщение

Ключевые концепции:

  1. Сессия Baileys: соединение через WhatsApp Linked Devices
  2. Политика DM: контроль того, кто может отправлять личные сообщения AI
  3. Поддержка нескольких аккаунтов: один Gateway управляет несколькими аккаунтами WhatsApp
  4. Подтверждение сообщений: автоматическая отправка эмодзи/отчётов о прочтении для улучшения пользовательского опыта

Следуйте за мной ​

Шаг 1: Настройка базовых настроек ​

Зачем Установить политику контроля доступа WhatsApp, чтобы защитить вашего AI-ассистента от злоупотреблений.

Отредактируйте ~/.clawdbot/clawdbot.json, добавьте конфигурацию WhatsApp:

json
{
  "channels": {
    "whatsapp": {
      "dmPolicy": "pairing",
      "allowFrom": ["+15551234567"]
    }
  }
}

Описание полей:

ПолеТипПо умолчаниюОписание
dmPolicystring"pairing"Политика доступа DM: pairing (пара), allowlist (белый список), open (открытый), disabled (отключено)
allowFromstring[][]Список разрешённых номеров телефонов (формат E.164, например +15551234567)

Сравнение политик DM:

ПолитикаПоведениеПрименимые сценарии
pairingНеизвестные отправители получают код пары, требующий ручного одобренияРекомендуется, баланс между безопасностью и удобством
allowlistРазрешены только номера из списка allowFromСтрогий контроль, известные пользователи
openЛюбой может отправлять (требуется allowFrom содержать "*")Публичное тестирование или служба сообщества
disabledИгнорировать все сообщения WhatsAppВременно отключить канал

Что вы должны увидеть: файл конфигурации успешно сохранён без ошибок формата JSON.

Шаг 2: Вход в WhatsApp ​

Зачем Подключить аккаунт WhatsApp к Clawdbot через QR-код, Baileys будет поддерживать состояние сессии.

Выполните в терминале:

bash
clawdbot channels login whatsapp

Вход с несколькими аккаунтами:

Войти в конкретный аккаунт:

bash
clawdbot channels login whatsapp --account work

Войти в аккаунт по умолчанию:

bash
clawdbot channels login whatsapp

Порядок действий:

  1. Терминал покажет QR-код (или отобразится в интерфейсе CLI)
  2. Откройте приложение WhatsApp на телефоне
  3. Перейдите в Настройки → Связанные устройства
  4. Нажмите Привязать устройство
  5. Отсканируйте QR-код, отображаемый в терминале

Что вы должны увидеть:

✓ WhatsApp linked successfully!
Credentials stored: ~/.clawdbot/credentials/whatsapp/default/creds.json

Хранение учётных данных

Учётные данные входа WhatsApp сохраняются в ~/.clawdbot/credentials/whatsapp/<accountId>/creds.json. После первого входа сессия будет автоматически восстанавливаться при последующих запусках, нет необходимости повторно сканировать QR-код.

Шаг 3: Запуск Gateway ​

Зачем Запуск Gateway позволяет каналу WhatsApp начать приём и отправку сообщений.

bash
clawdbot gateway

Или используйте режим демона:

bash
clawdbot gateway start

Что вы должны увидеть:

[WhatsApp] Connected to WhatsApp Web
[WhatsApp] Default account linked: +15551234567
Gateway listening on ws://127.0.0.1:18789

Привет

Шаг 5: Настройка расширенных опций (необязательно) ​

Включить автоматическое подтверждение сообщений ​

Добавьте в clawdbot.json:

json
{
  "channels": {
    "whatsapp": {
      "ackReaction": {
        "emoji": "👀",
        "direct": true,
        "group": "mentions"
      }
    }
  }
}

Описание полей:

ПолеТипПо умолчаниюОписание
emojistring-Эмодзи подтверждения (например "👀", "✅"), пустая строка означает отключение
directbooleantrueОтправлять подтверждение в личных сообщениях
groupstring"mentions"Поведение в группах: "always" (все сообщения), "mentions" (только упоминания), "never" (никогда)

Настройка отчётов о прочтении ​

По умолчанию Clawdbot автоматически отмечает сообщения как прочитанные (синие галочки). Чтобы отключить:

json
{
  "channels": {
    "whatsapp": {
      "sendReadReceipts": false
    }
  }
}

Настройка ограничений сообщений ​

json
{
  "channels": {
    "whatsapp": {
      "textChunkLimit": 4000,
      "mediaMaxMb": 50,
      "chunkMode": "length"
    }
  }
}
ПолеПо умолчаниюОписание
textChunkLimit4000Максимальное количество символов в одном текстовом сообщении
mediaMaxMb50Максимальный размер медиафайлов (МБ)
chunkMode"length"Режим разбиения: "length" (по длине), "newline" (по абзацам)

Что вы должны увидеть: после вступления конфигурации в силу длинные сообщения автоматически разбиваются, размер медиафайлов контролируется.

Предупреждения о проблемах ​

Проблема 1: Ошибка сканирования QR-кода ​

Симптом: после сканирования QR-кода терминал показывает ошибку подключения или тайм-аут.

Причина: проблемы с сетевым подключением или нестабильность службы WhatsApp.

Решение:

  1. Проверьте сетевое подключение телефона
  2. Убедитесь, что сервер Gateway имеет доступ к интернету
  3. Выйдите и войдите снова:
    bash
    clawdbot channels logout whatsapp
clawdbot channels login whatsapp

Проблема 2: Сообщения не доставляются или задерживаются ​

Симптом: после отправки сообщения ответ приходит очень долго.

Причина: Gateway не запущен или соединение WhatsApp разорвано.

Решение:

  1. Проверьте статус Gateway: clawdbot gateway status
  2. Перезапустите Gateway: clawdbot gateway restart
  3. Просмотрите журналы: clawdbot logs --follow

Проблема 3: Код пары не получен ​

Симптом: после того как незнакомец отправил сообщение, код пары не получен.

Причина: dmPolicy не установлен в pairing.

Решение:

Проверьте настройку dmPolicy в clawdbot.json:

json
{
  "channels": {
    "whatsapp": {
      "dmPolicy": "pairing"  // ← Убедитесь, что "pairing"
    }
  }
}

Проблема 4: Проблемы с средой выполнения Bun ​

Симптом: WhatsApp и Telegram часто разрывают соединение или не удается войти.

Причина: Baileys и Telegram SDK нестабильны на Bun.

Решение:

Используйте Node ≥22 для запуска Gateway:

Проверьте текущую среду выполнения:

bash
node --version

Если нужно переключиться, используйте Node для запуска Gateway:

bash
clawdbot gateway --runtime node

Рекомендуемая среда выполнения

Для каналов WhatsApp и настоятельно рекомендуется использовать среду выполнения Node, Bun может привести к нестабильному соединению.

Ключевые параметры конфигурации:

  • dmPolicy: политика доступа DM (pairing/allowlist/open/disabled)
  • allowFrom: список разрешённых отправителей (номера телефонов в формате E.164)
  • ackReaction: конфигурация автоматического подтверждения сообщений ({emoji, direct, group})
  • sendReadReceipts: отправлять ли отчёты о прочтении (по умолчанию true)
  • textChunkLimit: ограничение разбиения текста (по умолчанию 4000 символов)
  • mediaMaxMb: ограничение размера медиафайлов (по умолчанию 50 МБ)

Ключевые функции:

  • loginWeb(): выполняет вход в WhatsApp через QR-код
  • startWebLoginWithQr(): запускает процесс генерации QR-кода
  • sendReactionWhatsApp(): отправляет реакцию эмодзи WhatsApp
  • handleWhatsAppAction(): обрабатывает специфичные действия WhatsApp (например, реакции)

Ключевые константы:

  • DEFAULT_ACCOUNT_ID: ID аккаунта по умолчанию ("default")
  • creds.json: путь хранения учётных данных аутентификации WhatsApp
