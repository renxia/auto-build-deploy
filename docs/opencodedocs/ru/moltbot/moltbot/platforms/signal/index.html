Почему стоит выбрать Signal?

Signal — это приложение для мгновенного обмена сообщениями с end-to-end шифрованием, все сообщения зашифрованы и могут быть прочитаны только отправителем и получателем. Clawdbot интегрируется через signal-cli, позволяя вам пользоваться функциями AI-ассистента, сохраняя конфиденциальность.

Когда использовать этот метод ​

Сценарии, подходящие для использования канала Signal:

  • Вам нужен канал связи с приоритетом конфиденциальности
  • Ваша команда или группа друзей использует Signal
  • Вам нужно запустить AI-ассистент на личном устройстве (локально приоритет)
  • Вам нужно контролировать доступ через защищенный механизм спаривания DM

Отдельный аккаунт Signal

Рекомендуется использовать отдельный номер Signal в качестве аккаунта бота, а не ваш личный номер. Это позволит избежать циклических сообщений (бот игнорирует свои собственные сообщения) и сохранить разделение между рабочими и личными коммуникациями.

Основная концепция ​

Интеграция Signal в Clawdbot основана на signal-cli и работает следующим образом:

  1. Режим демона: signal-cli работает как фоновый демон, предоставляя интерфейс HTTP JSON-RPC
  2. Поток событий (SSE): Clawdbot получает события Signal через Server-Sent Events (SSE)
  3. Стандартизация сообщений: Сообщения Signal конвертируются в унифицированный внутренний формат, затем маршрутизируются к AI Agent
  4. Детерминированная маршрутизация: Все ответы отправляются обратно отправителю или группе исходного сообщения

Ключевые принципы дизайна:

  • Локальный приоритет: signal-cli работает на вашем устройстве, все коммуникации зашифрованы
  • Поддержка нескольких аккаунтов: Можно настроить несколько аккаунтов Signal
  • Контроль доступа: По умолчанию включен механизм спаривания DM, незнакомцы должны получить одобрение для отправки сообщений
  • Изоляция контекста: Групповые сообщения имеют независимый контекст сессии, не смешиваются с личными сообщениями

Следуйте моим инструкциям ​

Шаг 1: Установка signal-cli ​

Зачем signal-cli — это интерфейс командной строки для Signal, через него Clawdbot взаимодействует с сетью Signal.

Методы установки

bash
brew install signal-cli
bash
# Посетите https://github.com/AsamK/signal-cli/releases для просмотра последней версии
# Скачайте последний пакет signal-cli (замените VERSION на фактический номер версии)
wget https://github.com/AsamK/signal-cli/releases/download/vVERSION/signal-cli-VERSION.tar.gz

# Распакуйте в каталог /opt
sudo tar -xvf signal-cli-VERSION.tar.gz -C /opt/

# Создайте символическую ссылку (необязательно)
sudo ln -s /opt/signal-cli-VERSION/bin/signal-cli /usr/local/bin/signal-cli
bash
# В WSL2 используйте метод установки для Linux
# Примечание: Windows использует WSL2, установка signal-cli следует процессу Linux

Проверка установки

bash
signal-cli --version
# Должен показать: номер версии signal-cli (например, 0.13.x или выше)

Вы должны увидеть: вывод номера версии signal-cli.

Требование к Java

signal-cli требует выполнения Java (обычно Java 11 или выше). Убедитесь, что Java установлена и настроена:

bash
java -version
# Должен показать: openjdk version "11.x.x" или выше

Примечание: Для конкретных требований к версии Java обратитесь к официальной документации signal-cli.

Шаг 3: Настройка канала Signal в Clawdbot ​

Зачем Файл конфигурации сообщает Clawdbot, как подключиться к signal-cli и как обрабатывать сообщения из Signal.

Методы настройки

Есть три способа настройки, выберите подходящий для вас:

Способ 1: Быстрая настройка (один аккаунт) ​

Это самый простой способ, подходит для сценария с одним аккаунтом.

Отредактируйте ~/.clawdbot/clawdbot.json:

json
{
  "channels": {
    "signal": {
      "enabled": true,
      "account": "+15551234567",
      "cliPath": "signal-cli",
      "dmPolicy": "pairing",
      "allowFrom": ["+15557654321"]
    }
  }
}

Описание полей:

ПолеЗначениеОписание
enabledtrueВключить канал Signal
account"+15551234567"Ваш аккаунт Signal (формат E.164)
cliPath"signal-cli"Путь к команде signal-cli
dmPolicy"pairing"Политика доступа DM (режим спаривания по умолчанию)
allowFrom["+15557654321"]Белый список номеров, которым разрешено отправлять DM

Способ 2: Настройка нескольких аккаунтов ​

Если вам нужно управлять несколькими аккаунтами Signal, используйте объект accounts:

json
{
  "channels": {
    "signal": {
      "enabled": true,
      "accounts": {
        "work": {
          "account": "+15551234567",
          "name": "Work Bot",
          "httpPort": 8080,
          "dmPolicy": "pairing",
          "allowFrom": ["+15557654321"]
        },
        "personal": {
          "account": "+15559876543",
          "name": "Personal Bot",
          "httpPort": 8081,
          "dmPolicy": "allowlist",
          "allowFrom": ["*"]
        }
      }
    }
  }
}

Описание настройки:

  • Каждый аккаунт имеет уникальный ID (например, work, personal)
  • Каждый аккаунт может иметь разные порты, политики и разрешения
  • name — необязательное отображаемое имя, используется в списках CLI/UI

Способ 3: Режим внешнего демона ​

Если вы хотите самостоятельно управлять signal-cli (например, в контейнере или при совместном использовании CPU), отключите автоматический запуск:

json
{
  "channels": {
    "signal": {
      "enabled": true,
      "httpUrl": "http://127.0.0.1:8080",
      "autoStart": false
    }
  }
}

Описание настройки:

  • httpUrl: полный URL демона (переопределяет httpHost и httpPort)
  • autoStart: установите в false, чтобы отключить автоматический запуск signal-cli
  • Clawdbot подключится к уже запущенному демону signal-cli

Вы должны увидеть: после сохранения файла конфигурации нет синтаксических ошибок.

Проверка конфигурации

Clawdbot проверяет конфигурацию при запуске. Если конфигурация неверна, в журналах отобразится подробное сообщение об ошибке.

Шаг 5: Отправка первого сообщения ​

Зачем Проверить правильность конфигурации, убедиться, что Clawdbot может принимать и обрабатывать сообщения Signal.

Отправка сообщения

    После успешного спаривания отправьте сообщение снова:

Контрольная точка ✅:

Свои сообщения будут игнорироваться

Если вы запускаете бота на личном номере Signal, бот будет игнорировать сообщения, отправленные вами самим (защита от циклов). Рекомендуется использовать отдельный номер для бота.

Предупреждения о подводных камнях ​

Проблема 1: Неудачный запуск signal-cli ​

Проблема: Демон signal-cli не запускается

Возможные причины:

  1. Java не установлена или версия слишком старая
  2. Порт уже занят
  3. Неверный путь к signal-cli

Решения:

bash
# Проверка версии Java
java -version

# Проверка занятости порта
lsof -i :8080  # macOS/Linux
netstat -ano | findstr :8080  # Windows (PowerShell)

# Проверка пути signal-cli
which signal-cli

Проблема 2: Код спаривания истек ​

Проблема: Код спаривания истекает через 1 час

Решение:

Отправьте сообщение снова, получите новый код спаривания и подтвердите его в течение 1 часа.

Проблема 3: Нет ответа на групповые сообщения ​

Проблема: @упоминание бота в группе Signal, но нет ответа

Возможные причины:

  • groupPolicy установлена в allowlist, но вы не находитесь в groupAllowFrom
  • Signal не поддерживает встроенные @упоминания (в отличие от Discord/Slack)

Решения:

Настройте групповую политику:

json
{
  "channels": {
    "signal": {
      "groupPolicy": "allowlist",
      "groupAllowFrom": ["+15557654321"]
    }
  }
}

Или используйте командный триггер (если настроено commands.config: true):

@clawdbot help

Проблема 4: Неудачная загрузка медиа-файлов ​

Проблема: Изображения или вложения в сообщениях Signal не могут быть обработаны

Возможные причины:

  • Размер файла превышает ограничение mediaMaxMb (по умолчанию 8MB)
  • ignoreAttachments установлено в true

Решения:

json
{
  "channels": {
    "signal": {
      "mediaMaxMb": 20,
      "ignoreAttachments": false
    }
  }
}

Проблема 5: Длинные сообщения обрезаются ​

Проблема: Отправляемые сообщения разделяются на несколько частей

Причина: Signal имеет ограничение на длину сообщения (по умолчанию 4000 символов), Clawdbot автоматически разбивает сообщения

Решения:

Настройте разбиение на части:

json
{
  "channels": {
    "signal": {
      "textChunkLimit": 2000,
      "chunkMode": "newline"
    }
  }
}

Опции chunkMode:

  • length (по умолчанию): разбивка по длине
  • newline: сначала разбивка по пустым строкам (границы абзацев), затем по длине
