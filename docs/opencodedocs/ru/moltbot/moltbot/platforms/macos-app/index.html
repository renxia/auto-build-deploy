Приложение macOS: управление в строке меню и голосовое взаимодействие ​

Что вы сможете сделать ​

После завершения этого учебника вы сможете:

  • ✅ Понять основные возможности приложения Clawdbot для macOS как панели управления в строке меню
  • ✅ Освоить использование Voice Wake для голосового пробуждения и Talk Mode для непрерывного диалога
  • ✅ Понять возможности в режиме узла: system.run, Canvas, Camera и другие
  • ✅ Настроить локальный vs удаленный режим для различных сценариев развертывания
  • ✅ Управлять механизмом одобрения Exec Approvals для контроля прав выполнения команд
  • ✅ Использовать глубокие ссылки для быстрого вызова AI-помощника
  • ✅ Удаленно получать доступ и управлять шлюзом через SSH/Tailscale

Ваша текущая проблема ​

Вы, возможно, думаете:

  • "Что на самом деле делает приложение macOS? Это сам шлюз?"
  • "Как использовать Voice Wake и Talk Mode? Нужно ли дополнительное оборудование?"
  • "В чем разница между режимом узла и обычным режимом? Когда использовать какой?"
  • "Как управлять разрешениями и настройками безопасности на macOS?"
  • "Могу ли я запустить шлюз на другом компьютере?"

Хорошая новость: Приложение Clawdbot для macOS - это графическая панель управления шлюзом, оно не запускает службу шлюза, а подключается, управляет и мониторит. Кроме того, оно также работает как узел, предоставляя функции, специфичные для macOS (например, system.run, Canvas, Camera), удаленному шлюзу.

Когда использовать этот подход ​

Когда вам нужно:

  • 🖥️ Графическое управление macOS — Состояние и управление в строке меню, более интуитивное, чем командная строка
  • 🎙️ Голосовое взаимодействие — Voice Wake для пробуждения + Talk Mode для непрерывного диалога
  • 💻 Локальное выполнение команд — Запуск команд, таких как system.run, на узле macOS
  • 🎨 Визуализация Canvas — Рендеринг интерфейсов на основе AI на macOS
  • 📷 Функции устройства — Камера для фотографирования, видеозаписи и захвата экрана
  • 🌐 Удаленный доступ — Управление удаленным шлюзом через SSH/Tailscale

Разница между узлом и шлюзом

  • Шлюз (Gateway): Запускает AI-модели, управляет сессиями, обрабатывает сообщения (может работать на любой машине)
  • Узел (Node): Предоставляет локальные функции устройства (Canvas, Camera, system.run) шлюзу
  • Приложение macOS: Может быть клиентом шлюза, а также использоваться как узел

Основная концепция ​

Приложение Clawdbot для macOS - это система с двойной ролью:

│     Clawdbot.app (macOS приложение)    │
│                                      │
│   ┌────────────────────────────┐      │
│   │  Панель управления строкой меню  │      │
│   │  • Управление подключением шлюза  │◄────► Gateway WebSocket
│   │  • Встроенное окно WebChat  │      │
│   │  • Настройки и конфигурация    │      │
│   │  • Voice Wake/Talk Mode    │      │
│   └────────────────────────────┘      │
│                                      │
│   ┌────────────────────────────┐      │
│   │  Служба узла              │      │
│   │  • system.run              │◄────► Gateway узловый протокол
│   │  • Canvas                 │      │
│   │  • Camera/Screen          │      │
│   └────────────────────────────┘      │
└──────────────────────────────────────────┘

Два режима работы:

Удаленный режимУдаленная машина (через SSH/Tailscale)ЗапускаетсяШлюз работает на другой машине

Основные функциональные модули:

  1. Управление строкой меню — Состояние подключения шлюза, WebChat, конфигурация, управление сессиями
  2. Voice Wake — Глобальное прослушивание голосового слова-триггера
  3. Talk Mode — Цикл непрерывного голосового диалога (голосовой ввод → ответ AI → воспроизведение TTS)
  4. Режим узла — Предоставление специфичных для macOS команд (system.run, canvas.*, camera.*)
  5. Exec Approvals — Одобрение выполнения команд system.run и контроль безопасности
  6. Глубокие ссылки — Быстрый запуск функций через протокол clawdbot://

Делайте вместе с нами ​

Шаг 1: Установка и запуск приложения macOS ​

Зачем Вам нужно установить приложение Clawdbot для macOS, чтобы получить управление в строке меню и голосовые функции.

Способы установки:

bash
brew install --cask clawdbot
bash
# Скачайте последний Clawdbot.app.dmg с https://github.com/moltbot/moltbot/releases
# Перетащите в папку Applications

Первый запуск:

bash
open /Applications/Clawdbot.app

Что вы должны увидеть:

  • В верхней строке меню macOS появляется иконка 🦞
  • Нажмите на иконку, чтобы развернуть выпадающее меню
  • Система выводит диалоговое окно запроса разрешений TCC

Запросы разрешений при первом запуске

Приложению macOS требуются следующие разрешения (система автоматически предложит их):

  • Разрешение на уведомления — Для отображения системных уведомлений
  • Разрешение на вспомогательные функции — Для Voice Wake и системных операций
  • Разрешение на микрофон — Требуется для Voice Wake и Talk Mode
  • Разрешение на запись экрана — Для функций Canvas и захвата экрана
  • Разрешение на распознавание речи — Для голосового ввода Voice Wake
  • Разрешение на автоматизацию — Для управления AppleScript (если требуется)

Все эти разрешения используются исключительно локально, данные не отправляются на какие-либо серверы.

Шаг 2: Настройка режима подключения (локальный vs удаленный) ​

Зачем Выберите локальный или удаленный режим в зависимости от ваших потребностей развертывания.

Режим A: Локальный режим (по умолчанию) ​

Применимый сценарий: Шлюз и приложение macOS работают на одной машине.

Этапы настройки:

  1. Убедитесь, что в приложении в строке меню отображается режим Local
  2. Если шлюз не запущен, приложение автоматически запустит службу launchd com.clawdbot.gateway
  3. Приложение подключится к ws://127.0.0.1:18789

Что вы должны увидеть:

  • Иконка в строке меню отображается зеленым цветом (состояние подключено)
  • Карточка статуса шлюза отображает "Local"
  • Служба узла не запущена (режим узла требуется только в удаленном режиме)

Режим B: Удаленный режим ​

Применимый сценарий: Шлюз работает на другой машине (например, сервер или Linux VPS), и вы хотите управлять им через Mac.

Этапы настройки:

  1. В приложении в строке меню переключитесь на режим Remote
  2. Введите WebSocket-адрес удаленного шлюза (например, ws://your-server:18789)
  3. Выберите метод аутентификации (Token или Password)
  4. Приложение автоматически установит SSH-туннель для подключения к удаленному шлюзу

Что вы должны увидеть:

  • Иконка в строке меню отображает состояние подключения (желтый/зеленый/красный)
  • Карточка статуса шлюза отображает адрес удаленного сервера
  • Служба узла автоматически запускается (чтобы удаленный шлюз мог вызывать локальные функции)

Механизм туннеля в удаленном режиме:

Приложение macOS                  Удаленный шлюз
    │                                  │
    ├── SSH туннель ───────────────────► ws://remote:18789
    │                                  │
    └── Служба узла ◄───────────────────── node.invoke

Преимущества удаленного режима

  • Централизованное управление: Запуск шлюза на мощной машине, доступ нескольких клиентов
  • Оптимизация ресурсов: Mac может оставаться легким, шлюз работает на высокопроизводительном сервере
  • Локализация устройства: Функции Canvas, Camera и другие все еще выполняются локально на Mac

Шаг 3: Использование панели управления в строке меню ​

Зачем Приложение в строке меню обеспечивает быстрый доступ ко всем основным функциям.

Основные пункты меню:

После нажатия на иконку в строке меню вы увидите:

  1. Карточка статуса

    • Состояние подключения шлюза (подключен/отключен/подключение)
    • Текущий режим (Local/Remote)
    • Список запущенных каналов (WhatsApp, Telegram и т.д.)

  2. Быстрые действия

    • Agent — Открыть окно диалога с AI (вызов шлюза)
    • WebChat — Открыть встроенный интерфейс WebChat
    • Canvas — Открыть окно визуализации Canvas
    • Settings — Открыть интерфейс конфигурации

  3. Переключатели функций

    • Talk — Включить/отключить Talk Mode
    • Voice Wake — Включить/отключить Voice Wake

  4. Меню информации

    • Usage — Просмотр статистики использования и затрат
    • Sessions — Управление списком сессий
    • Channels — Просмотр статуса каналов
    • Skills — Управление пакетами навыков

Что вы должны видеть:

  • Индикаторы статуса в реальном времени (зеленый = нормально, красный = отключен)
  • При наведении мыши отображается подробная информация о подключении
  • Нажатие на любой пункт меню быстро открывает соответствующую функцию

Шаг 4: Настройка и использование Voice Wake ​

Зачем Voice Wake позволяет вам активировать AI-помощника голосовым словом-триггером, без необходимости нажимать или вводить.

Как работает Voice Wake:

┌──────────────────────────────────┐
│   Среда выполнения Voice Wake  │
│                              │
│   Прослушивание микрофона ──► Обнаружение слова-триггера │
│                              │
│   Совпадение слова-триггера?      │
│       │                       │
│       ├─ Да ──► Запуск Agent │
│       │                       │
│       └─ Нет ──► Продолжение прослушивания │
└──────────────────────────────────┘

Настройка Voice Wake:

  1. Откройте Settings → Voice Wake
  2. Введите слово-триггер (по умолчанию: clawd, claude, computer)
  3. Можно добавить несколько слов-триггеров (разделенных запятыми)
  4. Включите переключатель Enable Voice Wake

Правила слов-триггеров:

  • Слова-триггеры хранятся в шлюзе: ~/.clawdbot/settings/voicewake.json
  • Все узлы используют один и тот же глобальный список слов-триггеров
  • После изменения рассылается на все подключенные устройства (macOS, iOS, Android)

Процесс использования:

  1. Убедитесь, что разрешение на микрофон предоставлено
  2. Включите Voice Wake в строке меню
  3. Сказите слово-триггер в микрофон (например, "Hey clawd")
  4. Подождите, чтобы услышать звуковой сигнал "ding" (успешное пробуждение)
  5. Произнесите свою команду или вопрос

Что вы должны видеть:

  • По центру экрана появляется слой Voice Wake
  • Отображается форма волны громкости микрофона
  • Отображается текст статуса "Listening"
  • AI начинает обрабатывать ваш голосовой ввод

Глобальная природа Voice Wake

Слова-триггеры — это глобальная конфигурация на уровне шлюза, не ограниченная одним устройством. Это означает:

  • После изменения слов-триггеров на macOS устройства iOS и Android также синхронизируются
  • Все устройства используют один и тот же набор слов-триггеров
  • Но каждое устройство может отдельно включать/отключать Voice Wake (на основе разрешений и предпочтений пользователя)

Шаг 5: Использование Talk Mode для непрерывного диалога ​

Зачем Talk Mode обеспечивает опыт непрерывного голосового диалога, подобный Siri/Alexa, без необходимости каждый раз пробуждать.

Цикл работы Talk Mode:

Прослушивание ──► Обработка AI ──► Воспроизведение TTS ──► Прослушивание
  │                                              │
  └────────────────────────────────────────┘

Включение Talk Mode:

  1. Нажмите кнопку Talk в строке меню
  2. Или используйте горячую клавишу (по умолчанию: нет, можно настроить в Settings)
  3. Появляется слой Talk Mode

Состояния интерфейса Talk Mode:

СостояниеОтображениеОписание
ListeningАнимация импульса облака + громкость микрофонаОжидание вашей речи
ThinkingАнимация погруженияAI обрабатывает
SpeakingАнимация радиального кольца + волныAI отвечает (воспроизведение TTS)

Управление взаимодействием:

  • Остановить речь: Нажмите на иконку облака, чтобы остановить воспроизведение TTS
  • Выйти из Talk Mode: Нажмите кнопку X в правом верхнем углу
  • Голосовое прерывание: Когда AI говорит, если вы начинаете говорить, воспроизведение автоматически останавливается

Настройка TTS:

Talk Mode использует ElevenLabs для синтеза речи. Место конфигурации: ~/.clawdbot/clawdbot.json

yaml
talk:
  voiceId: "elevenlabs_voice_id"  # ID голоса ElevenLabs
  modelId: "eleven_v3"            # Версия модели
  apiKey: "elevenlabs_api_key"     # Ключ API (или используйте переменную окружения)
  interruptOnSpeech: true           # Прерывание при речи
  outputFormat: "mp3_44100_128"   # Формат вывода

Настройка ElevenLabs

Если ключ API не настроен, Talk Mode попытается использовать:

  1. Переменную окружения ELEVENLABS_API_KEY
  2. Ключ в профиле оболочки шлюза
  3. Первый доступный голос ElevenLabs как по умолчанию

Шаг 6: Использование режима узла ​

Зачем Режим узла позволяет приложению macOS предоставлять локальные возможности удаленному шлюзу, обеспечивая истинное межустройственное сотрудничество.

Доступные команды в режиме узла:

Категория командПример командыОписание функции
Canvascanvas.present, canvas.navigate, canvas.evalРендеринг интерфейсов визуализации на macOS
Cameracamera.snap, camera.clipФотографирование или видеозапись

Включение режима узла:

Режим узла автоматически запускается в удаленном режиме, так как удаленный шлюз должен вызывать локальные функции.

Вы также можете вручную запустить службу узла:

bash
clawdbot node run --display-name "My Mac"

Управление разрешениями узла:

Приложение macOS сообщает через систему разрешений, какие функции доступны:

json
{
  "canvas": true,
  "camera": true,
  "screen": true,
  "system": {
    "run": true,
    "notify": true
  }
}

AI автоматически выберет доступные инструменты на основе разрешений.

Шаг 7: Настройка Exec Approvals (контроль безопасности system.run) ​

Зачемsystem.run может выполнять произвольные команды Shell, поэтому требуется механизм одобрения для предотвращения ошибок или злоупотреблений.

Трехуровневая модель безопасности Exec Approvals:

json
{
  "version": 1,
  "defaults": {
    "security": "deny",          // Стратегия по умолчанию: отказ
    "ask": "on-miss"           // Запрос, если команды нет в белом списке
  },
  "agents": {
    "main": {
      "security": "allowlist",    // Основная сессия: только разрешения из белого списка
      "ask": "on-miss",
      "allowlist": [
        { "pattern": "/usr/bin/git" },
        { "pattern": "/opt/homebrew/*/rg" }
      ]
    }
  }
}

Типы стратегий безопасности:

СтратегияПоведениеПрименимые сценарии
denyОтклонить все вызовы system.runВысокая безопасность, отключение всех команд
allowlistРазрешить только команды из белого спискаБаланс безопасности и удобства
askЗапрос подтверждения пользователя, если нет в белом спискеГибкость, но требуется подтверждение

Процесс одобрения:

Когда AI пытается выполнить неавторизованную команду:

  1. Приложение macOS выводит диалоговое окно одобрения
  2. Отображается полный путь к команде и параметры
  3. Предоставляются три варианта:
    • Allow Once — Разрешить только в этот раз
    • Always Allow — Добавить в белый список
    • Deny — Отклонить выполнение

Что вы должны видеть:

  • Диалоговое окно одобрения отображает подробности команды (например, /usr/bin/ls -la ~)
  • После выбора "Always Allow" в следующий раз не будет повторно запрашивать
  • После выбора "Deny" выполнение команды завершается неудачно и возвращает ошибку AI

Место конфигурации:

Exec Approvals хранятся локально на macOS:

  • Файл: ~/.clawdbot/exec-approvals.json
  • История одобрения: Просмотреть все одобренные/отклоненные команды в приложении

Шаг 8: Использование глубоких ссылок ​

Зачем Глубокие ссылки обеспечивают возможность быстрого запуска функций Clawdbot из других приложений.

Поддерживаемый протокол глубоких ссылок: clawdbot://

clawdbot://agent ​

Запускает запрос agent шлюза, эквивалентно выполнению clawdbot agent в терминале.

Параметры:

ПараметрОписаниеПример
message (обязательный)Сообщение, отправляемое AImessage=Hello%20from%20deep%20link
sessionKey (необязательный)Ключ целевой сессии, по умолчанию mainsessionKey=main
thinking (необязательный)Уровень мышления: off|minimal|low|medium|high|xhighthinking=high
deliver/to/channel (необязательный)Канал доставкиchannel=telegram
timeoutSeconds (необязательный)Время ожиданияtimeoutSeconds=30
key (необязательный)Ключ без подтверждения, для автоматизацииkey=your-secret-key

Примеры:

bash
# Базовый: отправка сообщения
open 'clawdbot://agent?message=Hello%20from%20deep%20link'

# Продвинутый: отправка в Telegram, высокий уровень мышления, таймаут 30 секунд
open 'clawdbot://agent?message=Summarize%20my%20day&to=telegram&thinking=high&timeoutSeconds=30'

# Автоматизация: использование ключа для пропуска подтверждения (безопасно использовать только в ваших скриптах)
open 'clawdbot://agent?message=Automated%20task&key=secure-random-string'

Что вы должны видеть:

  • Приложение Clawdbot для macOS автоматически открывается (если не запущено)
  • Появляется окно Agent и отображается сообщение
  • AI начинает обрабатывать и возвращать ответ

Безопасность глубоких ссылок

  • Без параметра key приложение выводит диалоговое окно подтверждения
  • С действительным key запрос выполняется молча (для автоматических скриптов)
  • Никогда не используйте глубокие ссылки из ненадежных источников

Предупреждения о проблемах ​

❌ Разрешения отклонены или не предоставлены ​

Проблема:

  • Voice Wake не может прослушивать микрофон
  • Canvas не может отображать содержимое
  • Выполнение команд system.run завершается неудачно

Решение:

  1. Откройте Системные настройки → Конфиденциальность и безопасность
  2. Найдите Clawdbot или Clawdbot.app
  3. Убедитесь, что разрешения Микрофон, Вспомогательные функции, Запись экрана, Автоматизация включены
  4. Перезапустите приложение Clawdbot

Устранение неполадок с разрешениями TCC

Если переключатель разрешений не может быть включен или сразу выключается:

  • Проверьте, включены ли какие-либо инструменты безопасности (например, Little Snitch)
  • Попробуйте полностью удалить приложение и переустановить
  • Просмотрите логи отклонения TCC в Console.app

❌ Не удалось подключиться к шлюзу ​

Проблема:

  • Иконка в строке меню отображается красным (состояние отключено)
  • Карточка статуса отображает "Connection Failed"
  • WebChat не может быть открыт

Возможные причины и решения:

ПричинаСпособ проверкиРешение
Шлюз не запущенВыполните clawdbot gateway statusЗапустите службу шлюза
Неверный адресПроверьте WebSocket URLПодтвердите правильность ws://127.0.0.1:18789 или удаленного адреса
Порт занятВыполните lsof -i :18789Закройте процесс, занимающий порт
Сбой аутентификацииПроверьте Token/PasswordПодтвердите правильность учетных данных аутентификации

❌ Talk Mode не может использоваться ​

Проблема:

  • После включения Talk Mode нет реакции
  • TTS не может воспроизводиться
  • Микрофон не может вводить

Решение:

  1. Проверьте конфигурацию ElevenLabs:

    • Убедитесь, что ключ API настроен
    • Проверьте действительность ключа: посетите консоль ElevenLabs

  2. Проверьте сетевое подключение:

    • Для TTS требуется интернет-подключение
    • Проверьте, не блокирует ли брандмауэр запросы API

  3. Проверьте аудиовывод:

    • Убедитесь, что системная громкость включена
    • Проверьте правильность устройства вывода по умолчанию

❌ Узел не может подключиться в удаленном режиме ​

Проблема:

  • Удаленный шлюз не может вызывать команды, такие как system.run, на macOS
  • В журнале ошибок отображается "Node not found" или "Node offline"

Решение:

  1. Подтвердите работу службы узла:

    bash
    clawdbot nodes list
# Должен увидеть узел macOS, отображающийся как "paired"

  2. Проверьте SSH-туннель:

    • Просмотрите состояние подключения SSH в настройках приложения macOS
    • Подтвердите возможность ручного подключения по SSH к удаленному шлюзу

  3. Перезапустите службу узла:

    bash
    # На macOS
clawdbot node restart
