Полное руководство по настройке канала Slack ​

Чему вы научитесь ​

  • ✅ Взаимодействовать с Clawdbot в Slack, используя AI-помощника для выполнения задач
  • ✅ Настроить политику безопасности DM для защиты личной конфиденциальности
  • ✅ Интегрировать Clawdbot в группы для интеллектуального ответа на упоминания и команды
  • ✅ Использовать инструменты Slack Actions (отправка сообщений, управление Pin, просмотр информации о членах и т.д.)
  • ✅ Выбрать один из двух способов подключения — Socket Mode или HTTP Mode

Ваши текущие трудности ​

Slack — это ключевой инструмент для командного взаимодействия, но вы можете столкнуться со следующими проблемами:

  • Обсуждения команды разбросаны по нескольким каналам, и важная информация теряется
  • Необходимо быстро запросить исторические сообщения, Pin или информацию о членах, но интерфейс Slack неудобен
  • Хотите использовать AI-возможности непосредственно в Slack, не переключаясь на другие приложения
  • Боитесь, что включение AI-помощника в группах приведет к засорению сообщениями или утечке конфиденциальности

Когда использовать этот метод ​

  • Ежедневное командное общение: Slack — основной инструмент коммуникации вашей команды
  • Необходимость нативной интеграции со Slack: использование функций Reaction, Pin, Thread и т.д.
  • Несколько учетных записей: необходимо подключить несколько Slack Workspace
  • Сценарий удаленного развертывания: использование HTTP Mode для подключения к удаленному Gateway

🎒 Подготовка перед началом ​

Предварительная проверка

Перед началом убедитесь:

  • Завершили руководство Быстрый старт
  • Gateway запущен и работает
  • Имеете права администратора Slack Workspace (создание App)

Вам понадобятся следующие ресурсы:

  • Консоль Slack API — создание и управление Slack App
  • Конфигурационный файл Clawdbot — обычно расположен в ~/.clawdbot/clawdbot.json

Основная концепция ​

Канал Slack Clawdbot основан на фреймворке Bolt и поддерживает два режима подключения:

РежимСценарийПреимуществаНедостатки
Socket ModeЛокальный Gateway, личное использованиеПростая настройка (только Token)Требуется постоянное соединение WebSocket
HTTP ModeРазвертывание на сервере, удаленный доступПрохождение через брандмауэр, поддержка балансировки нагрузкиТребуется публичный IP, сложная настройка

По умолчанию используется Socket Mode, подходит для большинства пользователей.

Механизм аутентификации:

  • Bot Token (xoxb-...) — обязательно, для вызовов API
  • App Token (xapp-...) — обязательно для Socket Mode, для соединения WebSocket
  • User Token (xoxp-...) — опционально, для операций только для чтения (исторические записи, Pin, Reactions)
  • Signing Secret — обязательно для HTTP Mode, для проверки Webhook-запросов

Пошаговое руководство ​

Шаг 1: Создание Slack App ​

Зачем это нужно Slack App — это мост между Clawdbot и Workspace, предоставляющий аутентификацию и контроль разрешений.

  1. Перейдите на консоль Slack API
  2. Нажмите Create New App → выберите From scratch
  3. Заполните информацию о приложении:
    • App Name: Clawdbot (или любое название на ваш выбор)
    • Pick a workspace to develop your app in: выберите ваш Workspace
  4. Нажмите Create App

Что вы должны увидеть: Приложение успешно создано, переход на страницу базовой настройки.

Шаг 2: Настройка Socket Mode (рекомендуется) ​

Подсказка

Если вы используете локальный Gateway, рекомендуется Socket Mode, настройка проще.

Зачем это нужно Socket Mode не требует публичного IP, соединяется через сервис WebSocket Slack.

  1. На странице настройки приложения найдите Socket Mode, переключите в On
  2. Прокрутите до App-Level Tokens, нажмите Generate Token and Scopes
  3. В разделе Token выберите scope:
    • Отметьте connections:write
  4. Нажмите Generate Token, скопируйте сгенерированный App Token (начинается с xapp-)

Что вы должны увидеть: Сгенерированный Token выглядит так: xapp-1-A0B1C2D3E4F5G6H7I8J9K0L1M2N3O4P

Предупреждение о безопасности

App Token — это конфиденциальная информация, храните его надежно, не разглашайте в публичных репозиториях.

Шаг 3: Настройка Bot Token и разрешений ​

  1. Прокрутите до OAuth & Permissions → Bot Token Scopes
  2. Добавьте следующие scopes (разрешения):

Bot Token Scopes (обязательные):

yaml
    chat:write                    # Отправка/редактирование/удаление сообщений
    channels:history              # Чтение истории каналов
    channels:read                 # Получение информации о каналах
    groups:history                # Чтение истории групп
    groups:read                   # Получение информации о группах
    im:history                   # Чтение истории DM
    im:read                      # Получение информации DM
    im:write                     # Открытие сессий DM
    mpim:history                # Чтение истории групповых DM
    mpim:read                   # Получение информации групповых DM
    users:read                   # Запрос информации о пользователях
    app_mentions:read            # Чтение @упоминаний
    reactions:read               # Чтение Reaction
    reactions:write              # Добавление/удаление Reaction
    pins:read                    # Чтение списка Pin
    pins:write                   # Добавление/удаление Pin
    emoji:read                   # Чтение пользовательских Emoji
    commands                     # Обработка команд с косой чертой
    files:read                   # Чтение информации о файлах
    files:write                  # Загрузка файлов

Объяснение

Выше приведены обязательные разрешения для Bot Token, обеспечивающие нормальную работу Bot по чтению сообщений, отправке ответов, управлению Reaction и Pin.

  1. Прокрутите вверх страницы, нажмите Install to Workspace
  2. Нажмите Allow для авторизации App на доступ к вашему Workspace
  3. Скопируйте сгенерированный Bot User OAuth Token (начинается с xoxb-)

Что вы должны увидеть: Token выглядит так: xoxb-YOUR-BOT-TOKEN-HERE

Подсказка

Если вам нужен User Token (для операций только для чтения), прокрутите до User Token Scopes, добавьте следующие разрешения:

  • channels:history, groups:history, im:history, mpim:history
  • channels:read, groups:read, im:read, mpim:read
  • users:read, reactions:read, pins:read, emoji:read
  • search:read

Затем на странице Install App скопируйте User OAuth Token (начинается с xoxp-).

User Token Scopes (опционально, только для чтения):

  • Используется только для чтения исторических записей, Reaction, Pin, Emoji и поиска
  • Операции записи (отправка сообщений, добавление Reaction и т.д.) по-прежнему используют Bot Token (если не установлено userTokenReadOnly: false)

Шаг 4: Настройка подписки на события ​

  1. На странице настройки приложения найдите Event Subscriptions, включите Enable Events
  2. В Subscribe to bot events добавьте следующие события:
yaml
    app_mention                  # @упоминание Bot
    message.channels              # Сообщения каналов
    message.groups               # Сообщения групп
    message.im                   # Сообщения DM
    message.mpim                # Сообщения групповых DM
    reaction_added               # Добавление Reaction
    reaction_removed             # Удаление Reaction
    member_joined_channel       # Участник присоединился к каналу
    member_left_channel          # Участник покинул канал
    channel_rename               # Переименование канала
    pin_added                   # Добавление Pin
    pin_removed                 # Удаление Pin
  1. Нажмите Save Changes

Шаг 5: Включение функции DM ​

  1. На странице настройки приложения найдите App Home
  2. Включите Messages Tab → Enable Messages Tab
  3. Убедитесь, что отображается Messages tab read-only disabled: No

Что вы должны увидеть: Messages Tab включен, пользователи могут вести DM-диалог с Bot.

Шаг 6: Настройка Clawdbot ​

Зачем это нужно Настройка Slack Token в Clawdbot для установления соединения.

Способ 1: Использование переменных окружения (рекомендуется) ​

bash
    # Установка переменных окружения
    export SLACK_BOT_TOKEN="xoxb-ВашBotToken"
    export SLACK_APP_TOKEN="xapp-ВашAppToken"

    # Перезапуск Gateway
    clawdbot gateway restart

Что вы должны увидеть: В журналах запуска Gateway отображается Slack: connected.

Способ 2: Конфигурационный файл ​

Отредактируйте ~/.clawdbot/clawdbot.json:

json
{
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "xoxb-ВашBotToken",
      "appToken": "xapp-ВашAppToken"
    }
  }
}

Если у вас есть User Token:

json
{
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "xoxb-ВашBotToken",
      "appToken": "xapp-ВашAppToken",
      "userToken": "xoxp-ВашUserToken",
      "userTokenReadOnly": true
    }
  }
}

Что вы должны увидеть: После перезапуска Gateway соединение с Slack успешно.

Шаг 7: Приглашение Bot в канал ​

  1. В Slack откройте канал, в который хотите добавить Bot
  2. Введите /invite @Clawdbot (замените на имя вашего Bot)
  3. Нажмите Add to channel

Что вы должны увидеть: Bot успешно добавлен в канал, отображается сообщение "Clawdbot has joined the channel".

Шаг 8: Настройка политики безопасности групп ​

Зачем это нужно Предотвратить автоматический ответ Bot во всех каналах для защиты конфиденциальности.

Отредактируйте ~/.clawdbot/clawdbot.json:

json
{
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "xoxb-ВашBotToken",
      "appToken": "xapp-ВашAppToken",
      "groupPolicy": "allowlist",
      "channels": {
        "C1234567890": {
          "allow": true,
          "requireMention": true
        },
        "#general": {
          "allow": true,
          "requireMention": true
        }
      }
    }
  }
}

Описание полей:

  • groupPolicy: политика групп
    • "open" — разрешить все каналы (не рекомендуется)
    • "allowlist" — разрешить только перечисленные каналы (рекомендуется)
    • "disabled" — запретить все каналы
  • channels: конфигурация каналов
    • allow: разрешить/запретить
    • requireMention: требуется ли @упоминание Bot для ответа (по умолчанию true)
    • users: дополнительный белый список пользователей
    • skills: ограничение навыков, используемых в этом канале
    • systemPrompt: дополнительные системные подсказки

Что вы должны увидеть: Bot отвечает на сообщения только в настроенных каналах и требует @упоминание.

Шаг 9: Настройка политики безопасности DM ​

Зачем это нужно Предотвратить взаимодействие незнакомцев с Bot через DM, защитить конфиденциальность.

Отредактируйте ~/.clawdbot/clawdbot.json:

json
{
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "xoxb-ВашBotToken",
      "appToken": "xapp-ВашAppToken",
      "dm": {
        "enabled": true,
        "policy": "pairing",
        "allowFrom": ["U1234567890", "@alice", "user@example.com"]
      }
    }
  }
}

Описание полей:

  • dm.enabled: включить/отключить DM (по умолчанию true)
  • dm.policy: политика DM
    • "pairing" — незнакомцы получают код парной аутентификации, требуется одобрение (по умолчанию)
    • "open" — разрешить DM для всех
    • "allowlist" — разрешить только пользователей из белого списка
  • dm.allowFrom: список белого списка
    • Поддержка ID пользователя (U1234567890)
    • Поддержка @упоминания (@alice)
    • Поддержка электронной почты (user@example.com)

Процесс парной аутентификации:

  1. Незнакомец отправляет DM Bot
  2. Bot отвечает кодом парной аутентификации (срок действия 1 час)
  3. Пользователь предоставляет код парной аутентификации администратору
  4. Администратор выполняет: clawdbot pairing approve slack <код парной аутентификации>
  5. Пользователь добавляется в белый список и может нормально использовать

Что вы должны видеть: Неизвестный отправитель получает код парной аутентификации, Bot не обрабатывает их сообщения.

Шаг 10: Тестирование Bot ​

  1. В настроенном канале отправьте сообщение: @Clawdbot привет
  2. Или отправьте DM Bot
  3. Наблюдайте за ответом Bot

Что вы должны увидеть: Bot нормально отвечает на ваши сообщения.

Предупреждения о типичных проблемах ​

Распространенная ошибка 1: Bot не отвечает ​

Проблема: Bot не отвечает после отправки сообщения.

Возможные причины:

  1. Bot не добавлен в канал → используйте /invite @Clawdbot для приглашения
  2. requireMention установлено в true → при отправке сообщения требуется @Clawdbot
  3. Ошибка конфигурации Token → проверьте, правильны ли Token в clawdbot.json
  4. Gateway не запущен → выполните clawdbot gateway status для проверки состояния

Распространенная ошибка 2: Сбой соединения Socket Mode ​

Проблема: В журналах Gateway отображается сбой соединения.

Решение:

  1. Проверьте, правильный ли App Token (начинается с xapp-)
  2. Проверьте, включен ли Socket Mode
  3. Проверьте сетевое соединение

Распространенная ошибка 3: Недостаточно разрешений User Token ​

Проблема: Некоторые операции не выполняются, выводится ошибка разрешений.

Решение:

  1. Убедитесь, что User Token содержит необходимые разрешения (см. шаг 3)
  2. Проверьте настройку userTokenReadOnly (по умолчанию true, только чтение)
  3. Если требуются операции записи, установите "userTokenReadOnly": false

Распространенная ошибка 4: Сбой разрешения ID канала ​

Проблема: Настроенное имя канала не может быть разрешено в ID.

Решение:

  1. Приоритетное использование ID канала (например, C1234567890) вместо имени
  2. Убедитесь, что имя канала начинается с # (например, #general)
  3. Проверьте, есть ли у Bot права доступа к этому каналу

Продвинутая конфигурация ​

Описание разрешений ​

Bot Token vs User Token

  • Bot Token: обязательно, для основных функций Bot (отправка сообщений, чтение истории, управление Pin/Reaction и т.д.)
  • User Token: опционально, только для операций только для чтения (исторические записи, Reaction, Pin, Emoji, поиск)
    • По умолчанию userTokenReadOnly: true, обеспечивающий только чтение
    • Операции записи (отправка сообщений, добавление Reaction и т.д.) по-прежнему используют Bot Token

Возможно требуемые разрешения в будущем:

Следующие разрешения не являются обязательными в текущей версии, но могут быть добавлены в будущем:

РазрешениеНазначение
groups:writeУправление приватными каналами (создание, переименование, приглашение, архивация)
mpim:writeУправление сессиями групповых DM
chat:write.publicПубликация сообщений в каналы, где Bot не участвует
files:readСписок/чтение метаданных файлов

Если вы хотите включить эти функции, добавьте соответствующие разрешения в Bot Token Scopes Slack App.

HTTP Mode (развертывание на сервере) ​

Если ваш Gateway развернут на удаленном сервере, используйте HTTP Mode:

  1. Создайте Slack App, отключите Socket Mode
  2. Скопируйте Signing Secret (страница Basic Information)
  3. Настройте Event Subscriptions, установите Request URL на https://ваше-доменное-имя/slack/events
  4. Настройте Interactivity & Shortcuts, установите тот же Request URL
  5. Настройте Slash Commands, установите Request URL

Конфигурационный файл:

json
{
  "channels": {
    "slack": {
      "enabled": true,
      "mode": "http",
      "botToken": "xoxb-ВашBotToken",
      "signingSecret": "ВашSigningSecret",
      "webhookPath": "/slack/events"
    }
  }
}

Настройка нескольких учетных записей ​

Поддержка подключения нескольких Slack Workspace:

json
{
  "channels": {
    "slack": {
      "accounts": {
        "workspace1": {
          "name": "Команда A",
          "enabled": true,
          "botToken": "xoxb-Workspace1Token",
          "appToken": "xapp-Workspace1Token"
        },
        "workspace2": {
          "name": "Команда B",
          "enabled": true,
          "botToken": "xoxb-Workspace2Token",
          "appToken": "xapp-Workspace2Token"
        }
      }
    }
  }
}

Настройка команд с косой чертой ​

Включение команды /clawd:

  1. На странице настройки приложения найдите Slash Commands
  2. Создайте команду:
    • Command: /clawd
    • Request URL: для Socket Mode не требуется (обрабатывается через WebSocket)
    • Description: Send a message to Clawdbot

Конфигурационный файл:

json
{
  "channels": {
    "slack": {
      "slashCommand": {
        "enabled": true,
        "name": "clawd",
        "ephemeral": true
      }
    }
  }
}

Настройка режима ответа в потоке ​

Контроль способа ответа Bot в каналах:

json
{
  "channels": {
    "slack": {
      "replyToMode": "off",
      "replyToModeByChatType": {
        "direct": "all",
        "group": "first"
      }
    }
  }
}
РежимПоведение
offПо умолчанию, ответ в основном канале
firstПервый ответ в потоке, последующие ответы в основном канале
allВсе ответы в потоке

Включение инструментов Slack Actions ​

Разрешить Agent вызывать специфические операции Slack:

json
{
  "channels": {
    "slack": {
      "actions": {
        "reactions": true,
        "messages": true,
        "pins": true,
        "memberInfo": true,
        "emojiList": true
      }
    }
  }
}

Доступные операции:

  • sendMessage — отправить сообщение
  • editMessage — редактировать сообщение
  • deleteMessage — удалить сообщение
  • readMessages — прочитать исторические сообщения
  • react — добавить Reaction
  • reactions — перечислить Reactions
  • pinMessage — закрепить сообщение (Pin)
  • unpinMessage — открепить сообщение
  • listPins — перечислить Pin
  • memberInfo — получить информацию об участнике
  • emojiList — перечислить пользовательские Emoji

Итоги урока ​

  • Канал Slack поддерживает два способа подключения — Socket Mode и HTTP Mode
  • Socket Mode проще в настройке, рекомендуется для локального использования
  • Политика безопасности DM по умолчанию — pairing, незнакомцы требуют одобрения
  • Политика безопасности групп поддерживает белый список и фильтрацию @упоминаний
  • Инструменты Slack Actions предоставляют богатые возможности операций
  • Поддержка нескольких учетных записей для подключения нескольких Workspace
