Руководство по настройке iOS узла ​

Что вы сможете сделать после изучения ​

После настройки iOS узла вы сможете:

  • ✅ Позволить AI ассистенту использовать камеру устройства iOS для съемки фото или записи видео
  • ✅ Отображать визуальный интерфейс Canvas на устройстве iOS
  • ✅ Использовать Voice Wake и Talk Mode для голосового взаимодействия
  • ✅ Получать информацию о местоположении устройства iOS
  • ✅ Централизованно управлять несколькими узлами устройств через Gateway

Ваша текущая ситуация ​

Вы хотите расширить возможности AI ассистента на вашем устройстве iOS, чтобы он мог:

  • Использовать камеру для съемки фото или записи видео: когда вы говорите "сделай фото", AI автоматически использует iPhone для съемки
  • Отображать визуальный интерфейс: показывать на iPhone сгенерированные AI графики, формы или панели управления
  • Голосовое пробуждение и непрерывный диалог: без необходимости касаться устройства, просто скажите "Clawd" для пробуждения ассистента и начала диалога
  • Получать информацию об устройстве: позволить AI знать ваше местоположение, состояние экрана и другую информацию

Когда использовать этот подход ​

  • Мобильные сценарии: вы хотите, чтобы AI мог использовать возможности камеры, экрана и другие функции iPhone
  • Многоплатформенное взаимодействие: Gateway запущен на сервере, но необходимо вызывать функции локального устройства
  • Голосовое взаимодействие: вы хотите использовать iPhone как портативный терминал голосового ассистента

Что такое iOS узел?

iOS узел — это Companion приложение, работающее на iPhone/iPad, которое подключается к Clawdbot Gateway через WebSocket. Это не сам Gateway, а "периферийное устройство", предоставляющее возможности локальных операций на устройстве.

Отличие от Gateway:

  • Gateway: работает на сервере/macOS, отвечает за маршрутизацию сообщений, вызов AI моделей, распределение инструментов
  • iOS узел: работает на iPhone, отвечает за выполнение локальных операций на устройстве (камера, Canvas, местоположение и т.д.)

🎒 Подготовка к началу ​

Требования

Перед началом убедитесь:

  1. Gateway запущен и работает

    • Убедитесь, что Gateway работает на другом устройстве (macOS, Linux или Windows via WSL2)
    • Gateway привязан к доступному сетевому адресу (локальная сеть или Tailscale)

  2. Сетевая доступность

    • Устройство iOS и Gateway в одной локальной сети (рекомендуется) или подключены через Tailscale
    • Устройство iOS может обращаться к IP адресу и порту Gateway (по умолчанию 18789)

  3. Получение iOS приложения

    • iOS приложение в настоящее время находится в стадии внутренней предварительной версии, публично не распространяется
    • Необходимо собрать из исходного кода или получить тестовую версию TestFlight

Основная концепция ​

Рабочий процесс iOS узла:

[Gateway] ←→ [iOS узел]
     ↓            ↓
  [AI модель]   [Возможности устройства]
     ↓            ↓
  [Принятие решения и выполнение]   [Камера/Canvas/Голос]

Ключевые технические моменты:

  1. Автоматическое обнаружение: автоматическое обнаружение Gateway через Bonjour (локальная сеть) или Tailscale (кросс-сеть)
  2. Аутентификация пары: при первом подключении требуется ручное одобрение со стороны Gateway для установления доверенных отношений
  3. Протокольное взаимодействие: использование протокола WebSocket (node.invoke) для отправки команд
  4. Управление разрешениями: локальные команды устройства требуют авторизации пользователя (камера, местоположение и т.д.)

Архитектурные особенности:

  • Безопасность: все операции на устройстве требуют явной авторизации пользователя на стороне iOS
  • Изоляция: узел не запускает Gateway, а выполняет только локальные операции
  • Гибкость: поддержка различных сценариев использования: foreground, background, удаленный доступ и т.д.

Следуйте инструкциям ​

Шаг 1: Запуск Gateway ​

Запустите сервис на хосте Gateway:

bash
clawdbot gateway --port 18789

Вы должны увидеть:

✅ Gateway running on ws://0.0.0.0:18789
✅ Bonjour advertisement active: _clawdbot._tcp

Доступ через разные сети

Если Gateway и устройство iOS не находятся в одной локальной сети, используйте Tailscale Serve/Funnel:

bash
clawdbot gateway --port 18789 --tailscale funnel

Устройство iOS автоматически обнаружит Gateway через Tailscale.

Шаг 2: Подключение iOS приложения ​

В iOS приложении:

  1. Откройте Settings (Настройки)
  2. Найдите раздел Gateway
  3. Выберите автоматически обнаруженный Gateway (или включите Manual Host ниже для ручного ввода хоста и порта)

Вы должны увидеть:

  • Приложение пытается подключиться к Gateway
  • Статус показывает "Connected" или "Pairing pending"
Ручная настройка хоста

Если автоматическое обнаружение не сработало, введите адрес Gateway вручную:

  1. Включите Manual Host
  2. Введите хост Gateway (например, 192.168.1.100)
  3. Введите порт (по умолчанию 18789)
  4. Нажмите "Connect"

Шаг 3: Одобрение запроса на сопряжение ​

На хосте Gateway, одобрите запрос на сопряжение iOS узла:

bash
# Просмотр ожидающих утверждения узлов
clawdbot nodes pending

# Одобрить конкретный узел (замените <requestId>)
clawdbot nodes approve <requestId>

Вы должны увидеть:

✅ Node paired successfully
Node: iPhone (iOS)
ID: node-abc123

Отклонение сопряжения

Если вы хотите отклонить запрос на соединение определенного узла:

bash
clawdbot nodes reject <requestId>

Контрольная точка ✅: проверка статуса узла на Gateway

bash
clawdbot nodes status

Вы должны увидеть, что ваш iOS узел отображается в статусе paired.

Шаг 4: Проверка подключения узла ​

Проверка коммуникации узла из Gateway:

bash
# Вызов команды узла через Gateway
clawdbot gateway call node.list --params "{}"

Вы должны увидеть:

json
{
  "result": [
    {
      "id": "node-abc123",
      "displayName": "iPhone (iOS)",
      "platform": "ios",
      "capabilities": ["camera", "canvas", "location", "screen", "voicewake"]
    }
  ]
}

Использование функций узла ​

Камера для фото ​

iOS узел поддерживает съемку фото и запись видео:

bash
# Съемка фото (по умолчанию фронтальная камера)
clawdbot nodes camera snap --node "iPhone (iOS)"

# Съемка фото (тыловая камера, пользовательское разрешение)
clawdbot nodes camera snap --node "iPhone (iOS)" --facing back --max-width 1920

# Запись видео (5 секунд)
clawdbot nodes camera clip --node "iPhone (iOS)" --duration 5000

Вы должны увидеть:

MEDIA:/tmp/clawdbot-camera-snap-abc123.jpg

Требование foreground

Команды камеры требуют, чтобы iOS приложение было в foreground. Если приложение находится в background, будет возвращена ошибка NODE_BACKGROUND_UNAVAILABLE.

Параметры камеры iOS:

ПараметрТипПо умолчаниюОписание
facingfront|backfrontОриентация камеры
maxWidthnumber1600Максимальная ширина (пиксели)
quality0..10.9Качество JPEG (0-1)
durationMsnumber3000Длительность видео (миллисекунды)
includeAudiobooleantrueВключать ли аудио

Визуальный интерфейс Canvas ​

iOS узел может отображать визуальный интерфейс Canvas:

bash
# Навигация по URL
clawdbot nodes canvas navigate --node "iPhone (iOS)" --target "https://example.com"

# Выполнение JavaScript
clawdbot nodes canvas eval --node "iPhone (iOS)" --js "document.title"

# Скриншот (сохранить как JPEG)
clawdbot nodes canvas snapshot --node "iPhone (iOS)" --format jpeg --max-width 900

Вы должны увидеть:

MEDIA:/tmp/clawdbot-canvas-snap-abc123.jpg

Автоматическая отправка A2UI

Если Gateway настроен с canvasHost, iOS узел автоматически перейдет к интерфейсу A2UI при подключении.

Voice Wake голосовое пробуждение ​

Включите Voice Wake в Settings приложения iOS:

  1. Включите переключатель Voice Wake
  2. Установите слово пробуждения (по умолчанию: "clawd", "claude", "computer")
  3. Убедитесь, что iOS предоставил разрешение на использование микрофона

Глобальное слово пробуждения

Слово пробуждения Clawdbot — это глобальная конфигурация, управляемая Gateway. Все узлы (iOS, Android, macOS) используют один и тот же список слов пробуждения.

Изменение слова пробуждения автоматически синхронизируется на все устройства.

Talk Mode непрерывный диалог ​

После включения Talk Mode AI будет постоянно озвучивать ответы через TTS и непрерывно прослушивать голосовой ввод:

  1. Включите Talk Mode в Settings приложения iOS
  2. При ответе AI он автоматически будет озвучиваться
  3. Можно вести диалог голосом без необходимости ручного нажатия

Ограничения background

iOS может приостанавливать фоновое аудио. Когда приложение не находится в foreground, голосовые функции работают по мере возможности (best-effort).

Часто встречающиеся вопросы ​

Запрос на сопряжение никогда не появляется ​

Проблема: приложение iOS показывает "Connected", но Gateway не выводит запрос на сопряжение.

Решение:

bash
# 1. Ручной просмотр ожидающих узлов
clawdbot nodes pending

# 2. Одобрить узел
clawdbot nodes approve <requestId>

# 3. Проверить подключение
clawdbot nodes status

Неудачное подключение (после переустановки) ​

Проблема: после переустановки приложения iOS невозможно подключиться к Gateway.

Причина: токен сопряжения в Keychain был очищен.

Решение: повторно выполните процесс сопряжения (шаг 3).

A2UI_HOST_NOT_CONFIGURED ​

Проблема: команды Canvas не выполняются, выводится A2UI_HOST_NOT_CONFIGURED.

Причина: Gateway не настроен с URL canvasHost.

Решение:

Настройте хост Canvas в конфигурации Gateway:

bash
clawdbot config set canvasHost "http://<gateway-host>:18793/__clawdbot__/canvas/"

NODE_BACKGROUND_UNAVAILABLE ​

Проблема: команды камеры/Canvas не выполняются, возвращается NODE_BACKGROUND_UNAVAILABLE.

Причина: приложение iOS не находится в foreground.

Решение: переключите приложение iOS в foreground, затем повторите команду.

Ключевые константы:

  • GATEWAY_DEFAULT_PORT = 18789: порт Gateway по умолчанию
  • NODE_ROLE = "node": идентификатор роли подключения узла

Ключевые команды:

  • clawdbot nodes pending: вывести список ожидающих утверждения узлов
  • clawdbot nodes approve <requestId>: одобрить сопряжение узла
  • clawdbot nodes invoke --node <id> --command <cmd>: вызвать команду узла
  • clawdbot nodes camera snap --node <id>: сделать фото
  • clawdbot nodes canvas navigate --node <id> --target <url>: навигация Canvas

Методы протокола:

  • node.invoke.request: запрос вызова команды узла
  • node.invoke.result: результат выполнения команды узла
  • voicewake.get: получить список слов пробуждения
  • voicewake.set: установить список слов пробуждения
  • voicewake.changed: событие изменения слова пробуждения
