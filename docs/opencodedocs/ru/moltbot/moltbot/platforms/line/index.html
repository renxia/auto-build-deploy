Настройка и использование канала LINE ​

Чему вы научитесь ​

После завершения этого руководства вы сможете:

  • ✅ Создать канал LINE Messaging API и получить учетные данные
  • ✅ Настроить плагин LINE и Webhook в Clawdbot
  • ✅ Настроить сопряжение DM, контроль доступа в группах и ограничения для медиафайлов
  • ✅ Отправлять богатые медиа-сообщения (Flex-карточки, быстрые ответы, информация о местоположении)
  • ✅ Устранять распространенные проблемы с каналом LINE

Ваша текущая проблема ​

Возможно, вы думаете:

  • "Я хочу общаться с AI-помощником через LINE, как интегрировать?"
  • "Как настроить Webhook для LINE Messaging API?"
  • "Поддерживает ли LINE Flex-сообщения и быстрые ответы?"
  • "Как контролировать, кто может получить доступ к моему AI-помощнику через LINE?"

Хорошая новость: Clawdbot предоставляет полный плагин LINE, поддерживающий все основные функции Messaging API.

Когда использовать этот подход ​

Когда вам нужно:

  • 📱 Общаться с AI-помощником через LINE
  • 🎨 Использовать богатые медиа-сообщения (Flex-карточки, быстрые ответы, Rich Menu)
  • 🔒 Контролировать права доступа (сопряжение DM, белый список групп)
  • 🌐 Интегрировать LINE в существующие рабочие процессы

Основная идея ​

Канал LINE интегрируется через LINE Messaging API, используя Webhook для получения событий и отправки сообщений.

Пользователь LINE
    │
    ▼ (отправляет сообщение)
┌──────────────────┐
│  LINE Platform  │
│  (Messaging API)│
└────────┬─────────┘
         │ (Webhook POST)
         ▼
┌──────────────────┐
│  Clawdbot       │
│  Gateway        │
│  /line/webhook   │
└────────┬─────────┘
         │ (вызывает AI)
         ▼
     ┌────────┐
     │ Agent  │
     └───┬────┘
         │ (ответ)
         ▼
     Пользователь LINE

Ключевые понятия:

ПонятиеНазначение
Channel Access TokenТокен аутентификации для отправки сообщений
Channel SecretКлюч для проверки подписи Webhook
Webhook URLКонечная точка Clawdbot для получения событий LINE (обязательно HTTPS)
DM PolicyСтратегия доступа для неизвестных отправителей (pairing/allowlist/open/disabled)
Rich MenuФиксированное меню LINE, пользователи могут быстро запускать действия

Пошаговое руководство ​

Шаг 1: Установка плагина LINE ​

Зачем Канал LINE реализован через плагин, который нужно сначала установить.

bash
clawdbot plugins install @clawdbot/line

Вы должны увидеть:

✓ Installed @clawdbot/line plugin

Локальная разработка

Если вы запускаете из исходного кода, можно использовать локальную установку:

bash
clawdbot plugins install ./extensions/line

Шаг 2: Создание LINE Messaging API Channel ​

Зачем Нужно получить Channel Access Token и Channel Secret для настройки Clawdbot.

2.1 Войдите в LINE Developers Console ​

Перейдите: https://developers.line.biz/console/

2.2 Создайте Provider (если его еще нет) ​

  1. Нажмите "Create new provider"
  2. Введите имя Provider (например Clawdbot)
  3. Нажмите "Create"

2.3 Добавьте Messaging API Channel ​

  1. Под Provider нажмите "Add channel" → выберите "Messaging API"
  2. Настройте информацию о канале:
    • Channel name: Clawdbot AI Assistant
    • Channel description: Personal AI assistant powered by Clawdbot
    • Category: Communication
    • Subcategory: Bot
  3. Отметьте "Agree" → нажмите "Create"

2.4 Включите Webhook ​

  1. На странице настроек канала найдите вкладку "Messaging API"
  2. Нажмите переключатель "Use webhook" → установите в ON
  3. Скопируйте следующую информацию:
ЭлементРасположениеПример
Channel Access TokenBasic settings → Channel access token (long-lived)eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Channel SecretBasic settings → Channel secret1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6p7

Сохраните учетные данные!

Channel Access Token и Channel Secret являются конфиденциальной информацией. Храните их безопасно, не публикуйте в открытом репозитории.

Шаг 3: Настройка канала LINE в Clawdbot ​

Зачем Настроить Gateway для использования LINE Messaging API для отправки и получения сообщений.

Способ A: Настройка через командную строку ​

bash
clawdbot configure

Мастер запросит:

  • Включить ли канал LINE
  • Channel Access Token
  • Channel Secret
  • DM стратегию (по умолчанию pairing)

Способ B: Прямое редактирование файла конфигурации ​

Редактируйте ~/.clawdbot/clawdbot.json:

json5
{
  channels: {
    line: {
      enabled: true,
      channelAccessToken: "YOUR_CHANNEL_ACCESS_TOKEN",
      channelSecret: "YOUR_CHANNEL_SECRET",
      dmPolicy: "pairing",
      groupPolicy: "allowlist"
    }
  }
}

Использование переменных окружения

Вы также можете настроить через переменные окружения (только для учетной записи по умолчанию):

bash
export LINE_CHANNEL_ACCESS_TOKEN="your_token_here"
export LINE_CHANNEL_SECRET="your_secret_here"

Способ C: Использование файлов для хранения учетных данных ​

Более безопасный способ - хранить учетные данные в отдельных файлах:

json5
{
  channels: {
    line: {
      enabled: true,
      tokenFile: "/path/to/line-token.txt",
      secretFile: "/path/to/line-secret.txt",
      dmPolicy: "pairing"
    }
  }
}

Шаг 4: Настройка Webhook URL ​

Зачем LINE нужен Webhook URL для отправки событий сообщений в Clawdbot.

4.1 Убедитесь, что ваш Gateway доступен из интернета ​

Если вы разрабатываете локально, нужно использовать сервис туннелирования:

bash
# Использование ngrok
ngrok http 18789

# Вывод покажет HTTPS URL, например:
# Forwarding: https://abc123.ngrok.io -> http://localhost:18789

4.2 Настройте Webhook URL в LINE Console ​

  1. На странице настроек Messaging API найдите "Webhook settings"
  2. Введите Webhook URL:
    https://your-gateway-host/line/webhook
    • Путь по умолчанию: /line/webhook
    • Можно настроить через channels.line.webhookPath
  3. Нажмите "Verify" → убедитесь, что LINE может получить доступ к вашему Gateway

Вы должны увидеть:

✓ Webhook URL verification succeeded

4.3 Включите необходимые типы событий ​

В Webhook settings отметьте следующие события:

СобытиеНазначение
Message eventПолучение сообщений от пользователей
Follow eventПользователь добавляет Bot в друзья
Unfollow eventПользователь удаляет Bot
Join eventBot присоединяется к группе
Leave eventBot покидает группу
Postback eventБыстрые ответы и нажатия кнопок

Шаг 5: Запуск Gateway ​

Зачем Gateway должен работать для получения событий Webhook от LINE.

bash
clawdbot gateway --verbose

Вы должны увидеть:

✓ Gateway listening on ws://127.0.0.1:18789
✓ LINE webhook server started on /line/webhook
✓ LINE plugin initialized

Шаг 6: Тестирование канала LINE ​

Зачем Проверить правильность конфигурации, корректность ответов AI-помощника.

6.1 Добавьте Bot в друзья ​

  1. В LINE Console → Messaging API → Channel settings
  2. Скопируйте "Basic ID" или "QR Code"
  3. В LINE App найдите или отсканируйте QR Code, добавьте Bot в друзья

6.2 Отправьте тестовое сообщение ​

Отправьте сообщение Bot в LINE:

Здравствуйте, помогите мне подвести итог сегодняшней погоды.

Вы должны увидеть:

  • Bot показывает состояние "typing" (если настроены индикаторы набора)
  • AI-помощник возвращает ответ потоком
  • Сообщение корректно отображается в LINE

Шаг 7: Проверка сопряжения DM (опционально) ​

Зачем Если используется dmPolicy="pairing" по умолчанию, неизвестные отправители должны быть сначала одобрены.

Просмотр ожидающих запросов на сопряжение ​

bash
clawdbot pairing list line

Вы должны увидеть:

Pending pairing requests for LINE:
  CODE: ABC123 - User ID: U1234567890abcdef1234567890ab

Одобрение запроса на сопряжение ​

bash
clawdbot pairing approve line ABC123

Вы должны увидеть:

✓ Approved pairing request for LINE user U1234567890abcdef1234567890ab

Объяснение стратегии DM

  • pairing (по умолчанию): неизвестные отправители получают код сопряжения, сообщения игнорируются до одобрения
  • allowlist: только пользователи из белого списка могут отправлять сообщения
  • open: любой может отправлять сообщения (используйте с осторожностью)
  • disabled: отключить личные сообщения

Подводные камни ​

Распространенная проблема 1: Ошибка проверки Webhook ​

Симптомы:

Webhook URL verification failed

Причины:

  • Webhook URL не HTTPS
  • Gateway не работает или порт неверен
  • Брандмауэр блокирует входящие соединения

Решение:

  1. Убедитесь, что используете HTTPS: https://your-gateway-host/line/webhook
  2. Проверьте, работает ли Gateway: clawdbot gateway status
  3. Проверьте порт: netstat -an | grep 18789
  4. Используйте сервис туннелирования (ngrok/Tailscale/Cloudflare)

Распространенная проблема 2: Невозможно получить сообщения ​

Симптомы:

  • Проверка Webhook успешна
  • Но отправка сообщений Bot не отвечает

Причины:

  • Неверная настройка пути Webhook
  • Типы событий не включены
  • channelSecret в файле конфигурации не совпадает

Решение:

  1. Проверьте, совпадает ли channels.line.webhookPath с LINE Console
  2. Убедитесь, что "Message event" включен в LINE Console
  3. Проверьте, правильно ли скопирован channelSecret (без лишних пробелов)

Распространенная проблема 3: Ошибка загрузки медиафайлов ​

Симптомы:

Error downloading LINE media: size limit exceeded

Причины:

  • Файл медиа превышает лимит по умолчанию (10MB)

Решение: Увеличьте лимит в конфигурации:

json5
{
  channels: {
    line: {
      mediaMaxMb: 25  // Официальный лимит LINE 25MB
    }
  }
}

Распространенная проблема 4: Нет ответа на сообщения в группах ​

Симптомы:

  • DM работает нормально
  • В группах сообщения не отвечают

Причины:

  • По умолчанию groupPolicy="allowlist", группы не добавлены в белый список
  • Bot не упомянут @ в группе

Решение:

  1. Добавьте ID группы в белый список в конфигурации:
json5
{
  channels: {
    line: {
      groupAllowFrom: ["C1234567890abcdef1234567890ab"]
    }
  }
}
  1. Или упомяните @Bot в группе: @Clawdbot помоги мне обработать эту задачу

Rich Menu (фиксированное меню) ​

Rich Menu - это фиксированное меню LINE, пользователи могут быстро запускать действия нажатием.

bash
# Создать Rich Menu
clawdbot line rich-menu create

# Загрузить изображение меню
clawdbot line rich-menu upload --image /path/to/menu.png

# Установить как меню по умолчанию
clawdbot line rich-menu set-default --rich-menu-id <MENU_ID>

Ограничения Rich Menu

  • Размер изображения: 2500x1686 или 2500x843 пикселей
  • Формат изображения: PNG или JPEG
  • Максимум 10 пунктов меню

Конвертация Markdown ​

Clawdbot автоматически конвертирует формат Markdown в формат, поддерживаемый LINE:

MarkdownРезультат конвертации LINE
Блок кодаFlex-карточка
ТаблицаFlex-карточка
СсылкиАвтоматически обнаруживаются и конвертируются в Flex-карточку
Жирный/курсивУдаляются (LINE не поддерживает)

Сохранение формата

LINE не поддерживает формат Markdown, Clawdbot пытается конвертировать в Flex-карточки. Если вы хотите простой текст, можно отключить автоматическую конвертацию в конфигурации.
