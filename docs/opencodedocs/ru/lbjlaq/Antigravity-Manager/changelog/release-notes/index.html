Версионная эволюция: основан на встроенном Changelog README ​

Вы готовитесь обновить Antigravity Tools, больше всего боитесь не "не обновлено", а "обновил,才发现 есть изменения совместимости". Эта страница объясняет метод чтения Changelog версионной эволюции Antigravity Tools, чтобы вы могли определить до обновления: что повлияет на вас в этом обновлении.

Чему вы научитесь ​

  • На странице About быстро подтвердить текущую версию, проверить обновления и получить вход загрузки
  • В Changelog README читать только те сегменты версии, которые влияют на вас (а не перелистывать с начала до конца)
  • Перед обновлением сделать одну вещь: подтвердить, есть ли "напоминания, требующие изменить конфигурацию/сопоставление моделей вручную"
  • После обновления запустить минимальную проверку (/healthz), чтобы подтвердить, что прокси еще можно использовать

Что такое Changelog? ​

Changelog - это список изменений "что изменилось в этот раз" по версиям. Antigravity Tools пишет его напрямую в "Версионная эволюция" README, каждая версия отмечена датой и ключевыми изменениями. Перед обновлением сначала посмотрите Changelog, это уменьшит вероятность столкнуться с проблемами совместимости или регрессией.

Когда использовать эту страницу ​

  • Вы собираетесь обновить со старой версии на новую, хотите сначала подтвердить точки риска
  • Вы столкнулись с какой-то проблемой (например, 429/0 Token/Cloudflared), хотите подтвердить, исправлена ли она в последних версиях
  • Вы поддерживаете унифицированную версию в команде, нужно дать коллегам метод "читать изменения по версиям"

🎒 Подготовка перед началом ​

Сначала подготовьте путь обновления

Много способов установки/обновления (brew, ручная загрузка Releases, обновление внутри приложения). Если вы еще не определили, какой путь использовать, сначала смотрите Установка и обновление: лучший путь установки для настольной версии (brew / releases).

Делайте вместе со мной ​

Шаг 1: На странице About подтвердите версию, которую вы используете ​

Зачем Changelog организован по версиям. Сначала зная вашу текущую версию, вы знаете "с чего начать чтение".

Путь операции: Settings → About.

Вы должны увидеть: В заголовке страницы отображается имя приложения и номер версии (например, v3.3.49).

Шаг 2: Нажмите "Проверить обновления", получите последнюю версию и вход загрузки ​

Зачем Вам нужно сначала знать "какой номер последней версии", затем в Changelog выбрать сегменты версий между ними.

На странице About нажмите "Проверить обновления".

Вы должны увидеть:

  • Если есть обновление: появится подсказка "new version available", и появится кнопка загрузки (открывает download_url)
  • Если уже последняя: появится подсказка "latest version"

Шаг 3: В Changelog README читайте только те версии, которые вы перескочили ​

Зачем Вам нужно заботиться только о изменениях "между вашей текущей версией и последней версией", другие исторические версии можно сначала пропустить.

Откройте README, найдите "Версионная эволюция (Changelog)", начинайте с последней версии и читайте вниз, пока не увидите вашу текущую версию.

Вы должны увидеть: Версии перечислены в формате vX.Y.Z (YYYY-MM-DD), и каждая версия имеет групповое описание (например, основные исправления/улучшения функций).

Шаг 4: После обновления выполните минимальную проверку ​

Зачем Первая вещь после обновления не "запускать сложные сценарии", а сначала подтвердить, что прокси может нормально запуститься и может быть проверен клиентом.

По процессу Запуск локального reverse proxy и подключение первого клиента (/healthz + конфигурация SDK), минимум один раз проверьте GET /healthz.

Вы должны увидеть: /healthz возвращает успех (используется для подтверждения доступности службы).

Как быстро определить "повлияет ли это обновление на меня"

Приоритетно ищите эти два типа предложений:

  • Напоминания пользователям/вам нужно изменить: например, явно указано, что определенная модель по умолчанию удалена, требуется вручную настроить пользовательское сопоставление
  • Основные исправления/исправления совместимости: например, 0 Token, 429, мерцание Windows, сбой запуска и другие "из-за которых вы не можете использовать" проблемы
