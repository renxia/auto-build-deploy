Расширенные функции ​

Эта глава поможет вам углубленно освоить расширенные возможности плагина Antigravity Auth, включая балансировку нагрузки между несколькими аккаунтами, интеллектуальный выбор аккаунтов, обработку ограничений скорости, восстановление сессий, преобразование запросов и другие ключевые механизмы. Независимо от того, оптимизируете ли вы использование квот или разбираетесь со сложными проблемами, здесь вы найдете все необходимые ответы.

Предварительные условия ​

Перед началом убедитесь, что

  • ✅ Завершили Быструю установку и успешно добавили первый аккаунт
  • ✅ Завершили Первую аутентификацию и понимаете процесс OAuth
  • ✅ Завершили Первый запрос и проверили, что плагин работает корректно

Путь обучения ​

1. Настройка нескольких аккаунтов ​

Настройте несколько аккаунтов Google для объединения квот и балансировки нагрузки.

  • Добавьте несколько аккаунтов для увеличения общего лимита квот
  • Поймите двойную систему квот (Antigravity + Gemini CLI)
  • Выберите подходящее количество аккаунтов в зависимости от сценария использования

2. Стратегии выбора аккаунтов ​

Освойте лучшие практики использования трех стратегий выбора аккаунтов: sticky, round-robin и hybrid.

  • 1 аккаунт → стратегия sticky для сохранения кэша промптов
  • 2-3 аккаунта → стратегия hybrid для интеллектуального распределения запросов
  • 4+ аккаунтов → стратегия round-robin для максимальной пропускной способности

3. Обработка ограничений скорости ​

Поймите механизмы обнаружения ограничений скорости, автоматических повторных попыток и переключения аккаунтов.

  • Различайте 5 различных типов ошибок 429
  • Поймите алгоритм экспоненциального отката при автоматических повторных попытках
  • Освойте логику автоматического переключения аккаунтов в сценариях с несколькими аккаунтами

4. Восстановление сессий ​

Узнайте о механизмах восстановления сессий для автоматической обработки сбоев вызовов инструментов и прерываний.

  • Автоматическая обработка ошибок tool_result_missing
  • Исправление проблем thinking_block_order
  • Настройка параметров auto_resume и session_recovery

5. Механизм преобразования запросов ​

Углубленно поймите механизм преобразования запросов и то, как он обеспечивает совместимость с различными протоколами AI-моделей.

  • Поймите различия в протоколах моделей Claude и Gemini
  • Разбирайте ошибки 400, вызванные несовместимостью схем
  • Оптимизируйте конфигурацию Thinking для достижения наилучшей производительности

6. Руководство по конфигурации ​

Освойте все параметры конфигурации для настройки поведения плагина по требованию.

  • Расположение и приоритет конфигурационных файлов
  • Настройки поведения моделей, ротации аккаунтов и поведения приложений
  • Рекомендуемые конфигурации для сценариев с одним аккаунтом, несколькими аккаунтами и параллельными агентами

7. Оптимизация параллельных агентов ​

Оптимизируйте распределение аккаунтов для сценариев с параллельными агентами и включите смещение PID.

  • Поймите проблемы конфликтов аккаунтов в сценариях с параллельными агентами
  • Включите смещение PID для приоритизации различных аккаунтов в разных процессах
  • Комбинируйте со стратегией round-robin для максимизации использования нескольких аккаунтов

8. Отладочные журналы ​

Включите отладочные журналы для разбора проблем и мониторинга состояния работы.

  • Включите отладочные журналы для записи подробной информации
  • Поймите различные уровни журналов и сценарии их применения
  • Интерпретируйте содержимое журналов для быстрого определения проблем
