Часто Задаваемые Вопросы ​

В этом разделе собраны наиболее часто встречающиеся проблемы при использовании плагина Antigravity Auth и их решения. Здесь вы найдете руководства по диагностике — от сбоев OAuth аутентификации до ошибок запросов к моделям и проблем совместимости плагинов.

Предварительные Условия ​

Перед началом убедитесь

  • ✅ Быстрая установка выполнена и учетная запись успешно добавлена
  • ✅ Первая аутентификация выполнена и вы понимаете процесс OAuth

Путь Обучения ​

Выберите соответствующее руководство по диагностике в зависимости от типа возникшей проблемы:

1. Диагностика Сбоев OAuth Аутентификации ​

Решение распространенных проблем с OAuth аутентификацией, обновлением токенов и учетными записями.

  • Авторизация в браузере успешна, но терминал показывает "сбой авторизации"
  • Внезапная ошибка "Permission Denied" или "invalid_grant"
  • Сбой OAuth callback в браузере Safari
  • Невозможность завершить аутентификацию в средах WSL2/Docker

2. Миграция Учетных Записей ​

Миграция учетных записей между разными машинами и обработка обновлений версий.

  • Перенос учетной записи со старого компьютера на новый
  • Понимание изменений форматов хранения (v1/v2/v3)
  • Исправление ошибки invalid_grant после миграции

3. Диагностика "Модель Не Найдена" ​

Решение проблем, связанных с моделями — отсутствие моделей, ошибки 400 и т.д.

  • Диагностика ошибки Model not found
  • Исправление ошибки 400 Invalid JSON payload received. Unknown name "parameters"
  • Исправление ошибок вызовов сервера MCP

4. Совместимость Плагинов ​

Решение проблем совместимости с плагинами oh-my-opencode, DCP и другими.

  • Правильная настройка порядка загрузки плагинов
  • Отключение конфликтующих методов аутентификации в oh-my-opencode
  • Включение смещения PID для сценариев параллельных агентов

5. Предупреждения об ToS ​

Понимание рисков использования и предотвращение блокировки учетной записи.

  • Ознакомление с ограничениями Terms of Service Google
  • Идентификация высокорискованных сценариев (новые учетные записи, интенсивные запросы)
  • Освоение лучших практик предотвращения блокировки учетных записей
