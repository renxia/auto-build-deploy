Содержание

Функции платформы ​

В этой главе представлены две ключевые функции Plannotator: обзор планов и обзор кода. Вы научитесь визуально просматривать планы, созданные ИИ, добавлять различные типы аннотаций, прикреплять изображения с разметкой и проверять изменения кода через git diff.

Предварительные требования ​

Перед началом убедитесь

Прежде чем приступить к изучению этой главы, убедитесь, что вы выполнили следующие подготовительные шаги:

  • ✅ Завершили обучение по быстрому старту
  • ✅ Установили плагин Plannotator (Claude Code или OpenCode)

Содержание главы ​

Обзор планов ​

Узнайте, как проверять планы выполнения, созданные ИИ, добавлять предложения по изменениям и заставить ИИ действовать в соответствии с вашими намерениями.

Учебный материалОписание
Основы обзора плановНаучитесь визуально просматривать планы, созданные ИИ, включая утверждение или отклонение планов
Добавление аннотаций к планамОсвойте добавление различных типов аннотаций в планы (удаление, замена, вставка, комментарий)
Добавление разметки изображенийНаучитесь прикреплять изображения к обзору планов и использовать инструменты рисования, стрелки и круги для разметки

Обзор кода ​

Узнайте, как проверять изменения кода, добавлять построчные комментарии и находить проблемы перед фиксацией.

Учебный материалОписание
Добавление комментариев к кодуОсвойте добавление построчных комментариев при обзоре кода (comment/suggestion/concern)
Переключение типов diffНаучитесь переключаться между различными типами diff (uncommitted/staged/last commit/branch) в обзоре кода

Путь обучения ​

Рекомендуемый порядок обучения

В зависимости от вашего сценария использования, выберите подходящий путь обучения:

Путь A: Приоритет обзора планов (рекомендуется для начинающих)

  1. Основы обзора планов → Сначала изучите базовый процесс обзора планов
  2. Добавление аннотаций к планам → Научитесь точно изменять планы
  3. Добавление разметки изображений → Используйте изображения для более четкого выражения намерений
  4. Затем изучите серию обзора кода

Путь B: Приоритет обзора кода (для разработчиков с опытом code review)

  1. Основы обзора кода → Ознакомьтесь с интерфейсом обзора кода
  2. Добавление комментариев к коду → Изучите построчные комментарии
  3. Переключение типов diff → Освойте различные типы diff
  4. Затем изучите серию обзора планов

Следующие шаги ​

После завершения этой главы вы можете продолжить изучение:

  • Обмен по URL — Делитесь планами и аннотациями через URL для командной работы
  • Интеграция с Obsidian — Автоматически сохраняйте утвержденные планы в Obsidian
  • Удаленный режим — Используйте Plannotator в средах SSH, devcontainer, WSL
