Часто задаваемые вопросы и устранение неполадок ​

Этот раздел поможет вам решить распространённые проблемы, возникающие при использовании oh-my-opencode. От диагностики конфигурации до советов по использованию и рекомендаций по безопасности — здесь вы сможете быстро найти проблему и получить решение.

Путь обучения ​

Изучайте материалы в следующем порядке, чтобы постепенно освоить методы устранения неполадок и лучшие практики:

1. Диагностика конфигурации и устранение неполадок ​

Научитесь использовать команду Doctor для быстрой диагностики и решения проблем с конфигурацией.

  • Выполните команду Doctor для полной проверки работоспособности
  • Интерпретируйте результаты более чем 17 проверок (установка, конфигурация, аутентификация, зависимости, инструменты, обновления)
  • Выявляйте и устраняйте распространённые проблемы с конфигурацией
  • Используйте подробный режим и вывод JSON для углублённой диагностики

2. Часто задаваемые вопросы ​

Найдите и решите распространённые проблемы при использовании.

  • Быстрые ответы на вопросы об установке и конфигурации
  • Советы по использованию и лучшие практики (ultrawork, вызовы через прокси, фоновые задачи)
  • Информация о совместимости с Claude Code
  • Предупреждения о безопасности и рекомендации по оптимизации производительности
  • Ресурсы для участия в проекте и получения помощи

Предварительные требования ​

Перед началом изучения этого раздела убедитесь, что:

  • Выполнена быстрая установка и настройка
  • Вы знакомы с базовой структурой файлов конфигурации oh-my-opencode
  • У вас возникла конкретная проблема или вы хотите изучить лучшие практики

Рекомендуемое время для чтения

Рекомендуется после завершения базовой установки сначала прочитать раздел часто задаваемых вопросов (FAQ), чтобы узнать о распространённых ошибках и лучших практиках. При возникновении конкретных проблем используйте инструменты устранения неполадок для диагностики.

Быстрое руководство по устранению неполадок ​

Если у вас возникла срочная проблема, выполните следующие шаги для быстрого устранения:

bash
# Шаг 1: выполните полную диагностику
bunx oh-my-opencode doctor

# Шаг 2: просмотрите подробную информацию об ошибках
bunx oh-my-opencode doctor --verbose

# Шаг 3: проверьте определённую категорию (например, аутентификацию)
bunx oh-my-opencode doctor --category authentication

# Шаг 4: если проблему не удалось решить, ознакомьтесь с часто задаваемыми вопросами
# или обратитесь за помощью в GitHub Issues

Следующие шаги ​

После завершения этого раздела вы можете продолжить изучение:

  • Совместимость с Claude Code — узнайте о полной поддержке совместимости с Claude Code
  • Справочник по конфигурации — просмотрите полную схему файлов конфигурации и описание полей
  • Встроенные MCP-серверы — научитесь использовать встроенные MCP-серверы
