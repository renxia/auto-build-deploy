Журнал обновлений ​

На этой странице записана история изменений версий OpenSkills, помогающая вам узнать о новых возможностях, улучшениях и исправлениях каждой версии.

[1.5.0] - 2026-01-17 ​

Новые возможности ​

  • openskills update - Обновление установленных навыков из записанных источников (по умолчанию: обновление всех)
  • Отслеживание метаданных источника - При установке теперь записывается информация об источнике для надежного обновления навыков

Улучшения ​

  • Чтение нескольких навыков - Команда openskills read теперь поддерживает список имен навыков, разделенных запятыми
  • Генерация инструкций по использованию - Оптимизированы подсказки для вызова команды read в среде shell
  • README - Добавлены инструкции по обновлению и советы по ручному использованию

Исправления ​

  • Оптимизация процесса обновления - Пропуск навыков без метаданных источника с выводом списка таких навыков с предложением переустановить

[1.4.0] - 2026-01-17 ​

Улучшения ​

  • README - Уточнены способы установки по умолчанию для локального проекта, удалены избыточные подсказки sync
  • Сообщения об установке - Инсталлятор теперь четко различает установку по умолчанию для локального проекта и опцию --global

[1.3.2] - 2026-01-17 ​

Улучшения ​

  • Документация и руководства AGENTS.md - Все примеры команд и генерируемые инструкции по использованию теперь единообразно используют npx openskills

[1.3.1] - 2026-01-17 ​

Исправления ​

  • Установка в Windows - Исправлена проблема проверки пути в системах Windows ("Security error: Installation path outside target directory")
  • Версия CLI - npx openskills --version теперь правильно считывает номер версии из package.json
  • SKILL.md в корневом каталоге - Исправлена установка репозиториев с одним навыком, где SKILL.md находится в корне репозитория

[1.3.0] - 2025-12-14 ​

Новые возможности ​

  • Поддержка символических ссылок - Навыки теперь можно устанавливать в директорию навыков через символические ссылки (#3)

    • Поддержка обновлений навыков на основе git через создание символических ссылок из клонированного репозитория
    • Поддержка рабочих процессов локальной разработки навыков
    • Поврежденные символические ссылки корректно пропускаются

  • Настраиваемый путь вывода - Команда sync получила новую опцию --output / -o (#5)

    • Можно синхронизировать в любой файл .md (например, .ruler/AGENTS.md)
    • Если файл не существует, автоматически создается файл и добавляется заголовок
    • При необходимости автоматически создаются вложенные директории

  • Установка по локальному пути - Поддержка установки навыков из локальных директорий (#10)

    • Поддержка абсолютных путей (/path/to/skill)
    • Поддержка относительных путей (./skill, ../skill)
    • Поддержка раскрытия тильды (~/my-skills/skill)

  • Поддержка частных git репозиториев - Поддержка установки навыков из частных репозиториев (#10)

    • SSH URLs (git@github.com:org/private-skills.git)
    • HTTPS URLs с аутентификацией
    • Автоматическое использование системных SSH ключей

  • Полный набор тестов - 88 тестов в 6 тестовых файлах

    • Модульные тесты для обнаружения символических ссылок, парсинга YAML
    • Интеграционные тесты для команд install, sync
    • Энд-энд тесты для полного рабочего процесса CLI

Улучшения ​

  • Флаг --yes теперь пропускает все подсказки - Полностью неинтерактивный режим, подходит для CI/CD (#6)

    • Без подсказок при перезаписи существующих навыков
    • Отображение сообщения Overwriting: <skill-name> при пропуске подсказок
    • Все команды теперь можно запускать в headless-средах

  • Перестановка CI workflow - Этап сборки теперь запускается перед тестами

    • Обеспечивает существование dist/cli.js для энд-энд тестов

Безопасность ​

  • Защита от обхода пути - Проверка, что путь установки остается в пределах целевой директории
  • Разыменование символических ссылок - cpSync использует dereference: true для безопасного копирования целевого объекта символической ссылки
  • Нежадные регулярные выражения YAML - Защита от потенциальной атаки ReDoS при парсинге frontmatter

[1.2.1] - 2025-10-27 ​

Исправления ​

  • Очистка документации README - удалены дублирующиеся разделы и неверные флаги

[1.2.0] - 2025-10-27 ​

Новые возможности ​

  • Флаг --universal для установки навыков в .agent/skills/ вместо .claude/skills/
    • Подходит для многоагентных сред (Claude Code + Cursor/Windsurf/Aider)
    • Избегает конфликтов с плагинами нативного маркетплейса Claude Code

Улучшения ​

  • Локальная установка в проект теперь является опцией по умолчанию (ранее глобальная установка)
  • Навыки по умолчанию устанавливаются в ./.claude/skills/

[1.1.0] - 2025-10-27 ​

Новые возможности ​

  • Полная одностраничная документация README с глубоким техническим анализом
  • Параллельное сравнение с Claude Code

Исправления ​

  • Метки расположения теперь корректно отображают project или global в зависимости от места установки

[1.0.0] - 2025-10-26 ​

Новые возможности ​

  • Первоначальный выпуск
  • npx openskills install <source> - Установка навыков из репозиториев GitHub
  • npx openskills sync - Генерация XML <available_skills> для AGENTS.md
  • npx openskills list - Отображение установленных навыков
  • npx openskills read <name> - Загрузка содержимого навыка для агента
  • npx openskills manage - Интерактивное удаление навыков
  • npx openskills remove <name> - Удаление указанного навыка
  • Интерфейс TUI для всех команд
  • Поддержка формата SKILL.md от Anthropic
  • Прогрессивное раскрытие (загрузка навыков по требованию)
  • Поддержка упакованных ресурсов (references/, scripts/, assets/)
