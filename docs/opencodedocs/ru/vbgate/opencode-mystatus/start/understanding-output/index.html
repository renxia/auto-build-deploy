Содержание

Интерпретация вывода: индикатор прогресса, время сброса и несколько аккаунтов ​

Чему вы научитесь ​

  • Понимать каждую информацию в выводе mystatus
  • Понимать значение отображения индикатора прогресса (сплошные vs пустые блоки)
  • Знать циклы лимитов разных платформ (3 часа, 5 часов, ежемесячно)
  • Распознавать различия в квотах нескольких аккаунтов

Ваша текущая проблема ​

Вы выполнили /mystatus, видите кучу индикаторов прогресса, процентов, обратных отсчётов, но не понимаете:

  • Индикатор прогресса лучше заполненный или пустой?
  • Что значит "Resets in: 2h 30m"?
  • Почему у некоторых платформ отображаются два индикатора, а у других только один?
  • Почему у Google Cloud несколько аккаунтов?

В этом уроке мы по порядку разберём эту информацию.

Основная идея ​

Вывод mystatus имеет унифицированный формат, но разные платформы имеют различия:

Унифицированные элементы:

  • Индикатор прогресса: █ (сплошной) означает остаток, ░ (пустой) означает использовано
  • Процент: рассчитан на основе использованного количества
  • Время сброса: обратный отсчёт до следующего обновления квоты

Различия платформ:

ПлатформаЦикл лимитаОсобенности
OpenAI3 часа / 24 часаМожет отображать два окна
Zhipu AI / Z.ai5 часов Token / ежемесячная квота MCPДва разных типа лимитов
GitHub CopilotЕжемесячноОтображает конкретные значения (229/300)
Google CloudПо моделямКаждый аккаунт отображает 4 модели

Значение каждой части ​

1. Строка информации об аккаунте ​

Account:        user@example.com (team)
  • OpenAI / Copilot: отображает email + тип подписки
  • Zhipu AI / Z.ai: отображает замаскированный API Key + тип аккаунта (Coding Plan)
  • Google Cloud: отображает email, несколько аккаунтов разделены ###

2. Индикатор прогресса ​

██████████████████████████ 85% remaining
  • █ (сплошной блок): остаток квоты
  • ░ (пустой блок): использовано квоты
  • Процент: процент остатка (чем больше, тем лучше)

Мнемоника

Сплошные блоки заполнены, остаток больше → можно продолжать использовать Пустые блоки заполнены, использовано больше → нужно экономить

3. Обратный отсчёт времени сброса ​

Resets in: 2h 30m

Указывает время до следующего обновления квоты.

Цикл сброса:

  • OpenAI: окно 3 часов / окно 24 часов
  • Zhipu AI / Z.ai: лимит Token за 5 часов / ежемесячная квота MCP
  • GitHub Copilot: ежемесячно (отображает конкретную дату)
  • Google Cloud: у каждой модели независимое время сброса

4. Детализация значений (часть платформ) ​

Zhipu AI и Copilot отображают конкретные значения:

Used: 0.5M / 10.0M              # Zhipu AI: использовано / всего (единица: миллион Token)
Premium        24% (229/300)     # Copilot: процент остатка (использовано / общая квота)

Подробности различий платформ ​

OpenAI: двойное окно лимитов ​

OpenAI может отображать два индикатора прогресса:

3-hour limit
██████████████████████████ 85% remaining
Resets in: 2h 30m

24-hour limit
████████████████████████████ 90% remaining
Resets in: 20h 45m
  • 3-hour limit: скользящее окно 3 часов, подходит для частого использования
  • 24-hour limit: скользящее окно 24 часов, подходит для долгосрочного планирования

Командные аккаунты (Team):

  • Имеют два лимита: главное окно и дополнительное окно
  • Разные участники используют одну команду Team

Персональные аккаунты (Plus):

  • Обычно отображают только 3-часовое окно

Zhipu AI / Z.ai: два типа лимитов ​

5-hour token limit
██████████████████████████ 95% remaining
Used: 0.5M / 10.0M
Resets in: 4h

MCP limit
██████████████████████████ 100% remaining
Used: 0 / 1000
  • 5-hour token limit: лимит использования Token за 5 часов
  • MCP limit: ежемесячная квота MCP (Model Context Protocol), используется для функции поиска

WARNING

Квота MCP — ежемесячная, время сброса длиннее. Если отображается заполненной, нужно ждать до следующего месяца для восстановления.

Предупреждения о типичных ошибках ​

Частые недопонимания ​

НедопониманиеФакты
Индикатор полностью сплошной = не использованоСплошных блоков много = остаток много, можно смело использовать
Короткое время сброса = скоро закончится квотаКороткое время сброса = скоро сбросится, можно продолжить использовать
Процент 100% = всё использованоПроцент 100% = всё осталось
Zhipu AI отображает только один лимитФактически есть два типа: TOKENS_LIMIT и TIME_LIMIT 两种

Что делать, если лимит исчерпан? ​

Если индикатор прогресса полностью пустой (0% remaining):

  1. Краткосрочный лимит (например, 3 часа, 5 часов): подождите окончания обратного отсчёта
  2. Ежемесячный лимит (например, Copilot, MCP): подождите начала следующего месяца
  3. Несколько аккаунтов: переключитесь на другие аккаунты (Google Cloud поддерживает несколько аккаунтов)

INFO

mystatus — это инструмент только для чтения, не потребляет вашу квоту, не вызывает никаких API запросов.

Итог урока ​

  • Индикатор прогресса: сплошной █ = остаток, пустой ░ = использовано
  • Время сброса: обратный отсчёт до следующего обновления квоты
  • Различия платформ: у разных платформ разные циклы лимитов (3h/5h/ежемесячно)
  • Несколько аккаунтов: Google Cloud отображает несколько аккаунтов, удобно управлять квотами

Ключевые функции:

  • createProgressBar(percent, width): генерирует индикатор прогресса, сплошные блоки означают остаток
  • formatDuration(seconds): преобразует секунды в понятный формат времени (например, "2h 30m")
