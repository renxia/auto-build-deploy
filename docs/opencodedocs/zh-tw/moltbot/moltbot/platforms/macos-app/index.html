macOS 應用程式：選單列控制與語音互動 ​

學完你能做什麼 ​

完成本教學後，你將能夠：

  • ✅ 了解 Clawdbot macOS 應用程式作為選單列控制平面的核心功能
  • ✅ 掌握 Voice Wake 語音喚醒和 Talk Mode 連續對話的使用
  • ✅ 理解節點模式下的 system.run、Canvas、Camera 等能力
  • ✅ 設定本機 vs 遠端模式以適應不同部署場景
  • ✅ 管理 Exec Approvals 審批機制以控制指令執行權限
  • ✅ 使用深度鏈結快速觸發 AI 助手
  • ✅ 透過 SSH/Tailscale 遠端存取和控制 Gateway

你現在的困境 ​

你可能正在想：

  • "macOS 應用程式到底是做什麼的？是 Gateway 本身嗎？"
  • "Voice Wake 和 Talk Mode 怎麼用？需要額外硬體嗎？"
  • "節點模式和普通模式有什麼區別？什麼時候用哪個？"
  • "如何在 macOS 上管理權限和安全設定？"
  • "我可以在另一台機器上執行 Gateway 嗎？"

好消息是：Clawdbot macOS 應用程式是 Gateway 的圖形化控制平面，它不執行 Gateway 服務，而是連接、管理和監控。同時，它也作為節點暴露 macOS 特有功能（如 system.run、Canvas、Camera）給遠端 Gateway。

什麼時候用這一招 ​

當你需要：

  • 🖥️ macOS 圖形化管理 —— 選單列狀態和控制，比指令列更直觀
  • 🎙️ 語音互動 —— Voice Wake 喚醒 + Talk Mode 連續對話
  • 💻 本機指令執行 —— 在 macOS 節點上執行 system.run 等指令
  • 🎨 Canvas 視覺化 —— 在 macOS 上渲染 AI 驅動的視覺化介面
  • 📷 裝置功能 —— Camera 拍照、錄影和螢幕錄製
  • 🌐 遠端存取 —— 透過 SSH/Tailscale 控制遠端 Gateway

節點 vs Gateway 的區別

  • Gateway：執行 AI 模型、管理會話、處理訊息（可以在任何機器上執行）
  • 節點（Node）：暴露裝置本機功能（Canvas、Camera、system.run）給 Gateway
  • macOS 應用程式：既可以是 Gateway 客戶端，也可以作為節點使用

核心思路 ​

Clawdbot macOS 應用程式是一個雙重角色的系統：

┌──────────────────────────────────────────┐
│     Clawdbot.app (macOS 應用程式)       │
│                                      │
│   ┌────────────────────────────┐      │
│   │  選單列控制平面          │      │
│   │  • Gateway 連接管理         │◄────► Gateway WebSocket
│   │  • WebChat 內嵌視窗       │      │
│   │  • 設定和配置             │      │
│   │  • Voice Wake/Talk Mode    │      │
│   └────────────────────────────┘      │
│                                      │
│   ┌────────────────────────────┐      │
│   │  節點服務              │      │
│   │  • system.run              │◄────► Gateway 節點協定
│   │  • Canvas                 │      │
│   │  • Camera/Screen          │      │
│   └────────────────────────────┘      │
└──────────────────────────────────────────┘

兩種執行模式：

模式Gateway 位置節點服務適用場景
本機模式（預設）本機（launchd 守護程序）不啟動Gateway 在這台 Mac 上執行
遠端模式遠端機器（透過 SSH/Tailscale）啟動Gateway 在其他機器上執行

核心功能模組：

  1. 選單列控制 —— Gateway 連接狀態、WebChat、設定、會話管理
  2. Voice Wake —— 全域語音喚醒詞監聽
  3. Talk Mode —— 連續語音對話迴圈（語音輸入 → AI 回覆 → TTS 播放）
  4. 節點模式 —— 暴露 macOS 特有指令（system.run、canvas.*、camera.*）
  5. Exec Approvals —— system.run 指令執行審批和安全控制
  6. 深度鏈結 —— clawdbot:// 協定快速觸發功能

跟我做 ​

第 1 步：安裝和啟動 macOS 應用程式 ​

為什麼 你需要安裝 Clawdbot macOS 應用程式來獲得選單列控制和語音功能。

安裝方式：

bash
brew install --cask clawdbot
bash
# 從 https://github.com/moltbot/moltbot/releases 下載最新的 Clawdbot.app.dmg
# 拖曳到 Applications 資料夾

首次啟動：

bash
open /Applications/Clawdbot.app

你應該看到：

  • macOS 頂部選單列出現 🦞 圖示
  • 點擊圖示展開下拉選單
  • 系統彈出 TCC 權限請求對話框

第一次啟動的權限請求

macOS 應用程式需要以下權限（系統會自動彈出提示）：

  • 通知權限 —— 顯示系統通知
  • 輔助功能權限 —— 用於 Voice Wake 和系統操作
  • 麥克風權限 —— Voice Wake 和 Talk Mode 需要
  • 螢幕錄製權限 —— Canvas 和螢幕錄製功能
  • 語音辨識權限 —— Voice Wake 的語音輸入
  • 自動化權限 —— AppleScript 控制（如需要）

所有這些權限都是完全本機使用，不會上傳到任何伺服器。

第 2 步：設定連接模式（本機 vs 遠端） ​

為什麼 根據你的部署需求選擇本機或遠端模式。

模式 A：本機模式（預設） ​

適用場景：Gateway 和 macOS 應用程式在同一台機器上執行。

設定步驟：

  1. 確保選單列應用程式中顯示 Local 模式
  2. 如果 Gateway 未執行，應用程式會自動啟動 com.clawdbot.gateway launchd 服務
  3. 應用程式會連接到 ws://127.0.0.1:18789

你應該看到：

  • 選單列圖示顯示綠色（已連接狀態）
  • Gateway 狀態卡片顯示"Local"
  • 節點服務未啟動（節點模式在遠端模式下才需要）

模式 B：遠端模式 ​

適用場景：Gateway 在另一台機器（如伺服器或 Linux VPS）上執行，你想透過 Mac 控制它。

設定步驟：

  1. 在選單列應用程式中切換到 Remote 模式
  2. 輸入遠端 Gateway 的 WebSocket 位址（如 ws://your-server:18789）
  3. 選擇認證方式（Token 或 Password）
  4. 應用程式會自動建立 SSH 隧道連接遠端 Gateway

你應該看到：

  • 選單列圖示顯示連接狀態（黃色/綠色/紅色）
  • Gateway 狀態卡片顯示遠端伺服器位址
  • 節點服務自動啟動（以便遠端 Gateway 呼叫本機功能）

遠端模式的隧道機制：

macOS 應用程式                     遠端 Gateway
    │                                  │
    ├── SSH 隧道 ───────────────────► ws://remote:18789
    │                                  │
    └── 節點服務 ◄───────────────────── node.invoke

遠端模式的優勢

  • 集中管理：在一台強大的機器上執行 Gateway，多台客戶端存取
  • 資源最佳化：Mac 可以保持輕量，Gateway 執行在高效能伺服器上
  • 裝置本地化：Canvas、Camera 等功能仍在 Mac 本機執行

第 3 步：使用選單列控制平面 ​

為什麼 選單列應用程式提供了快速存取所有核心功能的介面。

核心選單項：

點擊選單列圖示後，你會看到：

  1. 狀態卡片

    • Gateway 連接狀態（已連接/斷開/連接中）
    • 目前模式（Local/Remote）
    • 執行的頻道列表（WhatsApp、Telegram 等）

  2. 快速操作

    • Agent —— 開啟 AI 對話視窗（呼叫 Gateway）
    • WebChat —— 開啟內嵌 WebChat 介面
    • Canvas —— 開啟 Canvas 視覺化視窗
    • Settings —— 開啟設定介面

  3. 功能開關

    • Talk —— 啟用/停用 Talk Mode
    • Voice Wake —— 啟用/停用 Voice Wake

  4. 資訊選單

    • Usage —— 查看使用統計和成本
    • Sessions —— 管理會話列表
    • Channels —— 查看頻道狀態
    • Skills —— 管理技能包

你應該看到：

  • 即時更新的狀態指示器（綠色 = 正常，紅色 = 斷開）
  • 滑鼠游標懸停顯示詳細的連接資訊
  • 點擊任何選單項都能快速開啟對應功能

第 4 步：設定和使用 Voice Wake ​

為什麼 Voice Wake 允許你透過語音喚醒詞觸發 AI 助手，無需點擊或輸入。

Voice Wake 運作原理：

┌──────────────────────────────────┐
│   Voice Wake 運行時          │
│                              │
│   監聽麥克風 ──► 檢測喚醒詞  │
│                              │
│   喚醒詞匹配？                   │
│       │                       │
│       ├─ 是 ──► 觸發 Agent  │
│       │                       │
│       └─ 否 ──► 繼續監聽  │
└──────────────────────────────────┘

設定 Voice Wake：

  1. 開啟 Settings → Voice Wake
  2. 輸入喚醒詞（預設：clawd、claude、computer）
  3. 可以新增多個喚醒詞（用逗號分隔）
  4. 啟用 Enable Voice Wake 開關

喚醒詞規則：

  • 喚醒詞儲存在 Gateway：~/.clawdbot/settings/voicewake.json
  • 所有節點共享同一份全域喚醒詞列表
  • 更改後廣播到所有連接的裝置（macOS、iOS、Android）

使用流程：

  1. 確保麥克風權限已授予
  2. 在選單列啟用 Voice Wake
  3. 對著麥克風說喚醒詞（如"Hey clawd"）
  4. 等待聽到"叮"提示音（表示喚醒成功）
  5. 說出你的指令或問題

你應該看到：

  • 螢幕中央出現 Voice Wake 叠加層
  • 顯示麥克風音量波形
  • 顯示"Listening"狀態文字
  • AI 開始處理你的語音輸入

Voice Wake 的全域特性

喚醒詞是 Gateway 層級的全域設定，不限於單個裝置。這意味著：

  • 在 macOS 上修改喚醒詞後，iOS 和 Android 裝置也會同步
  • 所有裝置都使用同一組喚醒詞
  • 但每個裝置可以單獨啟用/停用 Voice Wake（基於權限和使用者偏好）

第 5 步：使用 Talk Mode 連續對話 ​

為什麼 Talk Mode 提供了類似 Siri/Alexa 的連續語音對話體驗，無需每次都喚醒。

Talk Mode 運作迴圈：

監聽 ──► AI 處理 ──► TTS 播放 ──► 監聽
  │                                              │
  └────────────────────────────────────────┘

啟用 Talk Mode：

  1. 在選單列點擊 Talk 按鈕
  2. 或者使用快速鍵（預設：無，可以在 Settings 中設定）
  3. Talk Mode 叠加層出現

Talk Mode 介面狀態：

狀態顯示說明
Listening雲朵脈衝動畫 + 麥克風音量等待你說話
Thinking下沉動畫AI 正在思考
Speaking輻射環動畫 + 漣漪AI 正在回覆（TTS 播放中）

互動控制：

  • 停止說話：點擊雲朵圖示停止 TTS 播放
  • 退出 Talk Mode：點擊右上角 X 按鈕
  • 語音中斷：AI 說話時如果你開始說話，自動停止播放

設定 TTS：

Talk Mode 使用 ElevenLabs 進行文字轉語音。設定位置：~/.clawdbot/clawdbot.json

yaml
talk:
  voiceId: "elevenlabs_voice_id"  # ElevenLabs 語音 ID
  modelId: "eleven_v3"            # 模型版本
  apiKey: "elevenlabs_api_key"     # API 金鑰（或使用環境變數）
  interruptOnSpeech: true           # 語音時中斷
  outputFormat: "mp3_44100_128"   # 輸出格式

ElevenLabs 設定

如果沒有設定 API 金鑰，Talk Mode 會嘗試使用：

  1. ELEVENLABS_API_KEY 環境變數
  2. Gateway shell profile 中的金鑰
  3. 使用第一個可用的 ElevenLabs 語音作為預設

第 6 步：使用節點模式 ​

為什麼 節點模式允許 macOS 應用程式暴露本機能力給遠端 Gateway，實現真正的跨裝置協作。

節點模式下的可用指令：

指令類別指令範例功能說明
Canvascanvas.present、canvas.navigate、canvas.eval在 macOS 上渲染視覺化介面
Cameracamera.snap、camera.clip拍照或錄影
Screenscreen.record螢幕錄製
Systemsystem.run、system.notify執行 Shell 指令或發送通知

啟用節點模式：

節點模式在 遠端模式下自動啟動，因為遠端 Gateway 需要呼叫本機功能。

你也可以手動啟動節點服務：

bash
clawdbot node run --display-name "My Mac"

節點權限管理：

macOS 應用程式透過權限系統回報哪些功能可用：

json
{
  "canvas": true,
  "camera": true,
  "screen": true,
  "system": {
    "run": true,
    "notify": true
  }
}

AI 會根據權限自動選擇可用的工具。

第 7 步：設定 Exec Approvals（system.run 安全控制） ​

為什麼system.run 可以執行任意 Shell 指令，因此需要審批機制防止誤操作或濫用。

Exec Approvals 三層安全模型：

json
{
  "version": 1,
  "defaults": {
    "security": "deny",          // 預設策略：拒絕
    "ask": "on-miss"           // 指令不在白名單時詢問
  },
  "agents": {
    "main": {
      "security": "allowlist",    // 主會話：僅允許白名單
      "ask": "on-miss",
      "allowlist": [
        { "pattern": "/usr/bin/git" },
        { "pattern": "/opt/homebrew/*/rg" }
      ]
    }
  }
}

安全策略類型：

策略行為適用場景
deny拒絕所有 system.run 呼叫高度安全，停用所有指令
allowlist僅允許白名單中的指令平衡安全性和便利性
ask不在白名單時提示使用者批准靈活但需要確認

審批流程：

當 AI 嘗試執行未授權的指令時：

  1. macOS 應用程式彈出審批對話框
  2. 顯示完整指令路徑和參數
  3. 提供三個選項：
    • Allow Once —— 僅本次允許
    • Always Allow —— 新增到白名單
    • Deny —— 拒絕執行

你應該看到：

  • 審批對話框顯示指令詳情（如 /usr/bin/ls -la ~）
  • 選擇"Always Allow"後，下次不會再次詢問
  • 選擇"Deny"後，指令執行失敗並回傳錯誤給 AI

設定位置：

Exec Approvals 儲存在 macOS 本機：

  • 檔案：~/.clawdbot/exec-approvals.json
  • 審批歷史：在應用程式中查看所有已批准/拒絕的指令

第 8 步：使用深度鏈結 ​

為什麼 深度鏈結提供了從其他應用程式快速觸發 Clawdbot 功能的能力。

支援的深度鏈結協定：clawdbot://

clawdbot://agent ​

觸發 Gateway agent 請求，相當於在終端機執行 clawdbot agent。

參數：

參數說明範例
message（必需）發送給 AI 的訊息message=Hello%20from%20deep%20link
sessionKey（可選）目標會話鍵，預設 mainsessionKey=main
thinking（可選）思考層級：off|minimal|low|medium|high|xhighthinking=high
deliver/to/channel（可選）送达頻道channel=telegram
timeoutSeconds（可選）逾時時間timeoutSeconds=30
key（可選）免確認金鑰，用於自動化key=your-secret-key

範例：

bash
# 基本：發送訊息
open 'clawdbot://agent?message=Hello%20from%20deep%20link'

# 進階：發送到 Telegram，高思考層級，30 秒逾時
open 'clawdbot://agent?message=Summarize%20my%20day&to=telegram&thinking=high&timeoutSeconds=30'

# 自動化：使用金鑰跳過確認（僅在你的腳本中安全使用）
open 'clawdbot://agent?message=Automated%20task&key=secure-random-string'

你應該看到：

  • Clawdbot macOS 應用程式自動開啟（如果未執行）
  • Agent 視窗出現並顯示訊息
  • AI 開始處理並回傳回應

深度鏈結的安全性

  • 沒有 key 參數時，應用程式會彈出確認對話框
  • 帶有有效 key 時，請求會靜默執行（用於自動化腳本）
  • 永遠不要在不可信來源中使用深度鏈結

踩坑提醒 ​

❌ 權限被拒絕或未授予 ​

問題：

  • Voice Wake 無法監聽麥克風
  • Canvas 無法顯示內容
  • system.run 指令執行失敗

解決方案：

  1. 開啟 系統設定 → 隱私與安全性
  2. 找到 Clawdbot 或 Clawdbot.app
  3. 確保 麥克風、輔助功能、螢幕錄製、自動化 等權限都已開啟
  4. 重新啟動 Clawdbot 應用程式

TCC 權限故障排除

如果權限開關無法啟用或立即關閉：

  • 檢查是否啟用了任何安全工具（如 Little Snitch）
  • 嘗試完全解除安裝應用程式後重新安裝
  • 查看 Console.app 中的 TCC 拒絕記錄

❌ Gateway 連接失敗 ​

問題：

  • 選單列圖示顯示紅色（斷開狀態）
  • 狀態卡片顯示"Connection Failed"
  • WebChat 無法開啟

可能原因和解決方案：

原因檢查方法解決方案
Gateway 未啟動執行 clawdbot gateway status啟動 Gateway 服務
位址錯誤檢查 WebSocket URL確認 ws://127.0.0.1:18789 或遠端位址正確
連接埠被佔用執行 lsof -i :18789關閉佔用連接埠的程序
認證失敗檢查 Token/Password確認認證憑證正確

❌ Talk Mode 無法使用 ​

問題：

  • 啟用 Talk Mode 後沒有反應
  • TTS 無法播放
  • 麥克風無法輸入

解決方案：

  1. 檢查 ElevenLabs 設定：

    • 確認 API 金鑰已設定
    • 測試金鑰是否有效：造訪 ElevenLabs 控制台

  2. 檢查網路連接：

    • TTS 需要網際網路連接
    • 檢查防火牆是否阻止 API 請求

  3. 檢查音訊輸出：

    • 確認系統音量已開啟
    • 檢查預設輸出裝置是否正確

❌ 遠端模式下節點無法連接 ​

問題：

  • 遠端 Gateway 無法呼叫 macOS 上的 system.run 等指令
  • 錯誤記錄顯示"Node not found"或"Node offline"

解決方案：

  1. 確認節點服務執行：

    bash
    clawdbot nodes list
# 應該看到 macOS 節點顯示為"paired"

  2. 檢查 SSH 隧道：

    • 在 macOS 應用程式設定中查看 SSH 連接狀態
    • 確認能手動 SSH 到遠端 Gateway

  3. 重新啟動節點服務：

    bash
    # 在 macOS 上
clawdbot node restart
