Funciones de Plataforma ​

Esta sección introduce las dos funciones principales de Plannotator: Revisión de Planes y Revisión de Código. Aprenderás a revisar visualmente los planes generados por IA, agregar varios tipos de anotaciones, adjuntar marcadores de imagen y revisar los cambios de código git diff.

Prerrequisitos ​

Confirma antes de comenzar

Antes de estudiar esta sección, asegúrate de haber completado los siguientes preparativos:

  • ✅ Completado el tutorial de Inicio Rápido
  • ✅ Instalado el plugin de Plannotator (Claude Code u OpenCode)

Contenido de Esta Sección ​

Revisión de Planes ​

Aprende a revisar los planes de ejecución generados por IA, agregar sugerencias de modificación y hacer que la IA ejecute según tus intenciones.

TutorialDescripción
Fundamentos de Revisión de PlanesAprende a usar Plannotator para revisar visualmente los planes generados por IA, incluyendo aprobar o rechazar planes
Agregar Anotaciones al PlanDomina cómo agregar diferentes tipos de anotaciones al plan (eliminar, reemplazar, insertar, comentar)
Agregar Marcadores de ImagenAprende a adjuntar imágenes en la revisión de planes y usar herramientas de pincel, flecha y círculo para marcar

Revisión de Código ​

Aprende a revisar cambios de código, agregar comentarios a nivel de línea y descubrir problemas antes de confirmar.

TutorialDescripción
Agregar Comentarios de CódigoDomina cómo agregar comentarios a nivel de línea en la revisión de código (comment/suggestion/concern)
Cambiar Vistas de DiffAprende a cambiar entre diferentes tipos de diff en la revisión de código (uncommitted/staged/last commit/branch)

Rutas de Aprendizaje ​

Orden de Aprendizaje Recomendado

Según tu caso de uso, elige la ruta de aprendizaje adecuada:

Ruta A: Revisión de Planes Primero (recomendado para principiantes)

  1. Fundamentos de Revisión de Planes → Aprende primero el flujo básico de revisión de planes
  2. Agregar Anotaciones al Plan → Aprende a modificar planes con precisión
  3. Agregar Marcadores de Imagen → Usa imágenes para expresar intenciones más claramente
  4. Luego aprende la serie de revisión de código

Ruta B: Revisión de Código Primero (adecuado para desarrolladores con experiencia en Code Review)

  1. Fundamentos de Revisión de Código → Familiarízate con la interfaz de revisión de código
  2. Agregar Comentarios de Código → Aprende comentarios a nivel de línea
  3. Cambiar Vistas de Diff → Domina diferentes tipos de diff
  4. Luego aprende la serie de revisión de planes

Siguientes Pasos ​

Después de completar esta sección, puedes continuar aprendiendo:

  • Compartir URL - Comparte planes y anotaciones a través de URL, logrando colaboración en equipo
  • Integración con Obsidian - Guarda automáticamente los planes aprobados en Obsidian
  • Modo Remoto - Usa Plannotator en entornos SSH, devcontainer, WSL
