Registro de Cambios ​

Este capítulo registra el historial de versiones, lanzamientos de nuevas funciones y correcciones de errores de Plannotator, ayudándote a comprender la evolución del proyecto y los últimos avances.

Contenido de Este Capítulo ​

📋 Historial de Versiones

Consulta el historial de versiones completo de Plannotator, incluyendo nuevas funciones, mejoras y correcciones de errores de cada versión. Aprende sobre la línea de tiempo de lanzamiento de funciones importantes como revisión de código, anotación de imágenes e integraciones con Obsidian/Bear.

Recomendaciones de Lectura ​

El registro de cambios es adecuado para los siguientes escenarios:

EscenarioRecomendación
Verificación Antes de ActualizarRevisa las nuevas funciones y cambios rupturistas de la versión objetivo
Exploración de FuncionesDescubre qué nuevas funciones se han añadido en la última versión
Solución de ProblemasConfirma si un error específico se ha corregido en la nueva versión
Compatibilidad de VersionesVerifica la relación de versiones entre Plannotator y Claude Code/OpenCode

Mantente Actualizado

Se recomienda revisar periódicamente el registro de cambios y actualizar a la última versión para obtener nuevas funciones y correcciones de seguridad.

Próximos Pasos ​

Después de leer el registro de cambios, puedes:

  • Volver a Inicio Rápido para aprender el uso básico
  • Consultar Preguntas Frecuentes para resolver problemas de uso
  • Leer la Referencia de la API para conocer todos los endpoints de la API
