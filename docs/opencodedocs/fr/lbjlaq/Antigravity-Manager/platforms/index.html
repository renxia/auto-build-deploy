Plateformes et intégrations ​

La capacité principale d'Antigravity Tools est de convertir les protocoles de plusieurs plateformes IA en une passerelle API locale unifiée. Ce chapitre détaille la méthode d'intégration, les limites de compatibilité et les meilleures pratiques pour chaque protocole.

Ce chapitre contient ​

TutorielDescription
API compatible OpenAIStratégies pour /v1/chat/completions et /v1/responses, intégration transparente du SDK OpenAI
API compatible AnthropicContrats clés pour /v1/messages et Claude Code, supportant les chaînes de pensée, prompts système et autres fonctionnalités principales
API native Gemini/v1beta/models et intégration des points de terminaison du SDK Google, support compatible avec x-goog-api-key
Génération d'images Imagen 3Mappage automatique des paramètres size/quality des Images OpenAI, supporte tous les ratios d'aspect
Transcription audioLimitations et gestion des gros volumes pour /v1/audio/transcriptions
Points de terminaison MCPExpose Web Search/Reader/Vision comme outils appelables
Tunnel CloudflaredExposez votre API locale sur Internet en un clic (pas sécurisé par défaut)

Recommandations de parcours d'apprentissage ​

Ordre recommandé

  1. Commencez par le protocole que vous utilisez : Si vous utilisez Claude Code, commencez par API compatible Anthropic ; si vous utilisez le SDK OpenAI, commencez par API compatible OpenAI
  2. Apprenez ensuite Gemini natif : Comprenez la méthode d'intégration directe du SDK Google
  3. Apprenez les fonctionnalités étendues selon vos besoins : Génération d'images, transcription audio, outils MCP
  4. Terminez par les tunnels : Consultez Tunnel Cloudflared seulement si vous avez besoin d'exposer sur Internet

Prérequis ​

Avant de commencer, vérifiez que

  • Installation et mise à niveau terminée
  • Ajout de comptes terminé
  • Démarrage du reverse proxy local terminé (au minimum capable d'accéder à /healthz)

Prochaines étapes ​

Après avoir terminé ce chapitre, vous pouvez continuer à apprendre :

  • Configuration avancée : Routage de modèles, gouvernance des quotas, programmation haute disponibilité et autres fonctionnalités avancées
  • Questions fréquentes : Guide de dépannage pour les erreurs 401/404/429, etc.
