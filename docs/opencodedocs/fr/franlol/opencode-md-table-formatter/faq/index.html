Questions fréquentes ​

Vue d'ensemble du chapitre ​

Ce chapitre couvre les questions fréquentes et les méthodes de dépannage du plugin opencode-md-table-formatter, vous aidant à identifier et résoudre rapidement divers problèmes rencontrés lors du formatage de tableaux.

Navigation des sous-pages ​

Questions fréquentes : Que faire si le tableau n'est pas formaté ​

Présentation complète des problèmes courants et des solutions lors de l'utilisation du plugin :

  • Tableau non formaté : Vérifiez la configuration, les exigences de version, redémarrez OpenCode
  • Structure de tableau invalide : Comprenez le sens de l'erreur "invalid structure", apprenez la syntaxe correcte des tableaux
  • Échec du formatage : Traitez l'erreur "table formatting failed" et les situations exceptionnelles
  • Types de tableaux non pris en charge : Comprenez les scénarios limités comme les tableaux HTML, les cellules multilignes, etc.
  • Désalignement après formatage : Identifiez les problèmes de mode caché, cellules trop longues, code en ligne, etc.

Suggestions de parcours d'apprentissage ​

Il est recommandé de consulter ce chapitre après avoir terminé Aperçu des fonctionnalités lorsque vous rencontrez des problèmes réels, ou de l'utiliser comme manuel de référence pour le dépannage.

Vous pouvez suivre l'ordre suivant :

  1. En cas de problème : Identifiez d'abord le type de problème en fonction des symptômes (tableau non formaté, messages d'erreur, etc.)
  2. Dépannage étape par étape : Vérifiez chaque élément de la liste de contrôle du guide de dépannage
  3. Compréhension approfondie : Lisez Spécifications des tableaux et Principe du mode caché pour comprendre les détails techniques
  4. Limitations connues : Consultez Limitations connues pour comprendre les limites du plugin

Prérequis ​

Avant d'étudier ce chapitre, il est recommandé d'avoir terminé :

  • ✅ Démarrage rapide : Installation et configuration - Comprendre comment installer et configurer le plugin
  • ✅ Aperçu des fonctionnalités : La magie du formatage automatique - Comprendre les fonctionnalités principales et le principe de fonctionnement du plugin

En outre, vous devriez posséder :

  • Une familiarité avec la syntaxe de base des tableaux Markdown
  • Une compréhension de la méthode de configuration des plugins OpenCode (opencode.jsonc)

Prochaines étapes ​

Après avoir terminé ce chapitre, vous pouvez continuer à apprendre :

  • Connaissances avancées :

    • Spécifications des tableaux : Quels tableaux peuvent être formatés - Maîtriser les exigences de structure des tableaux valides
    • Principe du mode caché : Pourquoi le calcul de la largeur est si important - Comprendre le principe de fonctionnement du mode caché d'OpenCode
    • Détails de l'alignement : Alignement à gauche, centré, alignement à droite - Maîtriser la syntaxe et les effets des trois méthodes d'alignement

  • Détails techniques :

    • Limitations connues : Où se trouvent les limites du plugin - Comprendre les limitations connues du plugin et les scénarios inapplicables
    • Détails techniques : Mécanisme de cache et optimisation des performances - Comprendre le mécanisme de cache interne et les stratégies d'optimisation des performances du plugin
