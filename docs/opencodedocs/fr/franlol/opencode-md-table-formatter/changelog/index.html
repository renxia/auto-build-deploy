Journal des modifications ​

Ce chapitre enregistre l'évolution des versions du plugin opencode-md-table-formatter, y compris les nouvelles fonctionnalités, les corrections de problèmes et les détails techniques de chaque version. Suivre ces changements peut vous aider à comprendre la direction de développement du plugin et ses limitations connues.

🎯 Suggestions de parcours d'apprentissage ​

Comme le contenu de ce chapitre est indépendant et simple, vous pouvez le lire directement dans l'ordre suivant :

  1. Lecture directe : Si vous souhaitez simplement connaître l'état de la version actuelle, lisez directement le Journal des modifications

  2. Consultation à la demande : Si vous rencontrez un problème spécifique ou souhaitez comprendre les changements d'une version particulière, vous pouvez utiliser les ancres dans la page pour vous positionner rapidement

Astuce

Le journal des modifications est un document de référence, il n'est pas nécessaire de l'apprendre du début à la fin. Consultez simplement les informations de la version concernée lorsque vous rencontrez un problème.

📖 Conditions préalables ​

Avant de lire ce chapitre, il est recommandé de comprendre d'abord :

  • Aperçu des fonctionnalités : La magie du formatage automatique - Comprendre les fonctionnalités principales du plugin aide à comprendre le contexte des changements de version
  • Limitations connues : Où se trouvent les limites du plugin - Comprendre les limitations du plugin évite de l'utiliser dans des scénarios non pris en charge

Optionnel

Si vous souhaitez simplement consulter rapidement le numéro de version et les changements clés, vous pouvez lire directement le journal des modifications sans remplir les conditions préalables.

🚀 Prochaines étapes ​

Après avoir terminé la lecture de ce chapitre, vous pouvez :

  1. Retourner à la documentation principale : Retour à la page d'accueil du tutoriel, continuer à apprendre d'autres chapitres

  2. Approfondir les détails techniques : Si vous êtes intéressé par les principes de mise en œuvre du plugin, vous pouvez lire :

    • Principe du mode de masquage : Pourquoi le calcul de la largeur est si important
    • Détails techniques : Mécanisme de cache et optimisation des performances

  3. Participer à la contribution : Si vous avez des suggestions de fonctionnalités ou souhaitez contribuer du code, n'hésitez pas à proposer vos idées sur GitHub Issues
