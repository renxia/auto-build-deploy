Auf dieser Seite

Plattformfunktionen ​

Dieser Abschnitt stellt die beiden Hauptfunktionen von Plannotator vor: Planüberprüfung und Codeüberprüfung. Sie lernen, wie Sie KI-generierte Pläne visuell überprüfen, verschiedene Arten von Kommentaren hinzufügen, Bildmarkierungen anhängen und Git-Diff-Codeänderungen überprüfen können.

Voraussetzungen ​

Bevor Sie beginnen

Stellen Sie sicher, dass Sie die folgenden Vorbereitungen abgeschlossen haben, bevor Sie mit diesem Abschnitt fortfahren:

  • ✅ Schnellstart Tutorial abgeschlossen
  • ✅ Plannotator-Plugin installiert (Claude Code oder OpenCode)

Kapitelinhalt ​

Planüberprüfung ​

Lernen Sie, wie Sie KI-generierte Ausführungspläne überprüfen, Änderungsvorschläge hinzufügen und die KI nach Ihren Absichten arbeiten lassen.

TutorialBeschreibung
Planüberprüfung-GrundlagenLernen Sie, Plannotator zu verwenden, um KI-generierte Pläne visuell zu überprüfen, einschließlich Genehmigen oder Ablehnen von Plänen
Plankommentare hinzufügenMeistern Sie das Hinzufügen verschiedener Arten von Kommentaren zu Plänen (Löschen, Ersetzen, Einfügen, Kommentieren)
Bildmarkierungen hinzufügenLernen Sie, Bilder an Planüberprüfungen anzuhängen und Stift-, Pfeil- und Kreistools für Markierungen zu verwenden

Codeüberprüfung ​

Lernen Sie, wie Sie Codeänderungen überprüfen, zeilenbasierte Kommentare hinzufügen und Probleme vor dem Committen erkennen.

TutorialBeschreibung
Codekommentare hinzufügenMeistern Sie das Hinzufügen zeilenbasierter Kommentare in Codeüberprüfungen (Kommentar/Vorschlag/Bedenken)
Diff-Ansicht wechselnLernen Sie, zwischen verschiedenen Diff-Typen in Codeüberprüfungen zu wechseln (nicht committet/staged/letzter Commit/Branch)

Lernpfad ​

Empfohlene Lernreihenfolge

Wählen Sie den passenden Lernpfad basierend auf Ihrem Anwendungsszenario:

Pfad A: Planüberprüfung priorisieren (Empfohlen für Einsteiger)

  1. Planüberprüfung-Grundlagen → Lernen Sie zuerst den grundlegenden Planüberprüfungsprozess
  2. Plankommentare hinzufügen → Lernen Sie, wie Sie Pläne präzise ändern
  3. Bildmarkierungen hinzufügen → Drücken Sie Ihre Absichten mit Bildern klarer aus
  4. Danach lernen Sie die Codeüberprüfungsreihe

Pfad B: Codeüberprüfung priorisieren (Geeignet für Entwickler mit Codeüberprüfungserfahrung)

  1. Codeüberprüfung-Grundlagen → Machen Sie sich mit der Codeüberprüfungsoberfläche vertraut
  2. Codekommentare hinzufügen → Lernen Sie zeilenbasierte Kommentare
  3. Diff-Ansicht wechseln → Meistern Sie verschiedene Diff-Typen
  4. Danach lernen Sie die Planüberprüfungsreihe

Nächste Schritte ​

Nach Abschluss dieses Abschnitts können Sie weiterlernen:

  • URL-Freigabe – Teilen Sie Pläne und Kommentare per URL für die Zusammenarbeit im Team
  • Obsidian-Integration – Speichern Sie genehmigte Pläne automatisch in Obsidian
  • Remote-Modus – Verwenden Sie Plannotator in SSH-, Devcontainer- und WSL-Umgebungen
