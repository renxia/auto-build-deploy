Plattformen und Integrationen ​

Die Kernfunktion von Antigravity Tools ist die Konvertierung mehrerer KI-Plattformprotokolle in ein einheitliches lokales API-Gateway. Dieses Kapitel beschreibt im Detail die Integrationsmethoden, Kompatibilitätsgrenzen und bewährte Praktiken für jedes Protokoll.

Inhalt dieses Kapitels ​

TutorialBeschreibung
OpenAI-kompatible APIImplementierungsstrategien für /v1/chat/completions und /v1/responses, nahtlose Integration mit OpenAI SDK
------
Gemini Native API/v1beta/models und Google SDK Endpunkte, unterstützt x-goog-api-key Kompatibilität
Imagen 3 BildgenerierungAutomatische Abbildung von OpenAI Images-Parametern size/quality, unterstützt beliebige Seitenverhältnisse
Audio-TranskriptionEinschränkungen von /v1/audio/transcriptions und Verarbeitung großer Datenvolumen
MCP EndpunkteWeb Search/Reader/Vision als aufrufbare Tools verfügbar machen
Cloudflared TunnelEin-Klick-Exposition der lokalen API ins öffentliche Internet (nicht standardmäßig sicher)

Empfohlener Lernpfad ​

Empfohlene Reihenfolge

  1. Lerne zuerst das von dir verwendete Protokoll: Wenn du Claude Code verwendest, lies zuerst Anthropic-kompatible API; wenn du OpenAI SDK verwendest, lies zuerst OpenAI-kompatible API
  2. Dann lerne Gemini Native: Verstehe die direkte Integrationsmethode des Google SDK
  3. Lerne Erweiterungsfunktionen nach Bedarf: Bildgenerierung, Audio-Transkription, MCP-Tools
  4. Zuletzt lerne Tunnel: Lies Cloudflared Tunnel, wenn du eine öffentliche Exposition benötigst

Voraussetzungen ​

Vor dem Start bitte bestätigen

  • Installation und Upgrade abgeschlossen
  • Konto hinzufügen abgeschlossen
  • Lokalen Reverse-Proxy starten abgeschlossen (mindestens Zugriff auf /healthz)

Nächste Schritte ​

Nach Abschluss dieses Kapitels kannst du weiter lernen:

  • Erweiterte Konfiguration: Modell-Routing, Quotenverwaltung, Hochverfügbarkeitsplanung und andere erweiterte Funktionen
  • Häufig gestellte Fragen: Fehlerbehebungsleitfaden für 401/404/429 und andere Fehler
