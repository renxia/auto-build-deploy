Signalチャンネル設定：signal-cliを使用した個人AIアシスタントへの接続 | Clawdbotチュートリアル ​

このチュートリアルでできること ​

このレッスンを完了すると、以下のことができるようになります：

  • ✅ signal-cliをインストール・設定する
  • ✅ ClawdbotでSignalチャンネルを設定する
  • ✅ DMとグループでAIアシスタントと対話する
  • ✅ DMペアリングメカニズムを使用してアカウントを保護する
  • ✅ 複数のSignalアカウントのサポートを設定する
  • ✅ Signalの入力インジケーター、既読スター、リアクションを使用する

現在直面している問題 ​

SignalでAIアシスタントを使用したいが、次のような問題に遭遇しています：

  • ❌ SignalとClawdbotの接続方法が不明確
  • ❌ プライバシーを心配し、データをクラウドにアップロードしたくない
  • ❌ 誰がAIアシスタントにメッセージを送信できるかを制御する方法が不明確
  • ❌ 複数のSignalアカウント間で切り替える必要がある

なぜSignalを選ぶのか？

Signalはエンドツーエンド暗号化されたインスタントメッセージングアプリです。すべての通信は暗号化されており、送信者と受信者のみがメッセージを読み取ることができます。Clawdbotはsignal-cliを介して統合し、プライバシーを保ちながらAIアシスタントの機能を享受できます。

いつこの機能を使用するか ​

Signalチャンネルの使用に適したシナリオ：

  • プライバシー優先の通信チャンネルが必要な場合
  • チームや友人グループがSignalを使用している場合
  • 個人デバイスでAIアシスタントを実行する必要がある場合（ローカル優先）
  • 保護されたDMペアリングメカニズムを介してアクセスを制御する必要がある場合

独立したSignalアカウント

独立したSignal番号をbotアカウントとして使用することを推奨します。個人の番号を使用しないでください。これにより、メッセージループ（botが自分のメッセージを無視）を回避し、仕事と個人の通信を分離できます。

🎒 開始前の準備 ​

開始する前に、次の手順が完了していることを確認してください：

前提条件

  • ✅ クイックスタートチュートリアルを完了している
  • ✅ Clawdbotがインストールされ、正常に実行できる状態である
  • ✅ 少なくとも1つのAIモデルプロバイダー（Anthropic、OpenAI、OpenRouterなど）が設定されている
  • ✅ Javaがインストールされている（signal-cliが必要）

核心コンセプト ​

ClawdbotのSignal統合はsignal-cliに基づいており、以下の方法で動作します：

  1. デーモンモード：signal-cliはバックグラウンドデーモンとして実行し、HTTP JSON-RPCインターフェースを提供
  2. イベントストリーム（SSE）：ClawdbotはServer-Sent Events（SSE）を介してSignalイベントを受信
  3. 標準化されたメッセージ：Signalメッセージは統一された内部形式に変換され、AI Agentにルーティング
  4. 決定論的ルーティング：すべての返信は元のメッセージの送信者またはグループに送信されます

重要な設計原則：

  • ローカル優先：signal-cliはあなたのデバイスで実行し、すべての通信は暗号化されています
  • マルチアカウントサポート：複数のSignalアカウントを設定可能
  • アクセス制御：デフォルトでDMペアリングメカニズムが有効。見知らぬ人は承認なしではメッセージを送信できません
  • コンテキスト分離：グループメッセージは独立したセッションコンテキストを持ち、DMと混合しません

実践チュートリアル ​

ステップ1：signal-cliをインストールする ​

なぜ必要か signal-cliはSignalのコマンドラインインターフェースであり、Clawdbotはこれを介してSignalネットワークと通信します。

インストール方法

bash
brew install signal-cli
bash
# https://github.com/AsamK/signal-cli/releases にアクセスして最新バージョンを確認
# 最新のsignal-cliリリースをダウンロード（VERSIONを実際のバージョン番号に置換）
wget https://github.com/AsamK/signal-cli/releases/download/vVERSION/signal-cli-VERSION.tar.gz

# /optディレクトリに解凍
sudo tar -xvf signal-cli-VERSION.tar.gz -C /opt/

# シンボリックリンクを作成（オプション）
sudo ln -s /opt/signal-cli-VERSION/bin/signal-cli /usr/local/bin/signal-cli
bash
# WSL2でLinuxインストール方法を使用
# 注意：WindowsはWSL2を使用しており、signal-cliのインストールはLinuxのプロセスに従います

インストールの検証

bash
signal-cli --version
# signal-cliのバージョン番号（0.13.x以降）が表示されるはずです

表示される内容：signal-cliのバージョン番号が出力されます。

Javaの要件

signal-cliにはJavaランタイムが必要です（通常Java 11以降）。Javaがインストールされ、設定されていることを確認してください：

bash
java -version
# openjdk version "11.x.x"以降が表示されるはずです

注意：具体的なJavaバージョン要件については、signal-cli公式ドキュメントを参照してください。

ステップ2：Signalアカウントを連携する ​

なぜ必要か アカウントを連携すると、signal-cliがSignal番号を代表してメッセージを送受信できるようになります。

推奨される方法：独立したSignal番号をbotアカウントとして使用してください。

連携手順

  1. リンクコマンドを生成：
bash
signal-cli link -n "Clawdbot"

-n "Clawdbot"はデバイス名を"Clawdbot"に指定します（カスタマイズ可能）。

  1. QRコードをスキャン：

コマンドを実行すると、ターミナルにQRコードが表示されます：

tsconfig: 2369:35 - warning - *! is deprecated: Use .nonNull().
  (deprecated-non-null)

To link your device, navigate to
  Signal Settings > Linked Devices > (+) Link New Device
  on your phone and scan the QR code displayed below.

████████████████████████████████████████████████
████████████████████████████████████████████████
████████████████████████████████████████████████
████████████████████████████████████████████████
████████████████████████████████████████████████
...

  1. Signalモバイルアプリで：

    • Signal設定を開く
    • "連携デバイス"（Linked Devices）を選択
    • "(+) 新しいデバイスを連携"（Link New Device）をタップ
    • ターミナルに表示されるQRコードをスキャン

表示される内容：連携が成功すると、ターミナルに次のような出力が表示されます：

INFO  Account restored (Number: +15551234567)
INFO  Configuration created at: ~/.local/share/signal-cli/data

マルチデバイスサポート

Signalは最大4つのデバイスの連携を許可しています。Clawdbotはそのうちの1つのデバイスとして実行されます。Signalモバイルアプリの"連携デバイス"でこれらのデバイスを表示・管理できます。

ステップ3：ClawdbotのSignalチャンネルを設定する ​

なぜ必要か 設定ファイルは、Clawdbotがsignal-cliに接続する方法と、Signalからのメッセージを処理する方法を指示します。

設定方法

3つの設定方法があります。最適なものを選択してください。

方法1：簡易設定（単一アカウント） ​

これは最も簡単な方法で、単一アカウントシナリオに適しています。

~/.clawdbot/clawdbot.jsonを編集：

json
{
  "channels": {
    "signal": {
      "enabled": true,
      "account": "+15551234567",
      "cliPath": "signal-cli",
      "dmPolicy": "pairing",
      "allowFrom": ["+15557654321"]
    }
  }
}

設定フィールドの説明：

フィールド値説明
enabledtrueSignalチャンネルを有効にする
account"+15551234567"Signalアカウント（E.164形式）
cliPath"signal-cli"signal-cliコマンドパス
dmPolicy"pairing"DMアクセスポリシー（デフォルト：ペアリングモード）
allowFrom["+15557654321"]DM送信許可番号のホワイトリスト

方法2：マルチアカウント設定 ​

複数のSignalアカウントを管理する必要がある場合、accountsオブジェクトを使用します：

json
{
  "channels": {
    "signal": {
      "enabled": true,
      "accounts": {
        "work": {
          "account": "+15551234567",
          "name": "Work Bot",
          "httpPort": 8080,
          "dmPolicy": "pairing",
          "allowFrom": ["+15557654321"]
        },
        "personal": {
          "account": "+15559876543",
          "name": "Personal Bot",
          "httpPort": 8081,
          "dmPolicy": "allowlist",
          "allowFrom": ["*"]
        }
      }
    }
  }
}

設定フィールドの説明：

  • 各アカウントは一意のID（work、personalなど）を持つ
  • 各アカウントは異なるポート、ポリシー、権限を持つことができる
  • nameはオプションの表示名で、CLI/UIリストに使用される

方法3：外部デーモンモード ​

signal-cliを自分で管理する場合（例：コンテナ内またはCPU共有）、自動起動を無効にします：

json
{
  "channels": {
    "signal": {
      "enabled": true,
      "httpUrl": "http://127.0.0.1:8080",
      "autoStart": false
    }
  }
}

設定フィールドの説明：

  • httpUrl：デーモンの完全なURL（httpHostとhttpPortを上書き）
  • autoStart：falseに設定するとsignal-cliの自動起動を無効化
  • Clawdbotは既に実行中のsignal-cliデーモンに接続

表示される内容：設定ファイルを保存すると、構文エラーが表示されないはずです。

設定の検証

Clawdbotは起動時に設定を検証します。設定に誤りがある場合、ログに詳細なエラー情報が表示されます。

ステップ5：最初のメッセージを送信する ​

なぜ必要か 設定が正しいかどうかを確認し、ClawdbotがSignalメッセージを受信・処理できることを確認します。

メッセージの送信

  1. Signalモバイルアプリから、bot番号にメッセージを送信します：
こんにちは、Clawdbot！

  1. 最初のメッセージ処理：

    dmPolicy="pairing"（デフォルト）の場合、見知らぬ人はペアリングコードを受け取ります：

    ❌ 未承認の送信者

続行するには、次のコードでこのペアリングを承認してください：

📝 ペアリングコード：ABC123

コードは1時間後に期限切れになります。

承認するには、次のコマンドを実行してください：
clawdbot pairing approve signal ABC123

  2. ペアリングを承認：

    bash
    clawdbot pairing list signal

    承認待ちのペアリング要求を一覧表示：

    Pending Pairings (Signal):
  ├─ ABC123
  │   ├─ Sender: +15557654321
  │   ├─ UUID: uuid:123e4567-e89b-12d3-a456-426614174000
  │   └─ Expires: 2026-01-27 12:00:00

    ペアリングを承認：

    bash
    clawdbot pairing approve signal ABC123

  3. 2回目のメッセージを送信：

    ペアリングが成功したら、再度メッセージを送信：

    こんにちは、Clawdbot！

表示される内容：

  • SignalモバイルアプリでAIからの返信を受け取ります：

    こんにちは！私はClawdbot、あなたの個人AIアシスタントです。何かお手伝いできることはありますか？

  • Gatewayログに表示されます：

    [INFO] Received Signal message from +15557654321
[INFO] Processing message through Agent...
[INFO] Sending Signal response to +15557654321

チェックポイント ✅：

  • [ ] signal-cliデーモンが実行中
  • [ ] Channelステータスが"Connected"と表示
  • [ ] メッセージ送信後にAIからの返信を受け取る
  • [ ] Gatewayログにエラー情報がない

自分のメッセージは無視されます

個人Signal番号でbotを実行している場合、botはあなたが送信したメッセージを無視します（ループ保護）。独立したbot番号を使用することを推奨します。

よくある問題 ​

問題1：signal-cliの起動に失敗する ​

問題：signal-cliデーモンを起動できない

可能な原因：

  1. Javaがインストールされていないか、バージョンが古い
  2. ポートがすでに使用されている
  3. signal-cliパスが正しくない

解決方法：

bash
# Javaバージョンを確認
java -version

# ポート使用状況を確認
lsof -i :8080  # macOS/Linux
netstat -ano | findstr :8080  # Windows (PowerShell)

# signal-cliパスを確認
which signal-cli

問題2：ペアリングコードの有効期限切れ ​

問題：ペアリングコードが1時間後に期限切れになる

解決方法：

メッセージを再送信して新しいペアリングコードを取得し、1時間以内に承認してください。

問題3：グループメッセージに応答しない ​

問題：Signalグループで@メンションしたが、応答がない

可能な原因：

  • groupPolicyがallowlistに設定されているが、groupAllowFromに含まれていない
  • Signalはネイティブな@メンションをサポートしていない（Discord/Slackとは異なる）

解決方法：

グループポリシーを設定：

json
{
  "channels": {
    "signal": {
      "groupPolicy": "allowlist",
      "groupAllowFrom": ["+15557654321"]
    }
  }
}

または、コマンドトリガーを使用（commands.config: trueが設定されている場合）：

@clawdbot help

問題4：メディアファイルのダウンロードに失敗する ​

問題：Signalメッセージ内の画像や添付ファイルを処理できない

可能な原因：

  • ファイルサイズがmediaMaxMb制限を超えている（デフォルト8MB）
  • ignoreAttachmentsがtrueに設定されている

解決方法：

json
{
  "channels": {
    "signal": {
      "mediaMaxMb": 20,
      "ignoreAttachments": false
    }
  }
}

問題5：長いメッセージが切り捨てられる ​

問題：送信されたメッセージが複数の部分に分割される

原因：Signalにはメッセージ長制限があり（デフォルト4000文字）、Clawdbotは自動的に分割します

解決方法：

分割設定を調整：

json
{
  "channels": {
    "signal": {
      "textChunkLimit": 2000,
      "chunkMode": "newline"
    }
  }
}

chunkModeオプション：

  • length（デフォルト）：長さで分割
  • newline：まず空行で分割（段落境界）、その後長さで分割

まとめ ​

このレッスンでは、Signalチャンネルの設定と使用を完了しました：

核心コンセプト：

  • Signalチャンネルはsignal-cliに基づき、HTTP JSON-RPC + SSEで通信
  • 独立したbot番号を使用することを推奨（メッセージループを回避）
  • デフォルトでDMペアリングメカニズムが有効、アカウントのセキュリティを保護

重要な設定：

  • account：Signalアカウント（E.164形式）
  • cliPath：signal-cliパス
  • dmPolicy：DMアクセスポリシー（デフォルトpairing）
  • allowFrom：DMホワイトリスト
  • groupPolicy / groupAllowFrom：グループポリシー

実用的な機能：

  • マルチアカウントサポート
  • グループメッセージ（独立したコンテキスト）
  • 入力インジケーター
  • 既読スター
  • リアクション（絵文字リアクション）

トラブルシューティング：

  • clawdbot channels statusでChannelステータスを確認
  • clawdbot pairing list signalで承認待ちのペアリング要求を確認
  • Gatewayログで詳細なエラー情報を確認
