macOS アプリ：メニューバー制御と音声対話 ​

学習完了後にできること ​

本チュートリアルを完了すると、以下のことができるようになります：

  • ✅ Clawdbot macOS アプリがメニューバー制御プレーンとしてのコア機能を理解できる
  • ✅ Voice Wake 音声ウェイクと Talk Mode 連続会話の使用方法をマスターできる
  • ✅ ノードモードでの system.run、Canvas、Camera などの能力を理解できる
  • ✅ 異なるデプロイシーンに適応するためにローカル vs リモートモードを設定できる
  • ✅ コマンド実行権限を制御するために Exec Approvals 承認メカニズムを管理できる
  • ✅ ディープリンクを使用して AI アシスタントを素早く起動できる
  • ✅ SSH/Tailscale を通じてリモートアクセスと制御 Gateway を使用できる

現在の課題 ​

あなたは今、次のように考えているかもしれません：

  • "macOS アプリは一体何をするもの？Gateway 本体？"
  • "Voice Wake と Talk Mode はどう使う？追加ハードウェアが必要？"
  • "ノードモードと普通のモードの違いは？いつどれを使う？"
  • "macOS で権限とセキュリティ設定を管理するには？"
  • "別のマシンで Gateway を実行できる？"

良いお知らせ：Clawdbot macOS アプリは Gateway のグラフィカル制御プレーンです。Gateway サービス自体は実行せず、接続、管理、監視を行います。同時に、ノードとして macOS 固有の機能（system.run、Canvas、Camera など）をリモート Gateway に公開します。

この機能を使用するタイミング ​

以下のような場合に使用します：

  • 🖥️ macOS グラフィカル管理 —— メニューバーの状態と制御、コマンドラインより直感的
  • 🎙️ 音声対話 —— Voice Wake ウェイク + Talk Mode 連続会話
  • 💻 ローカルコマンド実行 —— macOS ノードで system.run などのコマンドを実行
  • 🎨 Canvas 可視化 —— macOS で AI ドライブの可視化インターフェースをレンダリング
  • 📷 デバイス機能 —— Camera 撮影、録画、画面録画
  • 🌐 リモートアクセス —— SSH/Tailscale を通じてリモート Gateway を制御

ノード vs Gateway の違い

  • Gateway：AI モデルを実行、セッションを管理、メッセージを処理（どのマシンでも実行可能）
  • ノード（Node）：デバイスのローカル機能（Canvas、Camera、system.run）を Gateway に公開
  • macOS アプリ：Gateway クライアントとしても、ノードとしても使用可能

コアコンセプト ​

Clawdbot macOS アプリは二重の役割を持つシステムです：

┌──────────────────────────────────────────┐
│     Clawdbot.app (macOS アプリ)       │
│                                      │
│   ┌────────────────────────────┐      │
│   │  メニューバー制御プレーン  │      │
│   │  • Gateway 接続管理         │◄────► Gateway WebSocket
│   │  • WebChat 埋め込みウィンドウ│      │
│   │  • 設定と構成               │      │

│   └────────────────────────────┘      │
│                                      │
│   ┌────────────────────────────┐      │
│   │  ノードサービス            │      │
│   │  • system.run              │◄────► Gateway ノードプロトコル
│   │  • Canvas                 │      │
│   │  • Camera/Screen          │      │
│   └────────────────────────────┘      │
└──────────────────────────────────────────┘

2つの実行モード：

モードGateway の場所ノードサービス適用シーン
ローカルモード（デフォルト）ローカル（launchd デーモン）起動しないGateway がこの Mac で実行されている
リモートモードリモートマシン（SSH/Tailscale 経由）起動Gateway が他のマシンで実行されている

コア機能モジュール：

  1. メニューバー制御 —— Gateway 接続状態、WebChat、設定、セッション管理
  2. Voice Wake —— グローバル音声ウェイクワードの監視
  3. Talk Mode —— 連続音声対話ループ（音声入力 → AI 応答 → TTS 再生）
  4. ノードモード —— macOS 固有コマンドを公開（system.run、canvas.*、camera.*）
  5. Exec Approvals —— system.run コマンド実行の承認とセキュリティ制御
  6. ディープリンク —— clawdbot:// プロトコルで機能を素早く起動

一緒に進めよう ​

ステップ 1：macOS アプリのインストールと起動 ​

なぜ必要か Clawdbot macOS アプリをインストールして、メニューバー制御と音声機能を取得する必要があります。

インストール方法：

bash
brew install --cask clawdbot
bash
# https://github.com/moltbot/moltbot/releases から最新の Clawdbot.app.dmg をダウンロード
# Applications フォルダにドラッグ

初回起動：

bash
open /Applications/Clawdbot.app

以下が表示されるはずです：

  • macOS の上部メニューバーに 🦞 アイコンが表示される
  • アイコンをクリックしてドロップダウンメニューを展開
  • システムが TCC 権限リクエストダイアログをポップアップ

初回起動時の権限リクエスト

macOS アプリには以下の権限が必要です（システムが自動的にプロンプトを表示します）：

  • 通知権限 —— システム通知を表示
  • アクセシビリティ権限 —— Voice Wake とシステム操作に使用
  • マイク権限 —— Voice Wake と Talk Mode に必要
  • 画面収録権限 —— Canvas と画面収録機能
  • 音声認識権限 —— Voice Wake の音声入力
  • 自動化権限 —— AppleScript 制御（必要な場合）

これらの権限はすべて完全にローカルで使用され、どのサーバーにもアップロードされません。

ステップ 2：接続モードの設定（ローカル vs リモート） ​

なぜ必要か デプロイニーズに基づいて、ローカルモードかリモートモードかを選択します。

モード A：ローカルモード（デフォルト） ​

適用シーン：Gateway と macOS アプリが同じマシンで実行されている場合。

設定手順：

  1. メニューバーアプリで Local モードが表示されていることを確認
  2. Gateway が実行されていない場合、アプリは自動的に com.clawdbot.gateway launchd サービスを起動
  3. アプリは ws://127.0.0.1:18789 に接続

以下が表示されるはずです：

  • メニューバーアイコンが緑色（接続状態）
  • Gateway ステータスカードに "Local" が表示
  • ノードサービス未起動（ノードモードはリモートモードでのみ必要）

モード B：リモートモード ​

適用シーン：Gateway が別のマシン（サーバーや Linux VPS など）で実行されており、Mac から制御したい場合。

設定手順：

  1. メニューバーアプリで Remote モードに切り替え
  2. リモート Gateway の WebSocket アドレスを入力（例：ws://your-server:18789）
  3. 認証方法を選択（Token または Password）
  4. アプリは自動的に SSH トンネルを確立してリモート Gateway に接続

以下が表示されるはずです：

  • メニューバーアイコンが接続状態を表示（黄色/緑色/赤色）
  • Gateway ステータスカードにリモートサーバーアドレスが表示
  • ノードサービス自動起動（リモート Gateway がローカル機能を呼び出せるように）

リモートモードのトンネルメカニズム：

macOS アプリ                     リモート Gateway
    │                                  │
    ├── SSH トンネル ───────────────────► ws://remote:18789
    │                                  │
    └── ノードサービス ◄────────────────── node.invoke

リモートモードのメリット

  • 集中管理：強力なマシン上で Gateway を実行し、複数のクライアントからアクセス
  • リソース最適化：Mac を軽量に保ち、Gateway を高性能サーバーで実行
  • デバイスローカル化：Canvas、Camera などの機能は Mac 上でローカル実行

ステップ 3：メニューバー制御プレーンの使用 ​

なぜ必要か メニューバーアプリは、すべてのコア機能に素早くアクセスできるインターフェースを提供します。

コアメニュー項目：

メニューバーアイコンをクリックすると、以下が表示されます：

  1. ステータスカード

    • Gateway 接続状態（接続済み/切断/接続中）
    • 現在のモード（Local/Remote）
    • 実行中のチャンネルリスト（WhatsApp、Telegram など）

  2. クイック操作

    • Agent —— AI 対話ウィンドウを開く（Gateway を呼び出す）
    • WebChat —— 埋め込み WebChat インターフェースを開く
    • Canvas —— Canvas 可視化ウィンドウを開く
    • Settings —— 設定インターフェースを開く

  3. 機能スイッチ

    • Talk —— Talk Mode を有効/無効
    • Voice Wake —— Voice Wake を有効/無効

  4. 情報メニュー

    • Usage —— 使用統計とコストを表示
    • Sessions —— セッションリストを管理
    • Channels —— チャンネル状態を表示
    • Skills —— スキルパックを管理

以下が表示されるはずです：

  • リアルタイム更新のステータスインジケーター（緑色 = 正常、赤色 = 切断）
  • マウスホバー時に詳細な接続情報を表示
  • 任意のメニュー項目をクリックして対応する機能を素早く開く

ステップ 4：Voice Wake の設定と使用 ​

なぜ必要か Voice Wake を使用すると、音声ウェイクワードで AI アシスタントを起動でき、クリックや入力が不要になります。

Voice Wake の動作原理：

┌──────────────────────────────────┐
│   Voice Wake ランタイム        │
│                              │
│   マイク監視 ──► ウェイクワード検出 │
│                              │
│   ウェイクワード一致？             │
│       │                       │
│       ├─ はい ──► Agent 起動  │
│       │                       │
│       └─ いいえ ──► 監視継続 │
└──────────────────────────────────┘

Voice Wake の設定：

  1. Settings → Voice Wake を開く
  2. ウェイクワードを入力（デフォルト：clawd、claude、computer）
  3. 複数のウェイクワードを追加可能（カンマ区切り）
  4. Enable Voice Wake スイッチを有効にする

ウェイクワードルール：

  • ウェイクワードは Gateway に保存：~/.clawdbot/settings/voicewake.json
  • すべてのノードが同じグローバルウェイクワードリストを共有
  • 変更後はすべての接続済みデバイス（macOS、iOS、Android）にブロードキャスト

使用フロー：

  1. マイク権限が付与されていることを確認
  2. メニューバーで Voice Wake を有効にする
  3. マイクに向かってウェイクワードを言う（例："Hey clawd"）
  4. "ポン"という通知音を待つ（ウェイク成功を示す）
  5. 指示や質問を言う

以下が表示されるはずです：

  • 画面中央に Voice Wake オーバーレイが表示
  • マイク音量波形が表示
  • "Listening" 状態テキストが表示
  • AI が音声入力の処理を開始

Voice Wake のグローバル特性

ウェイクワードは Gateway レベルのグローバル設定であり、単一のデバイスに限定されません。つまり：

  • macOS でウェイクワードを変更すると、iOS と Android デバイスも同期
  • すべてのデバイスが同じウェイクワードセットを使用
  • ただし、各デバイスは個別に Voice Wake を有効/無効化可能（権限とユーザー設定に基づく）

ステップ 5：Talk Mode 連続会話の使用 ​

なぜ必要か Talk Mode は Siri/Alexa のような連続音声対話エクスペリエンスを提供し、毎回ウェイクする必要がありません。

Talk Mode 動作ループ：

監視 ──► AI 処理 ──► TTS 再生 ──► 監視
  │                                              │
  └────────────────────────────────────────┘

Talk Mode の有効化：

  1. メニューバーの Talk ボタンをクリック
  2. またはショートカットキーを使用（デフォルト：なし、Settings で設定可能）
  3. Talk Mode オーバーレイが表示

Talk Mode インターフェース状態：

状態表示説明
Listening雲パルスアニメーション + マイク音量話しかけるのを待っている
Thinking下沉アニメーションAI が思考中
Speaking放射リングアニメーション + 波紋AI が応答中（TTS 再生中）

対話制御：

  • 話すのを停止：雲アイコンをクリックして TTS 再生を停止
  • Talk Mode を終了：右上角の X ボタンをクリック
  • 音声割り込み：AI が話している時に話し始めると、自動的に再生を停止

TTS の設定：

Talk Mode は ElevenLabs を使用してテキストを音声に変換します。設定場所：~/.clawdbot/clawdbot.json

yaml
talk:
  voiceId: "elevenlabs_voice_id"  # ElevenLabs 音声 ID
  modelId: "eleven_v3"            # モデルバージョン
  apiKey: "elevenlabs_api_key"     # API キー（または環境変数を使用）
  interruptOnSpeech: true           # 音声時割り込み
  outputFormat: "mp3_44100_128"   # 出力フォーマット

ElevenLabs 設定

API キーが設定されていない場合、Talk Mode は以下を試行します：

  1. ELEVENLABS_API_KEY 環境変数
  2. Gateway shell profile のキー
  3. 最初の利用可能な ElevenLabs 音声をデフォルトとして使用

ステップ 6：ノードモードの使用 ​

なぜ必要か ノードモードを使用すると、macOS アプリがローカル機能をリモート Gateway に公開し、真のクロスデバイス協力を実現できます。

ノードモードで使用可能なコマンド：

コマンドカテゴリコマンド例機能説明
Canvascanvas.present、canvas.navigate、canvas.evalmacOS で可視化インターフェースをレンダリング
Cameracamera.snap、camera.clip写真撮影または録画
Screenscreen.record画面録画
Systemsystem.run、system.notifyShell コマンドの実行または通知の送信

ノードモードの有効化：

ノードモードは リモートモードで自動起動します。これはリモート Gateway がローカル機能を呼び出す必要があるからです。

ノードサービスを手動で起動することも可能：

bash
clawdbot node run --display-name "My Mac"

ノード権限管理：

macOS アプリは権限システムを通じて、どの機能が利用可能かを報告します：

json
{
  "canvas": true,
  "camera": true,
  "screen": true,
  "system": {
    "run": true,
    "notify": true
  }
}

AI は権限に基づいて利用可能なツールを自動的に選択します。

ステップ 7：Exec Approvals の設定（system.run セキュリティ制御） ​

なぜ必要かsystem.run は任意の Shell コマンドを実行できるため、誤操作や悪用を防ぐために承認メカニズムが必要です。

Exec Approvals 3層セキュリティモデル：

json
{
  "version": 1,
  "defaults": {
    "security": "deny",          // デフォルトポリシー：拒否
    "ask": "on-miss"           // コマンドがホワイトリストにない場合に尋ねる
  },
  "agents": {
    "main": {
      "security": "allowlist",    // メインセッション：ホワイトリストのみ許可
      "ask": "on-miss",
      "allowlist": [
        { "pattern": "/usr/bin/git" },
        { "pattern": "/opt/homebrew/*/rg" }
      ]
    }
  }
}

セキュリティポリシータイプ：

ポリシー動作適用シーン
denyすべての system.run 呼び出しを拒否高度なセキュリティ、すべてのコマンドを無効化
allowlistホワイトリスト内のコマンドのみ許可セキュリティと利便性のバランス
askホワイトリストにない場合にユーザーに承認を要求柔軟だが確認が必要

承認フロー：

AI が未承認のコマンドを実行しようとする場合：

  1. macOS アプリが承認ダイアログをポップアップ
  2. 完全なコマンドパスと引数を表示
  3. 3つのオプションを提供：
    • Allow Once —— 今回のみ許可
    • Always Allow —— ホワイトリストに追加
    • Deny —— 実行を拒否

以下が表示されるはずです：

  • 承認ダイアログにコマンド詳細が表示（例：/usr/bin/ls -la ~）
  • "Always Allow" を選択すると、次回は尋ねられない
  • "Deny" を選択すると、コマンド実行が失敗し、AI にエラーが返される

設定場所：

Exec Approvals は macOS ローカルに保存されます：

  • ファイル：~/.clawdbot/exec-approvals.json
  • 承認履歴：アプリですべての承認済み/拒否されたコマンドを表示

ステップ 8：ディープリンクの使用 ​

なぜ必要か ディープリンクを使用すると、他のアプリから Clawdbot 機能を素早く起動できます。

サポートされているディープリンクプロトコル：clawdbot://

clawdbot://agent ​

Gateway agent リクエストを起動し、ターミナルで clawdbot agent を実行するのと同じです。

パラメータ：

パラメータ説明例
message（必須）AI に送信するメッセージmessage=Hello%20from%20deep%20link
sessionKey（オプション）ターゲットセッションキー、デフォルト mainsessionKey=main
thinking（オプション）思考レベル：off|minimal|low|medium|high|xhighthinking=high
deliver/to/channel（オプション）配信チャンネルchannel=telegram
timeoutSeconds（オプション）タイムアウト時間timeoutSeconds=30
key（オプション）確認なしキー、自動化に使用key=your-secret-key

例：

bash
# 基本：メッセージを送信
open 'clawdbot://agent?message=Hello%20from%20deep%20link'

# 高度：Telegram に送信、高思考レベル、30秒タイムアウト
open 'clawdbot://agent?message=Summarize%20my%20day&to=telegram&thinking=high&timeoutSeconds=30'

# 自動化：キーを使用して確認をスキップ（スクリプトでのみ安全に使用）
open 'clawdbot://agent?message=Automated%20task&key=secure-random-string'

以下が表示されるはずです：

  • Clawdbot macOS アプリが自動的に開く（実行されていない場合）
  • Agent ウィンドウが表示され、メッセージが表示される
  • AI が処理を開始し、応答を返す

ディープリンクのセキュリティ

  • key パラメータがない場合、アプリは確認ダイアログを表示
  • 有効な key がある場合、リクエストはサイレントで実行される（自動化スクリプト用）
  • 信頼できないソースでディープリンクを使用しないでください

  • [ ] TTS 再生が明瞭（ElevenLabs API キーが必要）
  • [ ] 音声割り込み機能が正常（話す時に再生が停止）

  • [ ] "Always Allow" でホワイトリストに正しく追加される

よくある問題 ​

❌ 権限が拒否または未付与 ​

問題：

  • Voice Wake がマイクを監視できない
  • Canvas がコンテンツを表示できない
  • system.run コマンド実行が失敗

解決策：

  1. システム設定 → プライバシーとセキュリティ を開く
  2. Clawdbot または Clawdbot.app を見つける
  3. マイク、アクセシビリティ、画面収録、自動化 などの権限がすべて有効になっていることを確認
  4. Clawdbot アプリを再起動

TCC 権限トラブルシューティング

権限スイッチが有効にならない、または直ちに無効になる場合：

  • セキュリティツール（Little Snitch など）が有効になっていないか確認
  • アプリを完全にアンインストールして再インストール
  • Console.app で TCC 拒否ログを表示

❌ Gateway 接続失敗 ​

問題：

  • メニューバーアイコンが赤色表示（切断状態）
  • ステータスカードに "Connection Failed" が表示
  • WebChat が開けない

考えられる原因と解決策：

原因確認方法解決策
Gateway が起動していないclawdbot gateway status を実行Gateway サービスを起動
アドレスが間違っているWebSocket URL を確認ws://127.0.0.1:18789 またはリモートアドレスが正しいことを確認
ポートが使用されているlsof -i :18789 を実行ポートを占領しているプロセスを閉じる
認証失敗Token/Password を確認認証情報が正しいことを確認

❌ Talk Mode が使用できない ​

問題：

  • Talk Mode を有効にしても反応がない
  • TTS が再生できない
  • マイク入力ができない

解決策：

  1. ElevenLabs 設定を確認：

    • API キーが設定されていることを確認
    • キーが有効かテスト：ElevenLabs コンソールにアクセス

  2. ネットワーク接続を確認：

    • TTS にはインターネット接続が必要
    • ファイアウォールが API リクエストをブロックしていないか確認

  3. オーディオ出力を確認：

    • システム音量がオンになっていることを確認
    • デフォルト出力デバイスが正しいことを確認

❌ リモートモードでノードが接続できない ​

問題：

  • リモート Gateway が macOS 上の system.run などのコマンドを呼び出せない
  • エラーログに "Node not found" または "Node offline" が表示される

解決策：

  1. ノードサービスが実行中であることを確認：

    bash
    clawdbot nodes list
# macOS ノードが "paired" と表示されるはず

  2. SSH トンネルを確認：

    • macOS アプリの設定で SSH 接続状態を表示
    • 手動で SSH でリモート Gateway に接続できることを確認

  3. ノードサービスを再起動：

    bash
    # macOS 上で
clawdbot node restart
