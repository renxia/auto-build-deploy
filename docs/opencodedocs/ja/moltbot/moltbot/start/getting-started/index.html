クイックスタート：Clawdbot のインストール、設定、起動 ​

本チュートリアル完了後にできること ​

本チュートリアルを完了すると、以下が可能になります：

  • ✅ お使いのデバイスに Clawdbot をインストールする
  • ✅ AI モデルの認証を設定する（Anthropic / OpenAI / その他のプロバイダー）
  • ✅ Gateway デーモンを起動する
  • ✅ WebChat または設定したチャンネルで最初のメッセージを送信する

現在の課題 ​

あなたはおそらく以下のように考えているかもしれません：

  • "ローカル AI アシスタントは複雑そう、どこから始めればいい？"
  • "スマートフォン、PC など多数のデバイスがあるが、どう統合管理する？"
  • "WhatsApp/Telegram/Slack に慣れているが、これらで AI と会話できる？"

良いお知らせがあります：Clawdbot はまさにこれらの問題を解決するために設計されています。

いつこの方法を使うか ​

以下の場合に使用します：

  • 🚀 初回セットアップ：個人の AI アシスタントを初めて設定する
  • 🔧 マルチチャンネル設定：WhatsApp、Telegram、Slack、Discord など
  • 🤖 AI モデル接続：Anthropic Claude、OpenAI GPT など
  • 📱 クロスデバイス連携：macOS、iOS、Android ノード

なぜ Gateway モードを推奨するのか？

Gateway は Clawdbot のコントロールプレーンであり、以下の機能を提供します：

  • すべてのセッション、チャンネル、ツール、イベントを統合管理
  • マルチクライアントの同時接続をサポート
  • デバイスノードによるローカル操作の実行を許可

🎒 開始前の準備 ​

システム要件 ​

コンポーネント要件
Node.js≥ 22.12.0
オペレーティングシステムmacOS / Linux / Windows (WSL2)
パッケージマネージャーnpm / pnpm / bun

Windows ユーザーへ

Windows では WSL2 の使用を強く推奨します。理由：

  • 多くのチャンネルがローカルバイナリファイルに依存している
  • デーモン（launchd/systemd）は Windows では利用不可

推奨 AI モデル ​

どのモデルもサポートされていますが、以下を強く推奨します：

プロバイダー推奨モデル理由
AnthropicClaude Opus 4.5長いコンテキストの利点、より強力なプロンプトインジェクション耐性
OpenAIGPT-5.2 + Codex高いプログラミング能力、マルチモーダルサポート

コアコンセプト ​

Clawdbot のアーキテクチャは非常にシンプルです：1 つの Gateway、複数のチャンネル、1 つの AI アシスタント。

WhatsApp / Telegram / Slack / Discord / Signal / iMessage / WebChat
                │
                ▼
        ┌──────────────────┐
        │   Gateway       │  ← コントロールプレーン（デーモン）
        │   127.0.0.1:18789 │
        └────────┬─────────┘
                 │
                 ├─→ AI Agent (pi-mono RPC)
                 ├─→ CLI (clawdbot ...)
                 ├─→ WebChat UI
                 └─→ macOS / iOS / Android ノード

重要な概念：

概念役割
Gatewayデーモン。セッション管理、チャンネル接続、ツール呼び出しを担当
Channelメッセージチャンネル（WhatsApp、Telegram、Slack など）
AgentAI ランタイム（pi-mono ベースの RPC モード）
Nodeデバイスノード（macOS/iOS/Android）。デバイスローカル操作を実行

手順に従って実行 ​

ステップ 1：Clawdbot のインストール ​

理由 グローバルインストール後、clawdbot コマンドはどこでも使用できます。

方法 A：npm を使用（推奨） ​

bash
npm install -g clawdbot@latest

方法 B：pnpm を使用 ​

bash
pnpm add -g clawdbot@latest

方法 C：bun を使用 ​

bash
bun install -g clawdbot@latest

表示されるはずの内容：

added 1 package, and audited 1 package in 3s

開発者向けオプション

ソースコードから開発またはコントリビュートする場合は、付録：ソースからビルドに進んでください。

ステップ 2：オンボーディングウィザードを実行 ​

理由 ウィザードは、Gateway、チャンネル、スキルのすべての必要な設定をガイドします。

ウィザードを起動（推奨） ​

bash
clawdbot onboard --install-daemon

ウィザードが尋ねる内容：

ステップ質問説明
1AI モデル認証方式の選択OAuth / API Key
2Gateway の設定（ポート、認証）デフォルト：127.0.0.1:18789
3チャンネルの設定（WhatsApp、Telegram など）スキップ可能、後で設定
4スキルの設定（オプション）スキップ可能

表示されるはずの内容：

✓ Gateway configured
✓ Workspace initialized: ~/clawd
✓ Channels configured
✓ Skills installed

To start the gateway, run:
  clawdbot gateway

デーモンとは？

--install-daemon は Gateway デーモンをインストールします：

  • macOS: launchd サービス（ユーザーレベル）
  • Linux: systemd ユーザーサービス

これにより、Gateway はバックグラウンドで自動的に実行され、手動で起動する必要がありません。

ステップ 3：Gateway を起動 ​

理由 Gateway は Clawdbot のコントロールプレーンであり、先に起動する必要があります。

フォアグラウンドで起動（デバッグ用） ​

bash
clawdbot gateway --port 18789 --verbose

表示されるはずの内容：

[clawdbot] Gateway started
[clawdbot] Listening on ws://127.0.0.1:18789
[clawdbot] Ready to accept connections

バックグラウンドで起動（推奨） ​

ウィザードで --install-daemon を使用した場合、Gateway は自動的に起動します。

状態を確認：

bash
clawdbot gateway status

表示されるはずの内容：

Gateway is running
PID: 12345
Port: 18789

よく使うオプション

  • --port 18789：Gateway ポートを指定（デフォルト 18789）
  • --verbose：詳細ログを有効化（デバッグ時に有用）
  • --reset：Gateway を再起動（セッションをクリア）

ステップ 4：最初のメッセージを送信 ​

理由 インストールが成功したか確認し、AI アシスタントの応答を体験します。

方法 A：CLI を使用して直接会話 ​

bash
clawdbot agent --message "Ship checklist" --thinking high

表示されるはずの内容：

[clawdbot] Agent is thinking...
[clawdbot] 🚢 Ship checklist:
1. Check Node.js version (≥ 22)
2. Install Clawdbot globally
3. Run onboarding wizard
4. Start Gateway
5. Send test message

方法 B：チャンネル経由でメッセージを送信 ​

ウィザードでチャンネル（WhatsApp、Telegram など）を設定した場合、対応するアプリで AI アシスタントに直接メッセージを送信できます。

WhatsApp の例：

  1. WhatsApp を開く
  2. Clawdbot の番号を検索
  3. メッセージを送信：Hello, I'm testing Clawdbot!

表示されるはずの内容：

  • AI アシスタントがメッセージに返信

DM ペアリング保護

デフォルトで、Clawdbot は DM ペアリング保護 を有効にしています：

  • 未知の送信者はペアリングコードを受信
  • ペアリングが承認されるまでメッセージは処理されません

詳細：DM ペアリングとアクセス制御

チェックポイント ✅ ​

上記のステップを完了したら、以下ができるはずです：

  • [ ] clawdbot --version を実行してバージョン番号を確認
  • [ ] clawdbot gateway status を実行して Gateway が実行中であることを確認
  • [ ] CLI 経由でメッセージを送信し、AI からの返信を受信
  • [ ] （オプション）設定したチャンネルでメッセージを送信し、AI からの返信を受信

よくある問題

Q: Gateway の起動に失敗？ A: ポートが使用中か確認：

bash
lsof -i :18789  # macOS/Linux
netstat -ano | findstr :18789  # Windows

Q: AI が返信しない？ A: API Key が正しく設定されているか確認：

bash
clawdbot models list

Q: 詳細ログを確認する方法は？ A: 起動時に --verbose を追加：

bash
clawdbot gateway --verbose

よくある落とし穴 ​

❌ デーモンのインストールを忘れた ​

誤った方法：

bash
clawdbot onboard  # --install-daemon を忘れた

正しい方法：

bash
clawdbot onboard --install-daemon

フォアグラウンド vs バックグラウンド

  • フォアグラウンド：デバッグに適合。ターミナルを閉じると Gateway が停止
  • バックグラウンド：本番環境に適合。自動的に再起動

❌ Node.js バージョンが低すぎる ​

誤った方法：

bash
node --version
# v20.x.x  # 古すぎる

正しい方法：

bash
node --version
# v22.12.0 以上

❌ 設定ファイルのパスが間違っている ​

Clawdbot のデフォルト設定ファイルの場所：

オペレーティングシステム設定パス
macOS/Linux~/.clawdbot/clawdbot.json
Windows (WSL2)~/.clawdbot/clawdbot.json

設定ファイルを手動で編集する場合は、パスが正しいことを確認してください。

付録：ソースからビルド ​

ソースコードから開発またはコントリビュートする場合は、以下の手順を実行：

1. リポジトリをクローン ​

bash
git clone https://github.com/clawdbot/clawdbot.git
cd clawdbot

2. 依存関係をインストール ​

bash
pnpm install

3. UI をビルド（初回実行時） ​

bash
pnpm ui:build  # UI 依存関係を自動インストール

4. TypeScript をビルド ​

bash
pnpm build

5. オンボーディングを実行 ​

bash
pnpm clawdbot onboard --install-daemon

6. 開発ループ（自動リロード） ​

bash
pnpm gateway:watch  # TypeScript ファイル変更時に自動リロード

開発モード vs 本番モード

  • pnpm clawdbot ...：TypeScript を直接実行（開発モード）
  • pnpm build 後：dist/ ディレクトリを生成（本番モード）
