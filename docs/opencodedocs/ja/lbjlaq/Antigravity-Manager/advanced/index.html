高度な設定 ​

この章では Antigravity Tools の高度な機能を深く解説します：設定管理、セキュリティポリシー、アカウントスケジューリング、モデルルーティング、クォータガバナンス、モニタリング統計、およびサーバーデプロイメント戦略。これらをマスターすると、Antigravity Tools を「使える」から「使いやすく、安定した、運用可能」レベルにアップグレードできます。

この章に含まれる内容 ​

チュートリアル説明
設定完全解説AppConfig/ProxyConfig の完全なフィールド、保存場所とホットリロードの意味論
セキュリティとプライバシーauth_mode、allow_lan_access とセキュリティベースライン設計
高可用スケジューリングローテーション、固定アカウント、スティッキーセッションと失敗時リトライメカニズム
モデルルーティングカスタムマッピング、ワイルドカード優先順位とプリセット戦略
クォータガバナンスQuota Protection + Smart Warmup の組み合わせ戦術
Proxy Monitorリクエストログ、フィルタリング、詳細復元とエクスポート
Token Statsコスト視点の統計指標とチャート解説

システム能力多言語/テーマ/更新/起動時自動起動/HTTP API Server
サーバーデプロイメントDocker noVNC vs Headless Xvfb 選定と運用

学習パスの推奨 ​

推奨順序

この章の内容は多岐にわたるため、以下のモジュールに分けて学習することを推奨します：

第一フェーズ：設定とセキュリティ（必須）

設定完全解説 → セキュリティとプライバシー
config      security

まず設定システムを理解（どれを再起動が必要か、どれがホットリロード可能か）、その後セキュリティ設定を学習（特に LAN/インターネットに公開する場合）。

第二フェーズ：スケジューリングとルーティング（推奨）

高可用スケジューリング → モデルルーティング
scheduling    model-router

最小のアカウント数で最大の安定性を得る方法を学び、その後モデルルーティングでアップストリームの変動をシールド。

第三フェーズ：クォータとモニタリング（必要に応じて）

クォータガバナンス → Proxy Monitor → Token Stats
quota        monitoring      token-stats

クォータの知覚な消費を防ぎ、呼び出しブラックボックスを観測可能なシステムに変え、コストを定量化して最適化。

第四フェーズ：安定性とデプロイメント（高度）

長時間セッションの安定性 → システム能力 → サーバーデプロイメント
context-compression  system    deployment

長時間セッションの隠れた問題を解決し、クライアントをより製品に近づけ、最後にサーバーデプロイメントを学習。

クイック選択：

あなたのシナリオまず見るべき

長い会話で頻繁にエラー長時間セッションの安定性
LAN に公開したいセキュリティとプライバシー
サーバーにデプロイしたいサーバーデプロイメント

前提条件 ​

開始前に確認してください

  • クイックスタート 章を完了済み（少なくともインストール、アカウント追加、リバースプロキシ起動を完了）
  • プラットフォームと統合 の少なくとも 1 つのプロトコル接続を完了（例：OpenAI または Anthropic）
  • ローカルリバースプロキシが正常にリクエストに応答できる

次のステップ ​

この章を学習した後、以下を続けて学習できます：

  • よくある質問：401/404/429/ストリーミング中断などの問題が発生した場合のトラブルシューティングガイド
  • 付録：エンドポイント早見表、データモデル、z.ai 能力境界などの参考資料
