macOS 应用：菜单栏控制与语音交互 ​

学完你能做什么 ​

完成本教程后，你将能够：

  • ✅ 了解 Clawdbot macOS 应用作为菜单栏控制平面的核心功能
  • ✅ 掌握 Voice Wake 语音唤醒和 Talk Mode 连续对话的使用
  • ✅ 理解节点模式下的 system.run、Canvas、Camera 等能力
  • ✅ 配置本地 vs 远程模式以适应不同部署场景
  • ✅ 管理 Exec Approvals 审批机制以控制命令执行权限
  • ✅ 使用深度链接快速触发 AI 助手
  • ✅ 通过 SSH/Tailscale 远程访问和控制 Gateway

你现在的困境 ​

你可能正在想：

  • "macOS 应用到底是做什么的？是 Gateway 本身吗？"
  • "Voice Wake 和 Talk Mode 怎么用？需要额外硬件吗？"
  • "节点模式和普通模式有什么区别？什么时候用哪个？"
  • "如何在 macOS 上管理权限和安全设置？"
  • "我可以在另一台机器上运行 Gateway 吗？"

好消息是：Clawdbot macOS 应用是 Gateway 的图形化控制平面，它不运行 Gateway 服务，而是连接、管理和监控。同时，它也作为节点暴露 macOS 特有功能（如 system.run、Canvas、Camera）给远程 Gateway。

什么时候用这一招 ​

当你需要：

  • 🖥️ macOS 图形化管理 —— 菜单栏状态和控制，比命令行更直观
  • 🎙️ 语音交互 —— Voice Wake 唤醒 + Talk Mode 连续对话
  • 💻 本地命令执行 —— 在 macOS 节点上运行 system.run 等命令
  • 🎨 Canvas 可视化 —— 在 macOS 上渲染 AI 驱动的可视化界面
  • 📷 设备功能 —— Camera 拍照、录像和屏幕录制
  • 🌐 远程访问 —— 通过 SSH/Tailscale 控制远程 Gateway

节点 vs Gateway 的区别

  • Gateway：运行 AI 模型、管理会话、处理消息（可以在任何机器上运行）
  • 节点（Node）：暴露设备本地功能（Canvas、Camera、system.run）给 Gateway
  • macOS 应用：既可以是 Gateway 客户端，也可以作为节点使用

核心思路 ​

Clawdbot macOS 应用是一个双重角色的系统：

┌──────────────────────────────────────────┐
│     Clawdbot.app (macOS 应用)       │
│                                      │
│   ┌────────────────────────────┐      │
│   │  菜单栏控制平面          │      │
│   │  • Gateway 连接管理         │◄────► Gateway WebSocket
│   │  • WebChat 内嵌窗口       │      │
│   │  • 设置和配置             │      │
│   │  • Voice Wake/Talk Mode    │      │
│   └────────────────────────────┘      │
│                                      │
│   ┌────────────────────────────┐      │
│   │  节点服务              │      │
│   │  • system.run              │◄────► Gateway 节点协议
│   │  • Canvas                 │      │
│   │  • Camera/Screen          │      │
│   └────────────────────────────┘      │
└──────────────────────────────────────────┘

两种运行模式：

模式Gateway 位置节点服务适用场景
本地模式（默认）本机（launchd 守护进程）不启动Gateway 在这台 Mac 上运行
远程模式远程机器（通过 SSH/Tailscale）启动Gateway 在其他机器上运行

核心功能模块：

  1. 菜单栏控制 —— Gateway 连接状态、WebChat、配置、会话管理
  2. Voice Wake —— 全局语音唤醒词监听
  3. Talk Mode —— 连续语音对话循环（语音输入 → AI 回复 → TTS 播放）
  4. 节点模式 —— 暴露 macOS 特有命令（system.run、canvas.*、camera.*）
  5. Exec Approvals —— system.run 命令执行审批和安全控制
  6. 深度链接 —— clawdbot:// 协议快速触发功能

跟我做 ​

第 1 步：安装和启动 macOS 应用 ​

为什么 你需要安装 Clawdbot macOS 应用来获得菜单栏控制和语音功能。

安装方式：

bash
brew install --cask clawdbot
bash
# 从 https://github.com/moltbot/moltbot/releases 下载最新的 Clawdbot.app.dmg
# 拖动到 Applications 文件夹

首次启动：

bash
open /Applications/Clawdbot.app

你应该看到：

  • macOS 顶部菜单栏出现 🦞 图标
  • 点击图标展开下拉菜单
  • 系统弹出 TCC 权限请求对话框

第一次启动的权限请求

macOS 应用需要以下权限（系统会自动弹出提示）：

  • 通知权限 —— 显示系统通知
  • 辅助功能权限 —— 用于 Voice Wake 和系统操作
  • 麦克风权限 —— Voice Wake 和 Talk Mode 需要
  • 屏幕录制权限 —— Canvas 和屏幕录制功能
  • 语音识别权限 —— Voice Wake 的语音输入
  • 自动化权限 —— AppleScript 控制（如需要）

所有这些权限都是完全本地使用，不会上传到任何服务器。

第 2 步：配置连接模式（本地 vs 远程） ​

为什么 根据你的部署需求选择本地或远程模式。

模式 A：本地模式（默认） ​

适用场景：Gateway 和 macOS 应用在同一台机器上运行。

配置步骤：

  1. 确保菜单栏应用中显示 Local 模式
  2. 如果 Gateway 未运行，应用会自动启动 com.clawdbot.gateway launchd 服务
  3. 应用会连接到 ws://127.0.0.1:18789

你应该看到：

  • 菜单栏图标显示绿色（已连接状态）
  • Gateway 状态卡片显示"Local"
  • 节点服务未启动（节点模式在远程模式下才需要）

模式 B：远程模式 ​

适用场景：Gateway 在另一台机器（如服务器或 Linux VPS）上运行，你想通过 Mac 控制它。

配置步骤：

  1. 在菜单栏应用中切换到 Remote 模式
  2. 输入远程 Gateway 的 WebSocket 地址（如 ws://your-server:18789）
  3. 选择认证方式（Token 或 Password）
  4. 应用会自动建立 SSH 隧道连接远程 Gateway

你应该看到：

  • 菜单栏图标显示连接状态（黄色/绿色/红色）
  • Gateway 状态卡片显示远程服务器地址
  • 节点服务自动启动（以便远程 Gateway 调用本地功能）

远程模式的隧道机制：

macOS 应用                     远程 Gateway
    │                                  │
    ├── SSH 隧道 ───────────────────► ws://remote:18789
    │                                  │
    └── 节点服务 ◄───────────────────── node.invoke

远程模式的优势

  • 集中管理：在一台强大的机器上运行 Gateway，多台客户端访问
  • 资源优化：Mac 可以保持轻量，Gateway 运行在高性能服务器上
  • 设备本地化：Canvas、Camera 等功能仍在 Mac 本地执行

第 3 步：使用菜单栏控制平面 ​

为什么 菜单栏应用提供了快速访问所有核心功能的界面。

核心菜单项：

点击菜单栏图标后，你会看到：

  1. 状态卡片

    • Gateway 连接状态（已连接/断开/连接中）
    • 当前模式（Local/Remote）
    • 运行的渠道列表（WhatsApp、Telegram 等）

  2. 快速操作

    • Agent —— 打开 AI 对话窗口（调用 Gateway）
    • WebChat —— 打开内嵌 WebChat 界面
    • Canvas —— 打开 Canvas 可视化窗口
    • Settings —— 打开配置界面

  3. 功能开关

    • Talk —— 启用/禁用 Talk Mode
    • Voice Wake —— 启用/禁用 Voice Wake

  4. 信息菜单

    • Usage —— 查看使用统计和成本
    • Sessions —— 管理会话列表
    • Channels —— 查看渠道状态
    • Skills —— 管理技能包

你应该看到：

  • 实时更新的状态指示器（绿色 = 正常，红色 = 断开）
  • 鼠标悬停显示详细的连接信息
  • 点击任何菜单项都能快速打开对应功能

第 4 步：配置和使用 Voice Wake ​

为什么 Voice Wake 允许你通过语音唤醒词触发 AI 助手，无需点击或输入。

Voice Wake 工作原理：

┌──────────────────────────────────┐
│   Voice Wake 运行时          │
│                              │
│   监听麦克风 ──► 检测唤醒词  │
│                              │
│   唤醒词匹配？                   │
│       │                       │
│       ├─ 是 ──► 触发 Agent  │
│       │                       │
│       └─ 否 ──► 继续监听  │
└──────────────────────────────────┘

配置 Voice Wake：

  1. 打开 Settings → Voice Wake
  2. 输入唤醒词（默认：clawd、claude、computer）
  3. 可以添加多个唤醒词（用逗号分隔）
  4. 启用 Enable Voice Wake 开关

唤醒词规则：

  • 唤醒词存储在 Gateway：~/.clawdbot/settings/voicewake.json
  • 所有节点共享同一份全局唤醒词列表
  • 更改后广播到所有连接的设备（macOS、iOS、Android）

使用流程：

  1. 确保麦克风权限已授予
  2. 在菜单栏启用 Voice Wake
  3. 对着麦克风说唤醒词（如"Hey clawd"）
  4. 等待听到"叮"提示音（表示唤醒成功）
  5. 说出你的指令或问题

你应该看到：

  • 屏幕中央出现 Voice Wake 叠加层
  • 显示麦克风音量波形
  • 显示"Listening"状态文本
  • AI 开始处理你的语音输入

Voice Wake 的全局特性

唤醒词是 Gateway 级别的全局配置，不限于单个设备。这意味着：

  • 在 macOS 上修改唤醒词后，iOS 和 Android 设备也会同步
  • 所有设备都使用同一组唤醒词
  • 但每个设备可以单独启用/禁用 Voice Wake（基于权限和用户偏好）

第 5 步：使用 Talk Mode 连续对话 ​

为什么 Talk Mode 提供了类似 Siri/Alexa 的连续语音对话体验，无需每次都唤醒。

Talk Mode 工作循环：

监听 ──► AI 处理 ──► TTS 播放 ──► 监听
  │                                              │
  └────────────────────────────────────────┘

启用 Talk Mode：

  1. 在菜单栏点击 Talk 按钮
  2. 或者使用快捷键（默认：无，可以在 Settings 中设置）
  3. Talk Mode 叠加层出现

Talk Mode 界面状态：

状态显示说明
Listening云朵脉冲动画 + 麦克风音量等待你说话
Thinking下沉动画AI 正在思考
Speaking辐射环动画 + 波纹AI 正在回复（TTS 播放中）

交互控制：

  • 停止说话：点击云朵图标停止 TTS 播放
  • 退出 Talk Mode：点击右上角 X 按钮
  • 语音中断：AI 说话时如果你开始说话，自动停止播放

配置 TTS：

Talk Mode 使用 ElevenLabs 进行文本转语音。配置位置：~/.clawdbot/clawdbot.json

yaml
talk:
  voiceId: "elevenlabs_voice_id"  # ElevenLabs 语音 ID
  modelId: "eleven_v3"            # 模型版本
  apiKey: "elevenlabs_api_key"     # API 密钥（或使用环境变量）
  interruptOnSpeech: true           # 语音时中断
  outputFormat: "mp3_44100_128"   # 输出格式

ElevenLabs 配置

如果没有配置 API 密钥，Talk Mode 会尝试使用：

  1. ELEVENLABS_API_KEY 环境变量
  2. Gateway shell profile 中的密钥
  3. 使用第一个可用的 ElevenLabs 语音作为默认

第 6 步：使用节点模式 ​

为什么 节点模式允许 macOS 应用暴露本地能力给远程 Gateway，实现真正的跨设备协作。

节点模式下的可用命令：

命令类别命令示例功能说明
Canvascanvas.present、canvas.navigate、canvas.eval在 macOS 上渲染可视化界面
Cameracamera.snap、camera.clip拍照或录像
Screenscreen.record屏幕录制
Systemsystem.run、system.notify执行 Shell 命令或发送通知

启用节点模式：

节点模式在 远程模式下自动启动，因为远程 Gateway 需要调用本地功能。

你也可以手动启动节点服务：

bash
clawdbot node run --display-name "My Mac"

节点权限管理：

macOS 应用通过权限系统报告哪些功能可用：

json
{
  "canvas": true,
  "camera": true,
  "screen": true,
  "system": {
    "run": true,
    "notify": true
  }
}

AI 会根据权限自动选择可用的工具。

第 7 步：配置 Exec Approvals（system.run 安全控制） ​

为什么system.run 可以执行任意 Shell 命令，因此需要审批机制防止误操作或滥用。

Exec Approvals 三层安全模型：

json
{
  "version": 1,
  "defaults": {
    "security": "deny",          // 默认策略：拒绝
    "ask": "on-miss"           // 命令不在白名单时询问
  },
  "agents": {
    "main": {
      "security": "allowlist",    // 主会话：仅允许白名单
      "ask": "on-miss",
      "allowlist": [
        { "pattern": "/usr/bin/git" },
        { "pattern": "/opt/homebrew/*/rg" }
      ]
    }
  }
}

安全策略类型：

策略行为适用场景
deny拒绝所有 system.run 调用高度安全，禁用所有命令
allowlist仅允许白名单中的命令平衡安全性和便利性
ask不在白名单时提示用户批准灵活但需要确认

批准流程：

当 AI 尝试执行未授权的命令时：

  1. macOS 应用弹出审批对话框
  2. 显示完整命令路径和参数
  3. 提供三个选项：
    • Allow Once —— 仅本次允许
    • Always Allow —— 添加到白名单
    • Deny —— 拒绝执行

你应该看到：

  • 审批对话框显示命令详情（如 /usr/bin/ls -la ~）
  • 选择"Always Allow"后，下次不会再次询问
  • 选择"Deny"后，命令执行失败并返回错误给 AI

配置位置：

Exec Approvals 存储在 macOS 本地：

  • 文件：~/.clawdbot/exec-approvals.json
  • 审批历史：在应用中查看所有已批准/拒绝的命令

第 8 步：使用深度链接 ​

为什么 深度链接提供了从其他应用快速触发 Clawdbot 功能的能力。

支持的深度链接协议：clawdbot://

clawdbot://agent ​

触发 Gateway agent 请求，相当于在终端运行 clawdbot agent。

参数：

参数说明示例
message（必需）发送给 AI 的消息message=Hello%20from%20deep%20link
sessionKey（可选）目标会话键，默认 mainsessionKey=main
thinking（可选）思考级别：off|minimal|low|medium|high|xhighthinking=high
deliver/to/channel（可选）送达渠道channel=telegram
timeoutSeconds（可选）超时时间timeoutSeconds=30
key（可选）免确认密钥，用于自动化key=your-secret-key

示例：

bash
# 基本：发送消息
open 'clawdbot://agent?message=Hello%20from%20deep%20link'

# 高级：发送到 Telegram，高思考级别，30 秒超时
open 'clawdbot://agent?message=Summarize%20my%20day&to=telegram&thinking=high&timeoutSeconds=30'

# 自动化：使用密钥跳过确认（仅在你的脚本中安全使用）
open 'clawdbot://agent?message=Automated%20task&key=secure-random-string'

你应该看到：

  • Clawdbot macOS 应用自动打开（如果未运行）
  • Agent 窗口出现并显示消息
  • AI 开始处理并返回响应

深度链接的安全性

  • 没有 key 参数时，应用会弹出确认对话框
  • 带有有效 key 时，请求会静默执行（用于自动化脚本）
  • 永远不要在不可信来源中使用深度链接

踩坑提醒 ​

❌ 权限被拒绝或未授予 ​

问题：

  • Voice Wake 无法监听麦克风
  • Canvas 无法显示内容
  • system.run 命令执行失败

解决方案：

  1. 打开 系统设置 → 隐私与安全性
  2. 找到 Clawdbot 或 Clawdbot.app
  3. 确保 麦克风、辅助功能、屏幕录制、自动化 等权限都已开启
  4. 重启 Clawdbot 应用

TCC 权限故障排查

如果权限开关无法启用或立即关闭：

  • 检查是否启用了任何安全工具（如 Little Snitch）
  • 尝试完全卸载应用后重新安装
  • 查看 Console.app 中的 TCC 拒绝日志

❌ Gateway 连接失败 ​

问题：

  • 菜单栏图标显示红色（断开状态）
  • 状态卡片显示"Connection Failed"
  • WebChat 无法打开

可能原因和解决方案：

原因检查方法解决方案
Gateway 未启动运行 clawdbot gateway status启动 Gateway 服务
地址错误检查 WebSocket URL确认 ws://127.0.0.1:18789 或远程地址正确
端口被占用运行 lsof -i :18789关闭占用端口的进程
认证失败检查 Token/Password确认认证凭据正确

❌ Talk Mode 无法使用 ​

问题：

  • 启用 Talk Mode 后没有反应
  • TTS 无法播放
  • 麦克风无法输入

解决方案：

  1. 检查 ElevenLabs 配置：

    • 确认 API 密钥已设置
    • 测试密钥是否有效：访问 ElevenLabs 控制台

  2. 检查网络连接：

    • TTS 需要互联网连接
    • 检查防火墙是否阻止 API 请求

  3. 检查音频输出：

    • 确认系统音量已打开
    • 检查默认输出设备是否正确

❌ 远程模式下节点无法连接 ​

问题：

  • 远程 Gateway 无法调用 macOS 上的 system.run 等命令
  • 错误日志显示"Node not found"或"Node offline"

解决方案：

  1. 确认节点服务运行：

    bash
    clawdbot nodes list
# 应该看到 macOS 节点显示为"paired"

  2. 检查 SSH 隧道：

    • 在 macOS 应用设置中查看 SSH 连接状态
    • 确认能手动 SSH 到远程 Gateway

  3. 重启节点服务：

    bash
    # 在 macOS 上
clawdbot node restart
