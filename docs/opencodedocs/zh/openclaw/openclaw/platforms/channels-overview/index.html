学完你能做什么 ​

  • 了解 OpenClaw 支持的所有消息频道

核心思路 ​

OpenClaw 是一个多频道 AI 网关，支持与多种通讯平台集成。你可以同时连接多个频道，让 AI 助手在不同平台上响应消息。

第 2 步：了解频道配置结构 ​

为什么
每个频道都有特定的配置要求。

查看频道配置类型定义 (src/config/types.channels.ts)：

typescript
type ChannelsConfig = {
  defaults?: ChannelDefaultsConfig;
  whatsapp?: WhatsAppConfig;
  telegram?: TelegramConfig;
  discord?: DiscordConfig;
  irc?: IrcConfig;
  googlechat?: GoogleChatConfig;
  slack?: SlackConfig;
  signal?: SignalConfig;
  imessage?: IMessageConfig;
  msteams?: MSTeamsConfig;
  [key: string]: any;  // 扩展频道
};

type ChannelDefaultsConfig = {
  groupPolicy?: GroupPolicy;
  heartbeat?: ChannelHeartbeatVisibilityConfig;
};

第 3 步：配置默认频道策略 ​

为什么
默认策略影响所有频道的基本行为。

bash
# 设置默认群组策略
openclaw config set channels.defaults.groupPolicy "owner-only"

# 配置心跳显示
openclaw config set channels.defaults.heartbeat.showOk false
openclaw config set channels.defaults.heartbeat.showAlerts true

群组策略选项

策略说明
owner-only仅所有者可发送命令
admins管理员和所有者可发送
everyone所有成员可发送
none群组中禁用 Agent

第 4 步：启用/禁用频道 ​

为什么
根据需求启用特定频道。

bash
# 启用 WhatsApp
openclaw config set channels.whatsapp.enabled true

# 禁用特定频道
openclaw config set channels.discord.enabled false

# 查看频道配置
openclaw config get channels

踩坑提醒 ​

常见配置问题

  1. 频道冲突
    问题：多个频道同时响应对话
    解决：配置 allowFrom 限制特定频道的访问

  2. 消息重复
    问题：同一消息被多次处理
    解决：检查 deduplication 配置

  3. 群组权限
    问题：Agent 在群组中无响应
    解决：检查 groupPolicy 设置和权限

  4. 扩展频道未加载
    问题：安装了扩展但看不到
    解决：重启 Gateway 服务
