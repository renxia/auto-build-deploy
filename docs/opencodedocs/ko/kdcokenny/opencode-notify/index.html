테마

opencode-notify
OpenCode AI가 집중하는 동안 부드럽게 알려드립니다

창 전환의 불안을 떠나보내세요. opencode-notify는 OpenCode AI를 위한 네이티브 데스크톱 알림을 제공하며, 스마트 포커스 감지, 무음 시간대, 클릭 시 포커스(macOS) 기능을 지원하여 집중해서 작업하고 AI가 필요할 때만 정확하게 알림을 받을 수 있게 해줍니다.

빠른 시작 GitHub
왜 opencode-notify를 선택하나요?

집중 작업, 스마트 알림

🔔 네이티브 데스크톱 알림

운영 체제의 네이티브 알림 시스템(macOS 알림 센터, Windows Toast, Linux notify-send)을 사용하여 워크플로우에 완벽하게 통합됩니다.

🧠 스마트 포커스 감지

터미널 창 포커스를 자동으로 감지하여 중복 알림을 방지합니다. 기본적으로 부모 세션에만 알림을 보내고 하위 작업의 노이즈를 필터링하여 진정으로 중요한 정보에 집중할 수 있게 해줍니다.

⏰ 무음 시간대 설정

무음 시간대(예: 22:00-08:00)를 설정하여 야간에 자동으로 알림을 억제하고, 자정을 넘는 시간대도 지원하여 휴식 시간을 보장합니다.

🖱️ 클릭 시 터미널 포커스(macOS)

macOS 사용자는 알림을 클릭하면 터미널 창이 자동으로 포커스되어 앱 전환 시간을 절약하고 작업 효율성을 두 배로 높일 수 있습니다.

🎨 커스텀 사운드(macOS)

다양한 유형의 이벤트에 고유한 사운드(Glass, Basso, Submarine 등)를 설정하여 사운드로 알림 유형을 빠르게 식별할 수 있습니다.

💻 37+ 터미널 지원

Ghostty, iTerm2, WezTerm, VS Code 통합 터미널 등 37개 이상의 터미널 에뮬레이터를 지원하며 터미널 유형을 자동으로 인식합니다.

🔒 로컬 실행, 제로 프라이버시 리스크

모든 알림이 로컬에서 생성되며 데이터가 클라우드에 업로드되지 않아 작업 내용이 완전히 비공개이고 안전합니다.

⚡ 경량 플러그인

TypeScript + Bun 기반으로 구축되어 설치 후 바로 사용할 수 있으며, 구성 파일이 간결하고 복잡한 구성을 배울 필요가 없습니다.

강의 개요
총 4개 챕터, 12개 강의
시작 가이드 (Start)
빠른 시작

5분 안에 설치를 완료하고 첫 번째 알림을 경험하도록 도와줍니다

작동 원리

플러그인의 핵심 기능, 스마트 필터링 메커니즘 및 알림 트리거 조건을 설명합니다

플랫폼 기능 (Platforms)
macOS 플랫폼 기능

macOS 플랫폼의 전체 기능(포커스 감지, 클릭 시 포커스, 커스텀 사운드)을 소개합니다

Windows 플랫폼 기능

Windows 플랫폼의 기능(네이티브 알림, 터미널 감지)을 소개합니다

Linux 플랫폼 기능

Linux 플랫폼의 기능(네이티브 알림, 터미널 감지)을 소개합니다

지원되는 터미널

지원되는 모든 터미널 에뮬레이터(37개 이상)를 나열합니다

고급 기능 (Advanced)
구성 참조

완전한 구성 항목 설명과 예제를 제공합니다

무음 시간대 상세 설명

특정 시간대의 방해를 피하기 위해 무음 시간대를 설정하는 방법을 설명합니다

터미널 감지 원리

터미널 자동 감지 메커니즘과 터미널 유형을 수동으로 지정하는 방법을 설명합니다

고급 사용법

구성 팁과 모범 사례를 제공합니다

자주 묻는 질문 (FAQ)
문제 해결

일반적인 문제(알림이 표시되지 않음, 포커스 감지 실패, 구성 오류 등)를 해결합니다

자주 묻는 질문

사용자의 일반적인 질문(성능 영향, 프라이버시 보안, 플러그인 충돌 등)에 답변합니다
