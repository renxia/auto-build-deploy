고급 기능 ​

본 섹션에서는 Antigravity Auth 플러그인의 고급 기능을 심층적으로 다룹니다. 다중 계정 로드 밸런싱, 스마트 계정 선택, 속도 제한 처리, 세션 복구, 요청 변환 등 핵심 메커니즘을 학습합니다. 할당량 활용 최적화부터 복잡한 문제 해결까지, 필요한 모든 내용을 다룹니다.

사전 요구사항 ​

시작하기 전에 확인하세요

  • ✅ 빠른 설치를 완료하고 첫 번째 계정을 성공적으로 추가
  • ✅ 첫 번째 인증을 완료하고 OAuth 흐름 이해
  • ✅ 첫 번째 요청을 완료하고 플러그인 정상 작동 확인

학습 경로 ​

1. 다중 계정 설정 ​

여러 Google 계정을 설정하여 할당량 풀링과 로드 밸런싱을 구현합니다.

  • 여러 계정을 추가하여 전체 할당량 한도 증가
  • 이중 할당량 시스템 이해 (Antigravity + Gemini CLI)
  • 시나리오에 맞는 적절한 계정 수 선택

2. 계정 선택 전략 ​

sticky, round-robin, hybrid 세 가지 계정 선택 전략의 모범 사례를 마스터합니다.

  • 1개 계정 → sticky 전략으로 프롬프트 캐시 유지
  • 2-3개 계정 → hybrid 전략으로 스마트 요청 분배
  • 4개 이상 계정 → round-robin 전략으로 처리량 극대화

3. 속도 제한 처리 ​

속도 제한 감지, 자동 재시도, 계정 전환 메커니즘을 이해합니다.

  • 5가지 유형의 429 오류 구분
  • 자동 재시도의 지수 백오프 알고리즘 이해
  • 다중 계정 시나리오에서의 자동 전환 로직 마스터

4. 세션 복구 ​

세션 복구 메커니즘을 이해하고, 도구 호출 실패와 중단을 자동으로 처리합니다.

  • tool_result_missing 오류 자동 처리
  • thinking_block_order 문제 수정
  • auto_resume 및 session_recovery 옵션 설정

5. 요청 변환 메커니즘 ​

요청 변환 메커니즘을 심층적으로 이해하고, 다양한 AI 모델 간의 프로토콜 차이를 호환합니다.

  • Claude와 Gemini 모델의 프로토콜 차이 이해
  • Schema 비호환으로 인한 400 오류 해결
  • 최적의 성능을 위한 Thinking 설정 최적화

6. 설정 가이드 ​

모든 설정 옵션을 마스터하고, 필요에 따라 플러그인 동작을 커스터마이징합니다.

  • 설정 파일 위치 및 우선순위
  • 모델 동작, 계정 로테이션, 애플리케이션 동작 설정
  • 단일 계정/다중 계정/병렬 에이전트 시나리오별 권장 설정

7. 병렬 에이전트 최적화 ​

병렬 에이전트 시나리오에 맞게 계정 할당을 최적화하고, PID 오프셋을 활성화합니다.

  • 병렬 에이전트 시나리오에서의 계정 충돌 문제 이해
  • PID 오프셋을 활성화하여 서로 다른 프로세스가 다른 계정을 우선 선택하도록 설정
  • round-robin 전략과 함께 다중 계정 활용률 극대화

8. 디버그 로그 ​

디버그 로그를 활성화하여 문제를 해결하고 실행 상태를 모니터링합니다.

  • 디버그 로그를 활성화하여 상세 정보 기록
  • 다양한 로그 레벨과 적용 시나리오 이해
  • 로그 내용을 해석하여 문제를 빠르게 파악

다음 단계 ​

고급 기능 학습을 완료한 후:

  • 📖 자주 묻는 질문을 참조하여 사용 중 발생하는 문제 해결
  • 📚 부록을 읽고 아키텍처 설계와 전체 설정 레퍼런스 확인
  • 🔄 변경 로그를 확인하여 최신 기능과 변경 사항 파악
