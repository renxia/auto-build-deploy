ToS 경고 ​

이 과정을 마치면 Antigravity Auth 플러그인 사용으로 발생할 수 있는 리스크를 이해하고 Google 계정을 안전하게 보호하는 방법을 알 수 있습니다.

지금 당신의 고민 ​

Antigravity Auth 플러그인을 사용해 Google Antigravity의 AI 모델에 접근하는 것을 고려 중이지만 몇 가지 우려가 있습니다:

  • 커뮤니티에서 계정 정지나 쉐도우 밴을 당했다는 보고를 봤다
  • 비공식 도구 사용이 Google의 서비스 약관을 위반하는지 걱정된다
  • 새 계정을 사용해야 할지 오래된 계정을 사용해야 할지 확실하지 않다
  • 리스크를 낮추는 방법이 궁금하다

이러한 우려는 타당합니다. 비공식 도구를 사용하는 것에는 항상 일정한 리스크가 따르며, 본 문서는 구체적인 리스크 포인트와 대응 전략을 안내합니다.

이 과정이 필요한 경우 ​

  • 플러그인 설치 전: 리스크를 이해한 후 사용 여부를 결정할 때
  • 계정 선택 시: 어떤 Google 계정으로 인증할지 결정할 때
  • 정지를 당했을 때: 가능한 원인과 예방 조치를 이해할 때
  • 새로운 사용자 등록 시: 고위험 조작 패턴을 피할 때

핵심 리스크 개요 ​

중요 경고

이 플러그인을 사용하면 Google의 서비스 약관(Terms of Service)을 위반할 수 있습니다.

일부 사용자들은 Google 계정이 정지되거나 쉐도우 밴(Shadow Ban, 명확한 알림 없이 접근 제한)되었다고 보고했습니다.

이 플러그인을 사용하는 것은 다음 선언에 동의함을 의미합니다:

  1. 이것은 비공식 도구이며 Google의 인가나 보증을 받지 않았습니다
  2. Google 계정이 일시 정지되거나 영구 정지될 수 있습니다
  3. 이 플러그인 사용의 모든 리스크와 결과는 본인이 부담합니다

쉐도우 밴이란? ​

**쉐도우 밴(Shadow Ban)**은 Google이 의심스러운 계정에 취하는 제한 조치입니다. 직접적인 정지와 달리 쉐도우 밴은 명확한 오류 메시지를 표시하지 않고:

  • API 요청이 403 또는 429 오류를 반환합니다
  • 할당량은 표시되지만 실제로 호출할 수 없습니다
  • 다른 계정은 정상적으로 작동하고, 표시된 계정만 영향을 받습니다

쉐도우 밴은 보통 상당히 오래 지속됩니다(며칠에서 몇 주), 이의 신청으로 복구할 수 없습니다.

고위험 시나리오 ​

다음 시나리오는 계정이 표시되거나 정지될 리스크를 크게 높입니다:

🚨 시나리오 1: 완전히 새로운 Google 계정 ​

위험 등급: 매우 높음

새로 등록된 Google 계정으로 이 플러그인을 사용하면 정지될 확률이 매우 높습니다. 이유:

  • 새 계정은 역사적 행동 데이터가 부족해 Google의 남용 감지 시스템에 표시되기 쉽습니다
  • 새 계정에서의 다량의 API 호출은 비정상적 행동으로 간주됩니다
  • Google은 새 계정에 대한 심사를 더 엄격하게 합니다

권장 사항: 이 플러그인을 위해 새 계정을 만들지 마세요.

🚨 시나리오 2: 새 계정 + Pro/Ultra 구독 ​

위험 등급: 매우 높음

새로 등록한 계정에 즉시 Google의 Pro 또는 Ultra 서비스를 구독하면 종종 표시되고 정지됩니다. 이유:

  • 구독 후의 높은 사용량 패턴이 새 계정에서는 남용처럼 보입니다
  • Google은 새로운 유료 사용자에 대한 심사를 더 엄격하게 합니다
  • 이 패턴은 정상 사용자의 성장 경로와 너무 다릅니다

권장 사항: 계정이 "자연 성장"할 수 있도록 시간을 주세요(최소 몇 개월), 구독을 고려하기 전에.

🟡 시나리오 3: 짧은 시간 내에 다량의 요청 ​

위험 등급: 높음

짧은 시간 내에 다량의 API 요청을 하거나, 자주 병렬 에이전트/다중 세션을 사용하면 속도 제한과 남용 감지가 트리거됩니다. 이유:

  • OpenCode의 요청 패턴은 기본 앱보다 더 집중적입니다(도구 호출, 재시도, 스트리밍 등)
  • 높은 동시 요청은 Google의 보호 메커니즘을 트리거합니다

권장 사항:

  • 요청 빈도와 동시성을 제어하세요
  • 여러 병렬 에이전트를 동시에 시작하는 것을 피하세요
  • 다중 계정 순환으로 요청을 분산하세요

🟡 시나리오 4: 유일한 Google 계정 사용 ​

위험 등급: 중간

Google 계정이 하나뿐이고, Gmail, Drive 등 핵심 서비스에 의존하는 경우 위험이 높습니다. 이유:

  • 계정이 정지되면 일상 업무에 영향을 줍니다
  • 이의 신청으로 복구할 수 없습니다
  • 대체 계획이 없습니다

권장 사항: 핵심 서비스에 의존하지 않는 별도의 계정을 사용하세요.

최적의 실천 권장 사항 ​

✅ 권장 사항 ​

1. 확립된 Google 계정 사용

일정 기간(6개월 이상 권장) 사용해 온 Google 계정을 우선적으로 선택하세요:

  • 정상적인 Google 서비스 사용 이력이 있습니다(Gmail, Drive, Google 검색 등)
  • 과거 위반 기록이 없습니다
  • 휴대폰 번호가 연결되어 검증이 완료되었습니다

2. 다중 계정 구성

여러 Google 계정을 추가하여 순환으로 요청을 분산하세요:

  • 최소 2-3개 계정을 구성하세요
  • account_selection_strategy: "hybrid" 전략(기본값)을 사용하세요
  • 속도 제한을 만나면 자동으로 계정을 전환합니다

3. 사용량 제어

  • 짧은 시간 내에 집중적인 요청을 피하세요
  • 병렬 에이전트의 수를 줄이세요
  • antigravity.json에서 max_rate_limit_wait_seconds: 0을 설정하여 재시도 대신 빠르게 실패하세요

4. 계정 상태 모니터링

정기적으로 계정 상태를 점검하세요:

  • ~/.config/opencode/antigravity-accounts.json의 rateLimitResetTimes를 확인하세요
  • 디버그 로그 활성화: OPENCODE_ANTIGRAVITY_DEBUG=1 opencode
  • 403/429 오류를 만나면 24-48시간 사용을 중지하세요

5. 먼저 공식 인터페이스에서 "워밍업"

커뮤니티 사용자가 보고한 효과적인 방법:

  1. 브라우저에서 Antigravity IDE에 로그인하세요
  2. 몇 가지 간단한 프롬프트를 실행하세요(예: "안녕", "2+2는?")
  3. 5-10회 성공 호출 후 플러그인 사용을 시작하세요

원리: 공식 인터페이스를 통해 계정을 사용하면 Google이 이를 정상 사용자 행동으로 간주하여 표시될 위험을 낮춥니다.

❌ 피해야 할 것 ​

  • ❌ 이 플러그인 전용으로 새 Google 계정 만들기
  • ❌ 새로 등록한 계정에 즉시 Pro/Ultra 구독하기
  • ❌ 유일한 핵심 서비스 계정(예: 업무 이메일) 사용하기
  • ❌ 429 제한 트리거 후 반복 재시도하기
  • ❌ 동시에 다량의 병렬 에이전트 시작하기
  • ❌ antigravity-accounts.json을 버전 관리에 제출하기

FAQ ​

Q: 계정이 정지되면 이의 신청할 수 있나요? ​

A: 아니요.

이 플러그인으로 Google의 남용 감지가 트리거되어 정지되거나 쉐도우 밴이 된 경우, 일반적으로 이의 신청으로 복구할 수 없습니다. 이유:

  • 정지는 API 사용 패턴을 기반으로 자동 트리거됩니다
  • Google은 비공식 도구 사용에 엄격한 입장을 취합니다
  • 이의 신청 시 도구 사용 목적을 설명해야 하지만, 이 플러그인 자체가 위반으로 간주될 수 있습니다

권장 사항:

  • 영향받지 않은 다른 계정 사용하기
  • 모든 계정이 정지된 경우 Antigravity IDE를 직접 사용하기
  • 정지된 계정에서 계속 시도하는 것 피하기

Q: 이 플러그인을 사용하면 반드시 정지되나요? ​

A: 아니요.

대부분의 사용자는 이 플러그인 사용에 문제가 없습니다. 리스크는 다음에 따라 다릅니다:

  • 계정 연령과 과거 행동
  • 사용 빈도와 요청 패턴
  • 최적의 실천 방법 준수 여부

리스크 평가:

  • 오래된 계정 + 적절한 사용 + 다중 계정 순환 → 낮은 리스크
  • 새 계정 + 집중적 요청 + 단일 계정 → 높은 리스크

Q: 쉐도우 밴과 속도 제한의 차이점은 무엇인가요? ​

A: 본질이 다르며 복구 방법도 다릅니다.

특징쉐도우 밴속도 제한 (429)
트리거 원인남용 감지, 의심스러운 표시요청 빈도가 할당량 초과
오류 코드403 또는 조용한 실패429 Too Many Requests
지속 시간며칠에서 몇 주몇 시간에서 하루
복구 방법복구 불가, 다른 계정 필요재설정 대기 또는 계정 전환
예방 가능 여부최적의 실천 방법으로 리스크 감소요청 빈도 제어

Q: 기업 Google 계정을 사용할 수 있나요? ​

A: 권장하지 않습니다.

기업 계정은 일반적으로 핵심 서비스와 데이터에 연결되어 있어 정지의 영향이 더 심각합니다. 또한:

  • 기업 계정의 심사가 더 엄격합니다
  • 기업의 IT 정책을 위반할 수 있습니다
  • 개인이 리스크를 감수하지만 팀에 영향을 줍니다

권장 사항: 개인 계정을 사용하세요.

Q: 다중 계정으로 완전히 정지를 피할 수 있나요? ​

A: 완전히 피할 수는 없지만 영향을 줄일 수 있습니다.

다중 계정의 역할:

  • 요청을 분산시켜 개별 계정의 제한 트리거 확률을 낮춥니다
  • 한 계정이 정지되어도 다른 계정은 계속 사용 가능합니다
  • 제한을 만났을 때 자동 전환하여 가용성을 높입니다

하지만 다중 계정은 "부적"이 아닙니다:

  • 모든 계정이 남용 감지를 트리거하면 전부 정지될 수 있습니다
  • 다중 계정을 남용하여 집중적 요청을 하지 마세요
  • 각 계정은 여전히 최적의 실천 방법을 따라야 합니다

과정 요약 ​

이 과정에서는 Antigravity Auth 플러그인 사용으로 발생할 수 있는 리스크를 다룹니다:

  1. 핵심 리스크: Google ToS를 위반할 수 있으며, 계정이 정지되거나 쉐도우 밴당할 수 있습니다
  2. 고위험 시나리오: 새 계정, 새 계정 + 구독, 집중적 요청, 유일한 핵심 계정
  3. 최적의 실천: 오래된 계정 사용, 다중 계정 구성, 사용량 제어, 상태 모니터링, 먼저 "워밍업"
  4. FAQ: 이의 신청 불가, 리스크는 개인마다 다름, 다중 계정은 영향 감소 가능

이 플러그인을 사용하기 전에 리스크를 신중하게 평가하세요. 계정 정지의 결과를 감수할 수 없다면, Antigravity IDE를 직접 사용하는 것을 권장합니다.
