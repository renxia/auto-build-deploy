부록 ​

이 장에서는 Antigravity Auth 플러그인의 기술 참고 자료를 제공합니다. 아키텍처 설계, API 사양, 저장소 형식 및 전체 구성 매뉴얼을 포함하여 플러그인 내부 메커니즘을 깊이 이해할 수 있습니다.

학습 경로 ​

1. 아키텍처 개요 ​

플러그인의 모듈 구조와 요청 처리 프로세스를 이해합니다.

  • 모듈 계층 설계 및 책임 분할
  • OpenCode에서 Antigravity API로의 요청 전체 흐름
  • 다중 계정 부하 분산 및 세션 복구 메커니즘

2. API 사양 ​

Antigravity API의 기술적 세부 사항을 심층적으로 이해합니다.

  • 통합 게이트웨이 인터페이스 및 엔드포인트 구성
  • 요청/응답 형식 및 JSON 스키마 제약 사항
  • Thinking 모델 구성 및 함수 호출 규칙

3. 저장소 형식 ​

계정 저장 파일의 구조와 버전 관리를 이해합니다.

  • 저장 파일 위치 및 각 필드 의미
  • v1/v2/v3 버전 진화 및 자동 마이그레이션
  • 다른 머신 간 계정 구성 이동 방법

4. 전체 구성 옵션 ​

모든 구성 옵션의 전체 참고 매뉴얼.

  • 30+ 구성 항목의 기본값 및 사용 시나리오
  • 환경 변수로 구성을 재정의하는 방법
  • 다양한 사용 시나리오에 따른 최적 구성 조합

사전 요구 사항 ​

먼저 완료해야 합니다

이 장의 내용은 기술적 깊이가 높으므로 먼저 다음 학습을 완료하는 것이 좋습니다:

  • 빠른 설치 - 플러그인 설치 및 첫 인증 완료
  • 구성 가이드 - 일반적인 구성 방법 이해

다음 단계 ​

부록 학습을 마친 후 다음을 수행할 수 있습니다:

  • 자주 묻는 질문을 확인하여 사용 중 발생한 문제 해결
  • 업데이트 로그를 확인하여 버전 변경 사항 파악
  • 플러그인 개발에 참여하여 코드 또는 문서 기여
