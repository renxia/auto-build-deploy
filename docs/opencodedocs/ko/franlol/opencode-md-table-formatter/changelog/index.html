업데이트 로그 ​

이 섹션은 opencode-md-table-formatter 플러그인의 버전 진화 과정을 기록하며, 각 버전의 새로운 기능, 문제 해결 및 기술 세부사항을 포함합니다. 이러한 변경 사항을 추적하면 플러그인의 발전 방향과 알려진 제한 사항을 이해하는 데 도움이 됩니다.

🎯 학습 경로 제안 ​

이 섹션의 내용은 독립적이고 간단하므로 다음 순서대로 읽을 수 있습니다:

  1. 직접 읽기: 현재 버전의 상태만 알고 싶다면 업데이트 로그를 직접 읽으세요

  2. 필요 시 참조: 특정 문제가 발생했거나 특정 버전의 변경 사항을 알고 싶다면 페이지 내의 앵커를 사용하여 빠르게 이동할 수 있습니다

팁

업데이트 로그는 참조용 문서이므로 처음부터 끝까지 학습할 필요가 없습니다. 문제가 발생했을 때 관련 버전 정보를 참조하세요.

📖 사전 요구 사항 ​

이 섹션을 읽기 전에 다음 내용을 이해하는 것이 좋습니다:

  • 기능 개요: 자동 포맷팅의 마법 - 플러그인의 핵심 기능을 이해하면 버전 변경의 배경을 이해하는 데 도움이 됩니다
  • 알려진 제한 사항: 플러그인의 경계 - 플러그인의 제한 사항을 이해하면 지원되지 않는 시나리오에서 사용하지 않을 수 있습니다

선택 사항

버전 번호와 주요 변경 사항만 빠르게 확인하려면 업데이트 로그를 직접 읽을 수 있으며, 사전 요구 사항을 완료할 필요가 없습니다.

🚀 다음 단계 ​

이 섹션을 완료한 후 다음을 수행할 수 있습니다:

  1. 메인 문서로 돌아가기: 튜토리얼 홈으로 돌아가기, 다른 섹션 계속 학습

  2. 기술 세부사항 심층 학습: 플러그인의 구현 원리에 관심이 있다면 다음을 읽을 수 있습니다:

    • 숨김 모드 원리: 왜 너비 계산이 중요한가
    • 기술 세부사항: 캐싱 메커니즘 및 성능 최적화

  3. 기여 참여: 기능 제안이 있거나 코드를 기여하고 싶다면 GitHub Issues에서 아이디어를 제출하세요
