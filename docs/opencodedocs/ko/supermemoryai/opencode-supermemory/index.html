opencode-supermemory
OpenCode Agent에 지속적인 메모리 능력 제공

OpenCode 플러그인, Supermemory 기반으로 코딩 Agent에게 지속적인 메모리 능력 제공, 세션 간, 프로젝트 간 컨텍스트 유지. 메모리 저장, 검색, 삭제 지원, 세션 시작 시 자동 컨텍스트 주입, Token 사용률 기반 선점적 압축.

빠르게 시작 GitHub
왜 opencode-supermemory인가요?

Agent가 지속적인 기억을 갖도록, 매번 세션을 처음부터 시작하는 번거로움을 해결.

메모리 지속성

Agent 세션 종료 후 컨텍스트 손실 문제 해결, 중요 정보 영구 보존, 다음 세션 자동 복원.

프로젝트 간 재사용

사용자 코딩 선호도(예: '함수형 프로그래밍 선호')를 다른 프로젝트에서도 자동으로 적용, 반복 설명 불필요.

스마트 압축

컨텍스트가 거의 다 소진될 때 자동으로 요약 및 핵심 정보 저장, '잊어버림' 방지, 세션 연속성 유지.

프라이버시 마스킹

<private> 태그 자동 마스킹 지원, 민감한 데이터를 클라우드에 업로드하지 않아 프라이버시 보호.

완전한 도구 세트

add, search, profile, list, forget 다섯 가지 도구 제공, 자연어로 메모리 제어.

자동 컨텍스트 주입

세션 시작 시 자동으로 사용자 프로필과 프로젝트 지식 주입, Agent 즉시 상태 진입, 워밍업 불필요.

과정 목록
총 4개 장, 8개 과
입문 가이드 (Start)
빠르게 시작

플러그인 설치 완료, API Key 구성, 기본 기능 검증

프로젝트 초기화

/supermemory-init 명령을 사용하여 Agent가 프로젝트 구조를 깊이 스캔하고 기억

핵심 기능 (Platforms)
자동 컨텍스트 주입

세션 시작 시 Agent가 자동으로 사용자 프로필과 프로젝트 지식을 가져오는 방법 이해

도구 세트 상세

add, search, profile 등 도구 사용 시나리오 마스터

메모리 범위

User Scope와 Project Scope의 차이 이해, 메모리 추가/삭제/수정/조회 관리

고급 기능 (Advanced)
선점적 압축

Token 임계값에 의한 자동 압축 메커니즘 깊이 이해, 컨텍스트 오버플로우 방지

깊은 구성

압축 임계값, 키워드 트리거 규칙, 검색 매개변수 사용자 정의

자주 묻는 질문 (FAQ)
프라이버시와 보안
