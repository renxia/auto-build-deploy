빠른 시작: 설치부터 첫 메시지까지 ​

Clawdbot에 오신 것을 환영합니다! 이 장에서는 Clawdbot을 처음부터 사용하는 전체 과정을 안내합니다. AI 어시스턴트를 빠르게 경험하고 싶으신 분부터 구성 옵션을 자세히 알고 싶으신 분까지, 여기에 해당하는 튜토리얼이 준비되어 있습니다.

장 개요 ​

이 장은 5개의 튜토리얼로 구성되어 있으며, Clawdbot의 완전한 시작 과정을 다룹니다. 소프트웨어 설치, AI 모델 및 통신 채널 구성부터 Gateway 데몬 실행, 첫 메시지 전송, 마지막으로 기본 보안 보호 메커니즘 이해까지. 이 장을 완료하면 정상적으로 작동하는 개인 AI 어시스턴트를 갖게 됩니다.

학습 경로 권장사항 ​

다음 순서대로 튜토리얼을 학습하는 것을 권장합니다:

    C --> D[first-message<br>첫 메시지 전송]

학습 순서 설명:

  1. 빠른 시작 (필수): 기본 설치 및 구성을 완료하며, 이는 모든 후속 학습의 전제 조건입니다
  2. 마법사 구성 (권장): 마법사의 각 옵션을 자세히 이해하며, 세밀한 구성을 원하는 사용자에게 적합합니다
  3. Gateway 실행 (필수): Gateway 데몬 프로세스를 시작하고 관리하는 방법을 알아봅니다
  4. 첫 메시지 전송 (필수): 구성이 올바른지 검증하고 AI 어시스턴트 사용을 시작합니다
  5. DM 페어링 및 접근 제어 (권장): 기본 보안 메커니즘을 이해하고 AI 어시스턴트를 보호합니다

빠른 통로

빠르게 경험하고 싶다면 「빠른 시작」과 「Gateway 실행」 두 개의 튜토리얼만 학습한 후 바로 메시지를 전송하세요. 다른 튜토리얼은 필요에 따라 나중에 학습할 수 있습니다.

전제 조건 ​

이 장을 학습하기 전에 다음을 확인하세요:

  • Node.js: ≥ 22.12.0 (node -v로 확인)
  • 운영체제: macOS / Linux / Windows (WSL2)
  • 패키지 관리자: npm / pnpm / bun
  • AI 모델 계정 (권장):
    • Anthropic Claude 계정 (Pro/Max 구독), OAuth 흐름 지원
    • 또는 OpenAI / DeepSeek / OpenRouter 등 제공업체의 API Key 준비

Windows 사용자 주의

Windows에서는 WSL2 사용을 강력히 권장합니다. 이유는:

  • 많은 채널이 로컬 바이너리 파일에 의존합니다
  • 데몬(launchd/systemd)은 Windows에서 사용할 수 없습니다

하위 페이지 탐색 ​

1. 빠른 시작 ⭐ 핵심 튜토리얼 ​

학습 완료 후 할 수 있는 것:

  • ✅ 장치에 Clawdbot 설치
  • ✅ AI 모델 인증 구성 (Anthropic / OpenAI / 다른 제공업체)
  • ✅ Gateway 데몬 실행
  • ✅ WebChat 또는 구성된 채널을 통해 첫 메시지 전송

대상 사용자: 모든 사용자, 필수 튜토리얼

예상 소요 시간: 15-20분

주요 내용:

  • npm/pnpm/bun을 사용하여 Clawdbot 설치
  • onboarding 마법사 실행하여 기본 구성 완료
  • Gateway 실행 및 상태 검증
  • CLI 또는 채널을 통해 테스트 메시지 전송

전제 조건: 없음

2. 마법사 구성 ​

학습 완료 후 할 수 있는 것:

  • ✅ 대화형 마법사를 사용하여 완전한 구성 완료
  • ✅ QuickStart와 Manual 두 가지 모드의 차이점 이해
  • ✅ Gateway 네트워크, 인증 및 Tailscale 구성
  • ✅ AI 모델 제공업체 설정 (setup-token 및 API Key)
  • ✅ 통신 채널 활성화 (WhatsApp, Telegram 등)
  • ✅ 스킬 패키지 설치 및 관리

대상 사용자: 세밀한 구성을 원하는 사용자, 고급 옵션 알아보기

예상 소요 시간: 20-30분

주요 내용:

  • QuickStart vs Manual 모드 선택
  • Gateway 네트워크 구성 (포트, 바인딩, 인증)
  • AI 모델 인증 방식 (setup-token 권장)
  • 통신 채널 구성 프로세스
  • 스킬 시스템 소개

전제 조건: 빠른 시작

3. Gateway 실행 ⭐ 핵심 튜토리얼 ​

학습 완료 후 할 수 있는 것:

  • ✅ 명령줄을 사용하여 Gateway 포그라운드 프로세스 시작
  • ✅ Gateway를 백그라운드 데몬으로 구성 (macOS LaunchAgent / Linux systemd / Windows 예약 작업)
  • ✅ 다양한 바인딩 모드 (loopback / LAN / Tailnet) 및 인증 방식 이해
  • ✅ 개발 모드와 프로덕션 모드 간 전환
  • ✅ --force를 사용하여 사용 중인 포트 강제 해제

대상 사용자: 모든 사용자, 필수 튜토리얼

예상 소요 시간: 15-20분

주요 내용:

  • 포그라운드 모드 vs 데몬 모드
  • 바인딩 모드 선택 (loopback / LAN / Tailnet / Auto)
  • 인증 방식 구성 (Token / Password / Tailscale Identity)
  • 개발 모드 (--dev) 및 프로덕션 모드
  • 서비스 관리 명령 (install / start / stop / restart)
  • 포트 충돌 처리 (--force)

전제 조건: 마법사 구성

4. 첫 메시지 전송 ⭐ 핵심 튜토리얼 ​

학습 완료 후 할 수 있는 것:

  • ✅ WebChat 인터페이스를 통해 메시지 전송
  • ✅ 구성된 채널 (WhatsApp / Telegram / Slack 등)을 통해 AI 어시스턴트와 대화
  • ✅ 메시지 라우팅 및 응답 흐름 이해
  • ✅ AI 어시스턴트를 사용하여 기본 작업 수행 (조회, 요약, 코드 생성 등)

대상 사용자: 모든 사용자, 필수 튜토리얼

예상 소요 시간: 10-15분

주요 내용:

  • WebChat 인터페이스 사용
  • 각 채널 메시지 전송 방식
  • 메시지 형식 및 응답 메커니즘
  • 일반적인 작업 예제 (정보 조회, 코드 생성, 텍스트 요약)
  • 디버깅 및 문제 해결

전제 조건: Gateway 실행

5. DM 페어링 및 접근 제어 ​

학습 완료 후 할 수 있는 것:

  • ✅ 기본 DM 페어링 보호 메커니즘 이해
  • ✅ 모르는 발신자의 페어링 요청 승인 또는 거부
  • ✅ 화이트리스트 및 블랙리스트 구성
  • ✅ 접근 제어 정책 설정
  • ✅ 페어링 모드 및 보안 모범 사례 이해

대상 사용자: 보안에 관심 있는 사용자, 권장 학습

예상 소요 시간: 10-15분

주요 내용:

  • DM 페어링 메커니즘 원리
  • 페어링 프로세스 및 사용자 경험
  • 화이트리스트 및 블랙리스트 구성
  • 접근 제어 정책 설정
  • 보안 모범 사례

전제 조건: 첫 메시지 전송

자주 묻는 질문 ​

Q: 모든 튜토리얼을 학습해야 하나요? ​

A: 꼭 그런 것은 아닙니다. 빠르게 시작하고 싶다면 「빠른 시작」과 「Gateway 실행」 두 개의 핵심 튜토리얼만 학습한 후 바로 Clawdbot 사용을 시작할 수 있습니다. 다른 튜토리얼은 필요에 따라 나중에 학습할 수 있습니다.

Q: 일부 튜토리얼을 건너뛰면 영향이 있나요? ​

A: 없습니다. 각 튜토리얼은 독립적이지만, 「빠른 시작」은 설치 및 기본 구성을 포함하는 기초이므로 먼저 학습하는 것이 좋습니다. 다른 튜토리얼은 필요에 따라 선택적으로 학습할 수 있습니다.

Q: 이미 AI 어시스턴트에 익숙하다면 기본 튜토리얼을 건너뛸 수 있나요? ​

A: 가능합니다. 이미 유사한 AI 어시스턴트 도구에 익숙하다면 「빠른 시작」을 건너뛰고, 「마법사 구성」과 「Gateway 실행」을 바로 학습하여 Clawdbot의 특정 구성 및 시작 방식을 알아볼 수 있습니다.

Q: 이 장을 완료한 후 무엇을 할 수 있나요? ​

A: 이 장을 완료하면 정상적으로 작동하는 Clawdbot 시스템을 갖게 되며, 다음을 수행할 수 있습니다:

  • WebChat 또는 여러 채널을 통해 AI 어시스턴트와 대화
  • AI가 기본 작업 수행 (정보 조회, 코드 생성, 텍스트 요약 등)
  • DM 페어링 메커니즘을 사용하여 보안 보호
  • 고급 기능 계속 학습 (다중 채널 구성, 도구 시스템, 스킬 플랫폼 등)

다음 단계 안내 ​

이 장을 완료한 후 다음을 계속 학습할 수 있습니다:

  • 다중 채널 시스템 개요: Clawdbot이 지원하는 모든 통신 채널 및 특성 알아보기
  • WhatsApp 채널: WhatsApp 채널 구성 및 사용에 대해 깊이 있게 학습
  • Telegram 채널: Telegram 채널 구성 및 사용에 대해 깊이 있게 학습
  • WebChat 인터페이스: 내장 WebChat 인터페이스 기능 알아보기
