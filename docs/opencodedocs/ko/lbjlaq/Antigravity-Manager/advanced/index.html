고급 구성 ​

이 장에서는 Antigravity Tools의 고급 기능을 심층적으로 설명합니다: 구성 관리, 보안 정책, 계정 스케줄링, 모델 라우팅, 쿼터 거버넌스, 모니터링 통계, 및 서버 배포 방안. 이 내용을 마스터한 후에는 Antigravity Tools를 "사용 가능"에서 "사용하기 쉽고, 안정적이며, 운영 가능"으로 업그레이드할 수 있습니다.

이 장 포함 ​

튜토리얼설명
설정 완벽 가이드AppConfig/ProxyConfig의 완전 필드, 저장 위치 및 핫 리로드 의미
보안 및 프라이버시auth_mode, allow_lan_access 및 보안 기선 설계
고가용성 스케줄링로테이션, 고정 계정, 스티키 세션 및 실패 재시도 메커니즘
모델 라우팅커스텀 매핑, 와일드카드 우선순위 및 프리셋 전략
쿼터 거버넌스Quota Protection + Smart Warmup의 조합 전술
Proxy Monitor요청 로그, 필터링, 상세 복원 및 내보내기
Token Stats비용 관점의 통계 지표 및 차트 해석
긴 세션 안정성컨텍스트 압축, 서명 캐시 및 툴 결과 압축
시스템 능력다국어/테마/업데이트/시작 자동 시작/HTTP API Server
서버 배포Docker noVNC vs Headless Xvfb 선형 및 운영

학습 경로 제안 ​

추천 순서

이 장의 내용이 많으므로, 다음과 같이 모듈별로 배치 학습하는 것이 좋습니다:

첫 번째 단계: 구성 및 보안(필수)

설정 완벽 가이드 → 보안 및 프라이버시
config      security

먼저 구성 시스템을 이해하고(어떤 것이 재시작이 필요하고, 어떤 것이 핫 리로드되는지), 다음 보안 설정을 학습하세요(특히 LAN/공개 네트워크에 노출할 때).

두 번째 단계: 스케줄링 및 라우팅(추천)

고가용성 스케줄링 → 모델 라우팅
scheduling    model-router

최소 계정 수로 최대 안정성을 얻는 방법을 배우고, 다음 모델 라우팅으로 업스트림 변화를 차단하세요.

세 번째 단계: 쿼터 및 모니터링(필요에 따라)

쿼터 거버넌스 → Proxy Monitor → Token Stats
quota        monitoring      token-stats

쿼터가 무의식적으로 소진되는 것을 방지하고, 호출 블랙박스를 관찰 가능한 시스템으로 만들고, 비용 최적화를 정량화합니다.

네 번째 단계: 안정성 및 배포(고급)

긴 세션 안정성 → 시스템 능력 → 서버 배포
context-compression  system    deployment

긴 세션의 은닉적 문제를 해결하고, 클라이언트를 더 제품처럼 만들고, 마지막으로 서버 배포를 학습합니다.

빠른 선택:

시나리오먼저 볼 것 추천
다중 계정 로테이션이 불안정함고가용성 스케줄링
특정 모델 이름을 고정하고 싶음모델 라우팅
쿼터가 항상 다 소진됨쿼터 거버넌스
요청 로그를 보고 싶음Proxy Monitor
토큰 소비를 통계하고 싶음Token Stats
긴 대화가 자주 오류 발생긴 세션 안정성
LAN에 노출하려고 함보안 및 프라이버시
서버에 배포하려고 함서버 배포

전제 조건 ​

  • 빠른 시작 장을 완료했습니다(최소 설치, 계정 추가, 리버스 프록시 시작 완료)
  • 플랫폼 및 통합에서 최소 하나의 프로토콜 접속을 완료했습니다(예: OpenAI 또는 Anthropic)
  • 로컬 리버스 프록시가 요청에 정상적으로 응답할 수 있습니다
