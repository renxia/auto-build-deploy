On this page

Appendix ​

This section provides advanced reference documentation and development resources for Clawdbot, including complete configuration reference, Gateway WebSocket API protocol specification, deployment options, and development guide.

Target Audience

This section is suitable for users who need deep understanding of Clawdbot's internal mechanisms, perform advanced configuration, deploy, or participate in development. If you are just getting started, we recommend completing the Quick Start chapter first.

Subpage Navigation ​

Complete Configuration Reference ​

Detailed Configuration Reference - Covers all configuration options, default values, and examples. Find complete configuration documentation for Gateway, Agent, channels, tools, and other modules.

Gateway WebSocket API Protocol ​

Protocol Specification Document - Complete specification for the Gateway WebSocket protocol, including endpoint definitions, message formats, authentication methods, and event subscription mechanisms. Ideal for developers who need to build custom clients or integrate with Gateway.

Deployment Options ​

Deployment Method Guide - Deployment methods for different platforms: local installation, Docker, VPS, Fly.io, Nix, etc. Learn how to run Clawdbot in various environments.

Development Guide ​

Developer Documentation - Build from source, plugin development, testing, and contribution workflow. Learn how to participate in Clawdbot project development or write custom plugins.

Learning Path Recommendations ​

Choose the appropriate learning path based on your needs:

Configuration and Operations Personnel ​

  1. First read Complete Configuration Reference - Understand all configurable options
  2. Refer to Deployment Options - Choose the appropriate deployment solution
  3. Consult Gateway WebSocket API documentation as needed for integration

Application Developers ​

  1. Read Gateway WebSocket API Protocol - Understand protocol mechanisms
  2. View Complete Configuration Reference - Understand how to configure related functionality
  3. Build clients based on protocol examples

Plugin/Feature Developers ​

  1. Read Development Guide - Understand development environment and build process
  2. Deep dive into Gateway WebSocket API Protocol - Understand Gateway architecture
  3. Refer to Complete Configuration Reference - Understand configuration system

Prerequisites ​

Prerequisites

Before diving into this section, it's recommended that you have completed:

  • ✅ Completed Quick Start
  • ✅ Configured at least one channel (such as WhatsApp or Telegram)
  • ✅ Understood basic AI model configuration (see AI Models and Authentication)
  • ✅ Have basic understanding of JSON configuration files and TypeScript

Next Steps Guide ​

After completing this section, you can:

  • Perform advanced configuration - Customize your Clawdbot by referring to Complete Configuration Reference
  • Deploy to production - Choose the appropriate deployment solution according to Deployment Options
  • Develop custom features - Write plugins or contribute code by referring to Development Guide
  • Deep dive into other features - Explore Advanced Features sections, such as session management, tool system, etc.

Finding Help

If you encounter problems during use, consult Troubleshooting for solutions to common issues.
