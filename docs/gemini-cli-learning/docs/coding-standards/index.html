编码原则

指导代码编写的核心原则

可读性优先

代码应该易于理解和维护

  • 使用清晰的变量和函数名
  • 保持函数简短和专注
  • 添加必要的注释
  • 遵循一致的格式

性能考虑

编写高效的代码

  • 避免不必要的计算
  • 合理使用缓存
  • 优化算法复杂度
  • 减少内存分配

安全第一

确保代码安全性

  • 验证所有输入
  • 避免代码注入
  • 处理敏感数据
  • 使用安全的依赖

测试驱动

编写可测试的代码

  • 单一职责原则
  • 依赖注入
  • 避免全局状态
  • 模块化设计

命名约定

统一的命名规范

变量命名

使用 camelCase

const userName = "john"
const isAuthenticated = true
const maxRetryCount = 3
const apiEndpoint = "https://api.example.com"

函数命名

使用 camelCase，动词开头

function getUserData() { }
function validateInput() { }
async function fetchApiData() { }
function calculateTotal() { }

类名

使用 PascalCase

class UserManager { }
class ApiClient { }
class ConfigurationLoader { }
class DatabaseConnection { }

接口名

使用 PascalCase，可选 I 前缀

interface User { }
interface IApiResponse { }
interface ConfigOptions { }
interface IEventHandler { }

常量

使用 UPPER_SNAKE_CASE

const MAX_RETRY_COUNT = 3
const API_BASE_URL = "https://api.example.com"
const DEFAULT_TIMEOUT = 5000
const ERROR_MESSAGES = { }

枚举

使用 PascalCase

enum UserRole { Admin, User, Guest }
enum HttpStatus { Ok = 200, NotFound = 404 }
enum LogLevel { Debug, Info, Warning, Error }
enum ConnectionState { Connected, Disconnected }

代码风格

TypeScript 代码风格指南

// TypeScript 代码风格指南

// 1. 缩进：使用 2 个空格
if (condition) {
  doSomething()
}

// 2. 分号：总是使用分号
const message = 'Hello, World!';

// 3. 引号：优先使用单引号
const singleQuoted = 'preferred';
const doubleQuoted = "only when needed";

// 4. 对象和数组：尾随逗号
const config = {
  apiUrl: 'https://api.example.com',
  timeout: 5000,
  retries: 3, // 尾随逗号
};

// 5. 函数声明：优先使用函数表达式
const calculateSum = (a: number, b: number): number => {
  return a + b;
};

// 6. 类型注解：明确指定类型
interface UserData {
  id: number;
  name: string;
  email: string;
  isActive: boolean;
}

// 7. 错误处理：使用 try-catch
async function fetchUserData(id: number): Promise<UserData> {
  try {
    const response = await fetch(`/api/users/${id}`);
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
    return await response.json();
  } catch (error) {
    console.error('Failed to fetch user data:', error);
    throw error;
  }
}

测试标准

测试代码的质量要求

测试结构

组织和结构化测试代码

  • 使用 describe 和 it 组织测试
  • 每个测试文件对应一个模块
  • 测试文件以 .test.ts 或 .spec.ts 结尾
  • 使用 beforeEach 和 afterEach 设置和清理

测试命名

清晰描述测试意图

  • 使用描述性的测试名称
  • 遵循 "should ... when ..." 模式
  • 包含预期行为和条件
  • 避免技术术语，使用业务语言

测试覆盖

确保充分的测试覆盖

  • 单元测试覆盖率 > 80%
  • 测试所有公共方法
  • 覆盖边界条件和错误情况
  • 包含集成测试

测试质量

编写高质量的测试

  • 每个测试只验证一个行为
  • 使用 AAA 模式（Arrange, Act, Assert）
  • 避免测试实现细节
  • 使用合适的断言方法
